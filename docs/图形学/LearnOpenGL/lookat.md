### 相机

OpenGL的默认行为（也是视图空间/摄像机空间的基准坐标系）是假定观察者位于原点 (0, 0, 0)：
注视方向：沿着 Z轴负半轴 (0, 0, -1) 看过去。
头顶方向：即上向量，沿着 Y轴正半轴 (0, 1, 0)。
右侧方向：沿着 X轴正半轴 (1, 0, 0)。 这是一个标准的右手坐标系。

相机永远固定在 (0, 0, 0) 并且死死盯着 -Z 方向，我们要想“把相机移动到位置A，并看向位置B”，实际上的做法是把整个世界（所有物体）往相反的方向移动和旋转，让它们刚好落在默认相机的视野前。

将相机从原点移动到指定相机位置，并进行旋转

$ \vec{t} =  M_{trans} M_{rotate} \vec{v}$

则我们可以进行逆运算获得视图变换矩阵，由于正交矩阵的特性，其逆等于其转置
