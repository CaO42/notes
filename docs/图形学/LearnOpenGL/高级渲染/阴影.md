## 阴影映射

和现实中一样，当一个光源的光线因为被其他物体阻挡而无法照射到某个物体时，该物体便处于阴影之中。

光栅化的渲染并没有完美的阴影算法，但是我们可以采用一些近似的算法实现不错的阴影效果。

### 平行光/方向光

我们先考虑更好处理的平行光。

阴影映射的思路非常简单：我们以**光的位置为视角进行渲染**，我们能看到的东西都将被点亮，看不见的一定是在阴影之中了。我们用一张**阴影贴图**(shadow map)来存储渲染的数据。

显然，光源能够照射到的只有阴影贴图上每个像素最近的位置，我们只需要在进行场景渲染时计算片段的与光源直接的深度与阴影贴图比较，就能确定是否处在阴影之中。因此阴影贴图只需要用到**深度**数据，可以存储为深度贴图(Depth Map)。

第一步我们需要生成一张深度贴图(Depth Map)。深度贴图是从光的透视图里渲染的深度纹理，用它计算阴影。因为我们需要将场景的渲染结果储存到一个纹理中，我们将再次需要帧缓冲。

首先，我们要为渲染的深度贴图创建一个帧缓冲对象：

```c++
unsigned int depthMapFBO;
glGenFramebuffers(1, &depthMapFBO);
```

然后，创建一个2D纹理，提供给帧缓冲的深度缓冲使用：

```c++
const unsigned int SHADOW_WIDTH = 1024, SHADOW_HEIGHT = 1024;

unsigned int depthMap;
glGenTextures(1, &depthMap);
glBindTexture(GL_TEXTURE_2D, depthMap);
glTexImage2D(GL_TEXTURE_2D, 0, GL_DEPTH_COMPONENT, 
             SHADOW_WIDTH, SHADOW_HEIGHT, 0, GL_DEPTH_COMPONENT, GL_FLOAT, NULL);//纹理格式指定为GL_DEPTH_COMPONENT
glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MIN_FILTER, GL_NEAREST);
glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MAG_FILTER, GL_NEAREST);
glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_S, GL_REPEAT); 
glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_T, GL_REPEAT);  
```

生成深度贴图不太复杂。因为我们只关心深度值，我们要把纹理格式指定为GL_DEPTH_COMPONENT。我们还要把纹理的高宽设置为1024：这是深度贴图的分辨率。

把我们把生成的深度纹理作为帧缓冲的深度缓冲：

```c++
glBindFramebuffer(GL_FRAMEBUFFER, depthMapFBO);
glFramebufferTexture2D(GL_FRAMEBUFFER, GL_DEPTH_ATTACHMENT, GL_TEXTURE_2D, depthMap, 0);//将深度贴图挂载在深度插件上

// 不包含颜色缓冲的帧缓冲对象是不完整的，所以我们需要显式告诉OpenGL我们不会渲染任何颜色数据
glDrawBuffer(GL_NONE);
glReadBuffer(GL_NONE);
glBindFramebuffer(GL_FRAMEBUFFER, 0);
```

合理配置将深度值渲染到纹理的帧缓冲后，我们就可以开始第一步了：生成深度贴图。两个步骤的完整的渲染阶段，看起来有点像这样：

```c++
// 1. 首选渲染深度贴图
glViewport(0, 0, SHADOW_WIDTH, SHADOW_HEIGHT);//由于深度贴图分辨率不同，必须先改变视口大小再渲染 否则会得到错误的结果
glBindFramebuffer(GL_FRAMEBUFFER, depthMapFBO);
glClear(GL_DEPTH_BUFFER_BIT);
ConfigureShaderAndMatrices();//设置着色器和变换矩阵
RenderScene();
glBindFramebuffer(GL_FRAMEBUFFER, 0);

// 2. 像往常一样渲染场景，但这次使用深度贴图
glViewport(0, 0, SCR_WIDTH, SCR_HEIGHT);
glClear(GL_COLOR_BUFFER_BIT | GL_DEPTH_BUFFER_BIT);
ConfigureShaderAndMatrices();
glBindTexture(GL_TEXTURE_2D, depthMap);
RenderScene();
```

`ConfigureShaderAndMatrices`表示设置着色器和矩阵的过程，在第二个步骤中，这和往常一样：确保投影矩阵和视图矩阵都已经正确设置，并且为每个物体设置相应的模型矩阵。

然而，在第一个步骤中，我们需要使用不同的投影矩阵和视图矩阵来从光源角度渲染场景。

#### 光源空间的变换

因为我们使用的是一个所有光线都平行的定向光，我们将为光源使用**正交投影矩阵**，透视图将没有任何变形：

```c++
float near_plane = 1.0f, far_plane = 7.5f;
// 这是渲染深度贴图的projection矩阵
glm::mat4 lightProjection = glm::ortho(-10.0f, 10.0f, -10.0f, 10.0f, near_plane, far_plane);
```

我们使用glm::lookAt函数来创建view矩阵，这次从光源的位置看向场景中央。

```c++
// 这是渲染阴影贴图的view矩阵 我们假定了平行光源依然从一点出发（实际上是无穷远处）
glm::mat4 lightView = glm::lookAt(glm::vec3(-2.0f, 4.0f, -1.0f), 
                                  glm::vec3( 0.0f, 0.0f,  0.0f), 
                                  glm::vec3( 0.0f, 1.0f,  0.0f));  
```

二者相结合为我们提供了一个光源空间的变换矩阵，它将每个世界空间向量变换到在光源位置可以看到的的空间；这正是我们渲染深度贴图所需要的。

```c++
// 相当于MVP矩阵中的VP矩阵
glm::mat4 lightSpaceMatrix = lightProjection * lightView;
```

projection矩阵和view矩阵决定了可视区域的范围（例如哪些东西不会被裁切），为了确保阴影渲染的正确性我们需要控制数据保证所有物体都会被包括进去。

#### 渲染至深度贴图

当我们从光的角度来渲染场景的时候，我们会用一个比较简单的着色器，这个着色器只会把顶点变换到光空间。这个简单的着色器叫做`simpleDepthShader`，就是使用下面的这个着色器：

```c++
#version 330 core
layout (location = 0) in vec3 aPos;

uniform mat4 lightSpaceMatrix;
uniform mat4 model;

void main()
{
    gl_Position = lightSpaceMatrix * model * vec4(aPos, 1.0);
}  
```

这个顶点着色器接收模型矩阵和顶点数据，使用`lightSpaceMatrix`变换到光源空间中。

由于我们没有颜色缓冲并且禁止了读取和绘制缓冲，因此生成的片段不需要进行任何处理，所以我们可以简单地使用一个空片段着色器：

```c++
#version 330 core

void main()
{             
    // gl_FragDepth = gl_FragCoord.z;
  	// opengl会默认设置深度缓冲 因此不用显式写出
}
```

现在，渲染深度/阴影贴图的过程如下所示：

```c++
simpleDepthShader.use();
glUniformMatrix4fv(lightSpaceMatrixLocation, 1, GL_FALSE, glm::value_ptr(lightSpaceMatrix));

glViewport(0, 0, SHADOW_WIDTH, SHADOW_HEIGHT);
glBindFramebuffer(GL_FRAMEBUFFER, depthMapFBO);
glClear(GL_DEPTH_BUFFER_BIT);
RenderScene(simpleDepthShader);
glBindFramebuffer(GL_FRAMEBUFFER, 0);  
```

#### 渲染阴影

我们需要先在场景渲染的顶点着色器里进行光源空间的变换。

```c++
#version 330 core
layout (location = 0) in vec3 aPos;
layout (location = 1) in vec3 aNormal;
layout (location = 2) in vec2 aTexCoords;

out VS_OUT {
    vec3 FragPos;
    vec3 Normal;
    vec2 TexCoords;
    vec4 FragPosLightSpace;
} vs_out;

uniform mat4 projection;
uniform mat4 view;
uniform mat4 model;
uniform mat4 lightSpaceMatrix; // 将光源的vp矩阵传入

void main()
{    
    vs_out.FragPos = vec3(model * vec4(aPos, 1.0));
    vs_out.Normal = transpose(inverse(mat3(model))) * aNormal;
    vs_out.TexCoords = aTexCoords;
  	// 生成片段时会自动插值，片段着色器里的fs_in.FragPosLightSpace即是片段在光源空间的位置
    vs_out.FragPosLightSpace = lightSpaceMatrix * vec4(vs_out.FragPos, 1.0);
    gl_Position = projection * view * vec4(vs_out.FragPos, 1.0);
}
```

接着我们在片段着色器中进行计算

```c++
#version 330 core
out vec4 FragColor;

in VS_OUT {
    vec3 FragPos;
    vec3 Normal;
    vec2 TexCoords;
    vec4 FragPosLightSpace; //片段对应光源空间的位置（实际上是光源的裁剪空间）
} fs_in;

uniform sampler2D diffuseTexture;
uniform sampler2D shadowMap; //光源处渲染的阴影贴图

uniform vec3 lightPos;
uniform vec3 viewPos;

float ShadowCalculation(vec4 fragPosLightSpace)
{ ... }

void main()
{           
    vec3 color = texture(diffuseTexture, fs_in.TexCoords).rgb;
    vec3 normal = normalize(fs_in.Normal);
    vec3 lightColor = vec3(1.0);
    // 环境光
    vec3 ambient = 0.15 * lightColor;
    // 漫反射
    vec3 lightDir = normalize(lightPos - fs_in.FragPos);
    float diff = max(dot(lightDir, normal), 0.0);
    vec3 diffuse = diff * lightColor;
    // 镜面高光
    vec3 viewDir = normalize(viewPos - fs_in.FragPos);
    float spec = 0.0;
    vec3 halfwayDir = normalize(lightDir + viewDir);  
    spec = pow(max(dot(normal, halfwayDir), 0.0), 64.0);
    vec3 specular = spec * lightColor;    
    // 计算阴影值
    float shadow = ShadowCalculation(fs_in.FragPosLightSpace);  
  	// shadow只影响漫反射和镜像反射光 其取值为0.0/1.0 可以避免if运算 取1时表示在阴影中 
    vec3 lighting = (ambient + (1.0 - shadow) * (diffuse + specular)) * color;    

    FragColor = vec4(lighting, 1.0);
}
```

shadowCalculation函数用于判断是否处于阴影之中，其实现如下：

```c++
float ShadowCalculation(vec4 fragPosLightSpace)
{
    // 执行透视除法
  	// 当我们使用gl_Position将顶点位置传输至片段着色器时 会自动进行透视除法
  	// 但是其他的顶点位置我们需要自己进行该步骤
    vec3 projCoords = fragPosLightSpace.xyz / fragPosLightSpace.w;
  
    // 变换到[0,1]的范围（这是NDC到深度贴图的变换 无论是纹理坐标xy 还是深度值z都需要进行变换）
    projCoords = projCoords * 0.5 + 0.5;
    // 取得最近点的深度(使用[0,1]范围下的fragPosLight当坐标)
    float closestDepth = texture(shadowMap, projCoords.xy).r; 
    // 取得当前片段在光源视角下的深度
    float currentDepth = projCoords.z;
    // 检查当前片段是否在阴影中
    float shadow = currentDepth > closestDepth  ? 1.0 : 0.0;

    return shadow;
}
```

### 阴影映射的瑕疵

我们完成了一个看起来很清晰的阴影映射，不过如果你直接用以上的代码的话，会发现画面出现了很多的问题。

#### 阴影失真

我们直接用以上的代码进行阴影映射会发现出现了非常明显的摩尔纹。

这是由于**深度贴图并不能完美的反映片段的深度值**导致的。

![img](https://learnopengl-cn.github.io/img/05/03/01/shadow_mapping_acne_diagram.png)

这张图上阶梯状的黄色方格代表阴影贴图中的纹素贴图。其中光源与平面不平行或垂直。

可以发现由于贴图分辨率，一个纹素可能对应了平面上会产生的多个片段。

当我们计算取同一个纹素的片段时，由于片段位置和计算纹素时的位置不可能相同，计算出的深度会产生误差。由于光源与平面成角度，我们会发现前面的片段计算出的深度值会小于纹素对应的中间值的深度，因此被认为是点亮的。而后面计算的片段则深度略深而被认为在阴影中。

这导致出现明暗交错的摩尔纹。

下面这张图也展示了原因：

![img](https://picx.zhimg.com/v2-7c4be5cc0842d46d6ba0a817ff157785_1440w.jpg)

#### 阴影偏移

为了解决阴影失真的问题，我们可以使用一个技巧——阴影偏移（shadow bias）。其原理示意图如下：![img](https://learnopengl-cn.github.io/img/05/03/01/shadow_mapping_acne_bias.png)

```c++
float bias = 0.005;
float shadow = currentDepth - bias > closestDepth  ? 1.0 : 0.0;

// 采用这个偏移值比固定值更好 因为不同情况下所需的偏移值不同
// 在实际应用中 bias计算中的常量应该被设置为uniform便于调试
float bias = max(0.05 * (1.0 - max(dot(normal, lightDir), 0.0)), 0.005);
```

这个技巧的关键在于currentDepth - bias，相当于我们将整个渲染的平面都向光源方向稍稍偏移了一点。

因为深度值差距很大，原本处于阴影中的片段不太会因为这个微小偏移而导致离开阴影（这会导致新的小问题）。但是原本处于光照下的片段因为这个偏移不再会被错误的认为在阴影中了。

#### 阴影悬浮

阴影偏移修复了阴影失真这个最大的问题，但是也会带来新的小问题——阴影悬浮（ Peter Panning）。

当Shadow Bias 设置得**太大**，物体的实际深度被减去太多，导致本该在阴影起点的片段被判定为“被照亮”，导致阴影明显地偏离了实际物体。这就是阴影悬浮。

一个足够**合适的bias值**本身就不太会引起明显的阴影悬浮问题。

我们也可以使用，在渲染深度贴图时采用**正面剔除**的方法来解决这个问题。

![img](https://learnopengl-cn.github.io/img/05/03/01/shadow_mapping_culling.png)

正面剔除的本质在于：**阴影失真总是发生在物体的正面**，也就是光源处能看到的面上。

当场景中都是闭合物体时，采用正面和背面作为阴影映射的深度值理论上并不会影响阴影计算。

当我们采用背面的深度来进行正面的渲染时，物体本身的**厚度**就会变成天然的**bias值**，自动消除了阴影失真的情况。

我们依然需要bias值来处理比如正面和背面交界处的阴影失真问题，但是我们不再需要设置一个过大的bias值。从而解决了阴影悬浮的问题。

值得注意的是，正面剔除只适用于完全**封闭**的物体。对于非封闭物体，以及单个平面无法正确工作。

```c++
glCullFace(GL_FRONT); // 在渲染阴影贴图前切换为正面剔除
RenderSceneToDepthMap();

glCullFace(GL_BACK); // 不要忘记设回原先的面剔除 以进行后续正常渲染
...
```

#### 越界采样

有一个先前被我们忽略的问题，这个光源的投影矩阵的平截头体是有范围的，在这个范围内我们能正常进行渲染。但是如果物体在其之外就会产生错误。

由于采用平行光，我们希望超出平截头体范围的地方都能被照亮。

对于平截头体的xy方向，我们可以通过设置纹理类型的方式解决。

```c++
// GL_CLAMP_TO_BORDER表示纹理坐标超出范围时 取边界上的颜色值
glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_S, GL_CLAMP_TO_BORDER);
glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_T, GL_CLAMP_TO_BORDER);

// 我们手动为其设置一个深度值为1.0的边界颜色 表示无阴影
// 如果我们采样深度贴图[0,1]范围以外的区域，纹理函数总会返回一个1.0的深度值，由此得到的阴影值始终为0.0。
float borderColor[] = { 1.0, 1.0, 1.0, 1.0 };
glTexParameterfv(GL_TEXTURE_2D, GL_TEXTURE_BORDER_COLOR, borderColor);
```

但是这样还无法解决所有问题，对于z方向，我们可以在片段着色器加入条件判断解决。

```c++
// 片段着色器中的阴影判断函数
float ShadowCalculation(vec4 fragPosLightSpace)
{
    ...
    if(projCoords.z > 1.0 || projCoords.z < 0.0)
        shadow = 0.0; // 将超出视锥的部分默认点亮
		...
    return shadow;
}
/*
也可以在OpenGL上下文开启该选项 会将深度限制在[0,1]中
glEnable(GL_DEPTH_CLAMP);
*/
```

### PCF

关于阴影映射，还有一些问题。首先是由于纹理的分辨率有限，不可避免会出现**锯齿**现象。我们希望通过一个滤波/模糊操作来减轻锯齿现象。

另外，我们当前渲染的阴影被称为**硬阴影**。这代表这这种阴影明暗分界清晰，不存在从明到暗的过渡部分。这种阴影对于平行光来说是可以认为正确的，但当我们之后遇到点光源时，由于点光源的存在体积，实际上会存在明-明暗过渡-暗这样的结构。我们称这种阴影为**软阴影**。

**PCF——百分比渐进滤波(percentage-closer filtering)**是一种能够同时解决以上两个问题的算法。

#### PCF 的基本算法流程

在**普通阴影映射**里，为了判断屏幕上一个像素 `(x,y)` 是否在阴影中，我们会：

1. 把该点变换到光源空间，得到它在阴影贴图上的坐标 `(u,v)` 及该点相对于光源的深度 `d`；
2. 在阴影贴图的 `(u,v)` 处采样一个深度值 `z`；
3. 如果 `d - bias > z`，判定为“在阴影中”，结果为 `0`；否则为“被照亮”，结果为 `1`。

这样每个像素要么全亮，要么全暗，阴影边缘是硬过渡，也必然出现锯齿。

**PCF 的做法是：**

- 不要只在 `(u,v)` 处采一次样，而是在 `(u,v)` 周围的一个小邻域（如 3×3、5×5 的核）里进行多次采样；
- 对邻域里的每一个采样点，都独立执行一次深度比较，得到一系列 `0` 或 `1` 的结果；
- 对这个结果数组取平均值（或加权平均），得到一个 `[0,1]` 之间的浮点数值，作为该像素的阴影强度。

这个名字“**百分比渐近滤波**”的来源正是：它统计的是“邻域中**通过深度测试、更接近光源**的样本所占的**百分比**”，再用这个百分比来调制光照，实现从亮到暗的逐渐过渡。

```c++
// 使用3*3的核演示
float shadow = 0.0;
vec2 texelSize = 1.0 / textureSize(shadowMap, 0);
for(int x = -1; x <= 1; ++x)
{
    for(int y = -1; y <= 1; ++y)
    {
        float pcfDepth = texture(shadowMap, projCoords.xy + vec2(x, y) * texelSize).r; 
        shadow += currentDepth - bias > pcfDepth ? 1.0 : 0.0;        
    }    
}
shadow /= 9.0;
```

该算法能够解决抗锯齿，只要取决于其本质是对阴影贴图进行了一个低通滤波操作。去掉了高频信号。（在图片中高频为变化快的区域，也就是模糊了阴影边界）

而对于软阴影，则是由于其无意中模拟了片段的可见度计算，计算出的阴影值即为近似的可见百分比。而软阴影正是取决于该处光照的强度。不过值得注意的是，这是一种相当粗略的近似，并不能得到很好的效果。但是就目前的学习阶段已经很不错了。

> - PCF 的滤波核大小是**固定**的，而真实软阴影的半影区（Penumbra）大小应随 **遮挡物与接收面的距离** 以及 **光源物理尺寸** 动态变化。
> - PCF 只能做到“边缘模糊”，无法实现 **接触硬化（Contact Hardening）**（即靠近遮挡物处阴影硬，远离处阴影软）。

### 点光源

在了解了平行光的阴影映射后，理解点光源的阴影映射其实相当容易。主要只有两个不同之处：贴图类型和变换矩阵。

联系立方体贴图和反射映射的知识，我们可以得到类似的阴影映射处理：

从光源位置，向六个方向进行单独的深度渲染并绘制在同一张立方体贴图上。接下来我们正常进行纹理采样判断是否处于阴影中（此时采样使用光源到片段的方向向量而不是纹理坐标）。

```c++
// 独立渲染立方体贴图的每一个面
// 每次我们把立方体贴图纹理的一个面附加到帧缓冲对象上，渲染场景6次
for(int i = 0; i < 6; i++)
{
    GLuint face = GL_TEXTURE_CUBE_MAP_POSITIVE_X + i;
    glFramebufferTexture2D(GL_FRAMEBUFFER, GL_DEPTH_ATTACHMENT, face, depthCubemap, 0);
  	// lightViewMatrices[i]为每一个光源的vp变换矩阵
    BindViewMatrix(lightViewMatrices[i]);
    RenderScene();  
}
```

渲染场景6次会产生6次Draw Call，这可能会影响渲染的性能。我们可以用几何着色器的一个技巧来降低Draw Call的次数。

```c++
glBindFramebuffer(GL_FRAMEBUFFER, depthMapFBO);
// 此处使用glFramebufferTexture，将整个立方体贴图绑定至深度插件
glFramebufferTexture(GL_FRAMEBUFFER, GL_DEPTH_ATTACHMENT, depthCubemap, 0);
glDrawBuffer(GL_NONE);
glReadBuffer(GL_NONE);
glBindFramebuffer(GL_FRAMEBUFFER, 0);
```

当我们使用cubemap作为深度插件时，几何着色器中有一个内置变量**gl_Layer**，它可以指定输出的图元被传输至贴图的哪一个面。我们只需要在几何着色器中将每一个三角形进行变换，然后输入至贴图的对应面。

```c++
// 顶点着色器 输出世界坐标
gl_Position = model * vec4(position, 1.0);

// 几何着色器
#version 330 core
layout (triangles) in;
layout (triangle_strip, max_vertices=18) out;

uniform mat4 shadowMatrices[6];

void main()
{
    for(int face = 0; face < 6; ++face)
    {
        gl_Layer = face; // 用于指定贴图面的内置变量
        for(int i = 0; i < 3; ++i) 
        {
            gl_Position = shadowMatrices[face] * gl_in[i].gl_Position;
            EmitVertex();
        }    
        EndPrimitive();
    }
}

// 片段着色器
gl_FragDepth = gl_FragCoord.z; // 默认执行可以省略
```

需要注意的是，此时我们对六个面采用的都是**透视投影矩阵**。这意味着我们从纹理中获取的深度并非是线性的，我们需要先将其转换为线性后再进行阴影判断。

```c++
// 这是根据公式推导出的线性深度转换函数
// 其中的 near_plane 和 far_plane 需要作为uniform传入
float LinearizeDepth(float depth)
{
    float z = depth * 2.0 - 1.0; // 转换为 NDC
    return (2.0 * near_plane * far_plane) / (far_plane + near_plane - z * (far_plane - near_plane));
}
```

又或者，我们可以在渲染阴影贴图时，直接写入线性深度来省去这一换算过程：

```c++
// shadow.frag中
// 直接写入线性距离 需要在几何/顶点着色器中设置out变量fragPos
float dist = length(fragPos - lightPos);
gl_FragDepth = dist / far_plane; // 归一化到 [0,1]
```



#### 点光源的PCF

对于点光源，bias和正面剔除都能正常工作。但是我们使用的PCF需要进行一些修改。这主要是由于立方体贴图的采样方式不同。

我们只需要在引入一个z轴即可：

```c++
float shadow = 0.0;
float bias = 0.05; 
float samples = 4.0;
float offset = 0.1;
for(float x = -offset; x < offset; x += offset / (samples * 0.5))
{
    for(float y = -offset; y < offset; y += offset / (samples * 0.5))
    {
        for(float z = -offset; z < offset; z += offset / (samples * 0.5))
        {
            float closestDepth = texture(depthMap, fragToLight + vec3(x, y, z)).r; 
            closestDepth *= far_plane;   // Undo mapping [0;1]
            if(currentDepth - bias > closestDepth)
                shadow += 1.0;
        }
    }
}
shadow /= (samples * samples * samples);
```

但是这种算法的计算量太大，而且会有很多不必要的位置参与其中。（想象一个立方体被采样，一条向量穿过时会穿过多个子立方体）。

我们可以采用一个优化过的偏移方向数组来代替纯粹的遍历（它由人为决定且分布比较平均）：

```c++
vec3 sampleOffsetDirections[20] = vec3[]
(
   vec3( 1,  1,  1), vec3( 1, -1,  1), vec3(-1, -1,  1), vec3(-1,  1,  1), 
   vec3( 1,  1, -1), vec3( 1, -1, -1), vec3(-1, -1, -1), vec3(-1,  1, -1),
   vec3( 1,  1,  0), vec3( 1, -1,  0), vec3(-1, -1,  0), vec3(-1,  1,  0),
   vec3( 1,  0,  1), vec3(-1,  0,  1), vec3( 1,  0, -1), vec3(-1,  0, -1),
   vec3( 0,  1,  1), vec3( 0, -1,  1), vec3( 0, -1, -1), vec3( 0,  1, -1)
);
```
