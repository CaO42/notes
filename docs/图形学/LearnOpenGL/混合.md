 OpenGL 开启混合（`glEnable(GL_BLEND)`）后，一个片段在管线中的具体执行步骤：

### 1. 片段着色器输出 (Source)

片段着色器（Fragment Shader）执行完毕后，会输出一个或多个颜色值。这个输出的颜色在混合阶段被称为**源颜色 (Source Color)**，即公式中的 $C_{source}$。

### 2. 通过逐片段测试 (Per-Fragment Tests)

片段必须**通过所有的深度测试 (Depth Test)**、模板测试 (Stencil Test) 和裁剪测试 (Scissor Test)。如果片段在这些测试中被丢弃，混合就不会发生。

### 3. 读取颜色缓冲 (Destination)

当片段通过测试后，OpenGL 会去读取颜色缓冲区（Color Buffer）中**当前位置已有的颜色值**。这个颜色被称为**目标颜色 (Destination Color)**，即公式中的 $C_{destination}$。

### 4. 获取并计算混合因子 (glBlendFunc)

这就是你需要稍微修正的地方：混合因子**不一定非要使用 w 分量（通常在颜色上下文中称为 Alpha / A 分量）**。

`glBlendFunc(GLenum sfactor, GLenum dfactor)` 决定了如何计算 $F_{source}$ 和 $F_{destination}$。

- 虽然最常用的选项是 `GL_SRC_ALPHA` 和 `GL_ONE_MINUS_SRC_ALPHA`（提取源颜色的 Alpha 分量），但 OpenGL 也允许你使用源颜色的 RGB、目标颜色的 RGB、甚至常数颜色来作为因子（例如 `GL_SRC_COLOR`，`GL_ONE`，`GL_ZERO`）。
- 此外，你可以使用 `glBlendFuncSeparate` 为 RGB 通道和 Alpha 通道分别设置不同的混合因子。

### 5. 执行混合方程 (glBlendEquation)

你提供的公式是：

$$C_{result} = C_{source} * F_{source} + C_{destination} * F_{destination}$$

这是完全正确的**默认行为**。在 OpenGL 中，这被称为加法方程 (`GL_FUNC_ADD`)。但是，你可以通过 `glBlendEquation()` 改变这个“加号”：

- **`GL_FUNC_ADD`**: 默认，源与目标相加。
- **`GL_FUNC_SUBTRACT`**: 源减去目标 ($C_{source} * F_{source} - C_{destination} * F_{destination}$)。
- **`GL_FUNC_REVERSE_SUBTRACT`**: 目标减去源。
- **`GL_MIN` / `GL_MAX`**: 直接取源和目标中的最小值或最大值（此时因子 $F$ 会被忽略）。

### 6. 写入颜色缓冲 (更新)

计算出 $C_{result}$ 后，OpenGL 会将结果限制（Clamp）在 $[0, 1]$ 的范围内（针对标准的定点颜色缓冲），并最终将其写入颜色缓冲区，覆盖掉原来的 $C_{destination}$。
