### 二

#### 1

输出结果为：

```text
child: x = 15
parent: x = 2
```

`fork()` 后父子进程拥有各自独立的地址空间。子进程中 `x = 5 + 10 = 15`，父进程中 `x = 5 - 3 = 2`。由于父进程在输出前调用了 `wait(NULL)`，会等待子进程结束，所以子进程输出在前，父进程输出在后。

#### 2

共享内存的优点是速度快，建立共享区后进程可以直接读写内存，适合本机大量数据、高频率、低延迟通信。缺点是需要额外同步机制，否则容易出现竞态条件，编程也更复杂。

消息传递的优点是抽象简单、隔离性好，内核或通信系统可以帮助完成同步和保护，也适合分布式系统。缺点是通常需要内核介入，可能有上下文切换和数据复制开销，传输大量数据时效率较低。

如果是在同一台机器上进行大量数据交换，我倾向于共享内存；如果是跨主机通信、控制消息通信，或更强调隔离和接口清晰，我倾向于消息传递。

#### 3

僵尸进程是已经终止但父进程还没有调用 `wait()` 或 `waitpid()` 回收退出状态的子进程。它不再执行，但系统仍保留少量进程表信息。父进程调用 `wait()` 后可将其回收；如果父进程退出，通常由 `init` 或 `systemd` 收养并回收。

孤儿进程是父进程已经终止但子进程仍在运行的进程。操作系统会把孤儿进程交给 `init` 或 `systemd` 等进程收养，孤儿进程之后终止时由收养它的进程回收。

#### 4

循环执行 3 次，每次 `fork()` 都会让当前所有进程各自再创建一个子进程，所以进程数每次翻倍：

```text
1 -> 2 -> 4 -> 8
```

因此总共有 8 个进程，包括初始进程。循环结束后，每个进程都会执行一次 "LINE J" 处的 `printf("hello\n")`，所以该语句会执行 8 次，输出 8 行 `hello`，顺序不固定。

### 三

#### 1

##### a

循环缓冲区只使用 `in` 和 `out` 两个指针时，通常用：

```c
in == out
```

表示缓冲区为空。

如果允许把 `BUFFER_SIZE` 个位置全部写满，那么写满后 `in` 也会重新追上 `out`，仍然出现 `in == out`。这样无法区分缓冲区为空还是已满。

因此教材方案把判满条件设为：

```c
(in + 1) % BUFFER_SIZE == out
```

也就是故意空出一个位置，所以 `BUFFER_SIZE = 10` 时最多只能使用 9 个缓冲项。

##### b

可以增加一个计数变量 `count`，记录当前缓冲区中的元素个数：

```c
#define BUFFER_SIZE 10

item buffer[BUFFER_SIZE];
int in = 0;
int out = 0;
int count = 0;
```

判空条件：

```c
count == 0
```

判满条件：

```c
count == BUFFER_SIZE
```

生产者插入后执行：

```c
in = (in + 1) % BUFFER_SIZE;
count++;
```

消费者取出后执行：

```c
out = (out + 1) % BUFFER_SIZE;
count--;
```

这样可以通过 `count` 区分空和满，即使 `in == out` 也不会产生歧义，所有 `BUFFER_SIZE` 个缓冲项都可以被利用。并发实现时，需要用互斥锁或信号量保护 `in`、`out` 和 `count` 的修改。
