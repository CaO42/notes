# 操作系统原理  作业-7

第十章  Mass-Storage Structure（大容量存储结构）、第十一/十二章  File Systems（文件系统）与第十三章  I/O Systems（I/O 系统）

参考教材：Operating System Concepts, 9th Edition (Silberschatz et al.)

说明：以下题目依据磁盘、I/O 设备与文件系统核心知识点原创整理，适合作为课后作业。

## 一、选择题（每题3分，共30分）

1. 磁盘一次随机访问时间通常主要由哪三部分组成？（    ）

A. 编译时间、链接时间、装入时间

B. 寻道时间、旋转延迟和数据传输时间

C. CPU 调度时间、缺页处理时间和系统调用时间

D. 进程创建时间、上下文切换时间和终止时间

2. 在磁盘调度算法中，SCAN 算法的基本思想是（    ）。

A. 严格按照请求到达顺序服务

B. 每次选择距离当前磁头最近的请求

C. 磁头像电梯一样沿一个方向服务请求，到端点后再反向

D. 随机选择下一个磁道请求

3. 下列关于 SSTF（Shortest Seek Time First）磁盘调度的说法，正确的是（    ）。

A. 总是选择寻道距离最短的请求，但可能导致远处请求饥饿

B. 严格保证所有请求等待时间相同

C. 一定比所有其他算法公平

D. 只适用于固态硬盘，不适用于机械硬盘

4. RAID 1 的主要特点是（    ）。

A. 只进行条带化，不保存冗余信息

B. 镜像存储，同一数据写入两块或多块磁盘，提高可靠性

C. 仅使用一个校验盘保存所有文件名

D. 把内存页表复制到磁盘

5. 操作系统 I/O 子系统中，设备驱动程序的主要作用是（    ）。

A. 把设备无关的 I/O 请求转换成具体设备控制命令

B. 替代用户程序完成所有计算

C. 决定进程的逻辑地址空间大小

D. 只负责文件名解析，不接触硬件

6. 关于中断驱动 I/O，下列说法正确的是（    ）。

A. CPU 发起 I/O 后必须一直忙等直到设备完成

B. 设备完成或需要服务时通过中断通知 CPU，可减少忙等待

C. 它不需要任何硬件支持

D. 它只能用于只读文件

7. DMA（Direct Memory Access）的主要优点是（    ）。

A. 让设备控制器在较少 CPU 干预下直接在设备和内存之间传输数据

B. 让所有磁盘请求按 FCFS 排队

C. 强制所有文件连续分配

D. 完全取消中断机制

8. 在类 UNIX 文件系统中，目录项通常保存的是（    ）。

A. 文件名到 inode 或文件控制块编号的映射

B. CPU 寄存器到物理地址的映射

C. 所有文件内容的完整副本

D. 页面置换算法的运行结果

9. 下列文件分配方法中，最适合支持高效随机访问的是（    ）。

A. 连续分配

B. 单向链式分配且没有索引

C. 只按文件创建时间顺序顺序扫描

D. 每次访问都从目录根节点重新格式化磁盘

10. 位图（bitmap）用于空闲空间管理时，每一位通常表示（    ）。

A. 一个磁盘块是否空闲或已分配

B. 一个进程是否处于运行态

C. 一个页表项是否在 TLB 中命中

D. 一个 CPU 是否支持多级反馈队列

## 二、简答题（每题8分，共40分）

1. 请说明机械磁盘访问时间中寻道时间、旋转延迟和传输时间的含义。为什么磁盘调度主要关注减少寻道距离？

2. 请比较 FCFS、SSTF、SCAN 与 C-SCAN 四种磁盘调度算法的基本思想、优点和可能问题。

3. 什么是文件控制块（FCB）或 inode？它通常保存哪些元数据？目录项、路径名和文件控制块之间是什么关系？

4. 请比较连续分配、链式分配和索引分配三种文件分配方法在顺序访问、随机访问、外部碎片和可靠性方面的差异。

## 三、分析题（每题10分，共30分）

1. 某磁盘共有 0 到 199 号磁道，当前磁头位于 53 号磁道，请求队列按到达顺序为：

98, 183, 37, 122, 14, 124, 65, 67

请回答：

（a）分别使用 FCFS、SSTF、SCAN 和 C-SCAN 算法给出服务顺序，并计算总磁头移动距离。

（b）对 SCAN 和 C-SCAN，假设初始移动方向均为磁道号增大的方向，且移动到端点 199 后再按算法规则处理。

（c）结合计算结果，简要说明吞吐量与公平性之间的权衡。

2. 某文件系统磁盘块大小为 4KB，某 inode 采用如下索引结构：10 个直接块指针、1 个一级间接块指针、1 个二级间接块指针。每个块指针占 4B。

请回答：

（a）一个一级间接块最多可以索引多少个数据块？

（b）该 inode 通过直接块和一级间接块最多可表示多大的文件？

（c）如果文件大小为 50KB，需要多少个数据块？需要使用到哪几类指针？

（d）如果读取该 50KB 文件的最后 1KB，且 inode 已在内存中，但索引块和数据块均未缓存，最少需要几次磁盘块读取？

3. 某文件系统使用位图管理空闲磁盘块。磁盘共有 64 个块，位图中 0 表示空闲，1 表示已分配。当前位图按块号 0 到 63 顺序为：

11110000 11001100 00001111 11111100 00000000 10101010 11110000 00001111

请回答：

（a）请找出从低块号开始的前 10 个空闲块号。

（b）若采用连续分配，是否能为一个需要 6 个连续块的新文件找到空间？若能，请给出起始块号。

（c）若采用索引分配，该文件需要 6 个数据块和 1 个索引块，请给出一种可行的块分配方案。

（d）若系统突然断电，位图和目录/inode 信息可能不一致。请说明日志文件系统或 fsck 可以如何降低这种风险。
