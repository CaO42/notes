### 二

#### 1

寻道时间：磁头移动到目标磁道所需的时间。

旋转延迟：磁头到达目标磁道后，等待目标扇区旋转到磁头下方所需的时间。

传输时间：目标扇区到达磁头下方后，实际读写数据所需的时间。

磁盘调度主要关注减少寻道距离，是因为寻道时间涉及机械移动，通常开销较大。操作系统通过调整请求服务顺序，可以减少磁头在磁道之间来回移动的距离，从而提高吞吐量并降低平均响应时间。

#### 2

FCFS：按请求到达顺序服务。优点是简单、公平，不会产生饥饿；缺点是不考虑磁头位置，可能造成大量往返移动。

SSTF：每次选择距离当前磁头最近的请求。优点是通常能减少平均寻道距离、提高吞吐量；缺点是可能使远处请求长期等待，产生饥饿。

SCAN：磁头像电梯一样沿一个方向移动并服务沿途请求，到端点后再反向。优点是比 FCFS 减少无序往返，比 SSTF 更公平；缺点是不同位置请求的等待时间仍可能不均匀。

C-SCAN：只在一个方向服务请求，到达端点后回到另一端，再继续同方向扫描。优点是等待时间更均匀、公平性较好；缺点是回到起点会增加额外磁头移动。

#### 3

文件控制块 FCB 或 inode 是文件系统中描述文件的元数据结构。它通常保存文件类型、权限、所有者、所属组、文件大小、时间戳、链接计数、数据块指针或索引块指针等信息。

目录项保存文件名到 FCB/inode 的映射。路径名由一系列目录名和文件名组成，文件系统解析路径时，会逐级查找目录项，最终得到目标文件的 FCB/inode，再根据其中的信息访问文件数据块。

#### 4

连续分配：文件块连续存放。顺序访问和随机访问都很高效，但容易产生外部碎片，文件增长不方便。

链式分配：文件块通过指针链接。顺序访问较方便，不要求连续空间，因此没有外部碎片；但随机访问效率低，指针损坏可能导致后续块丢失。

索引分配：通过索引块保存文件各数据块地址。随机访问较高效，也不要求连续空间，没有外部碎片；但需要额外索引块空间，索引块损坏会影响文件数据定位。

##### a、b

FCFS：

```text
服务顺序：98, 183, 37, 122, 14, 124, 65, 67

```

SSTF：

```text
服务顺序：65, 67, 37, 14, 98, 122, 124, 183

```

SCAN，初始方向为磁道号增大方向，并移动到端点 199 后再反向：

```text
磁头路径：53 -> 65 -> 67 -> 98 -> 122 -> 124 -> 183 -> 199 -> 37 -> 14
服务顺序：65, 67, 98, 122, 124, 183, 37, 14
总移动距离：(199 - 53) + (199 - 14) = 331
```

C-SCAN，初始方向为磁道号增大方向，到 199 后回到 0 再继续同方向服务：

```text
磁头路径：53 -> 65 -> 67 -> 98 -> 122 -> 124 -> 183 -> 199 -> 0 -> 14 -> 37
服务顺序：65, 67, 98, 122, 124, 183, 14, 37
总移动距离：(199 - 53) + (199 - 0) + (37 - 0) = 382
```

##### c

SSTF 在本题中总移动距离最短，吞吐量较好，但可能使远处请求饥饿。FCFS 最公平，但磁头移动距离最大，吞吐量较差。SCAN 和 C-SCAN 在吞吐量与公平性之间折中，其中 C-SCAN 的等待时间更均匀，但可能因为回到起点而增加移动距离。

#### 2

##### a

一个一级间接块最多可索引：

```text
4KB / 4B = 1024
```

即 1024 个数据块。

##### b

直接块和一级间接块最多表示：

```text
(10 + 1024) * 4KB = 1034 * 4KB = 4136KB
```

即 4,235,264B。

##### c

50KB 文件需要：

```text
ceil(50KB / 4KB) = 13
```

即 13 个数据块。前 10 个使用直接块指针，剩余 3 个使用一级间接块指针，不需要二级间接块指针。

##### d

最后 1KB 位于第 13 个数据块中，该块需要通过一级间接块定位。inode 已在内存中，但索引块和数据块未缓存，因此最少需要读取：

```text
一级间接索引块 1 次 + 数据块 1 次 = 2 次磁盘块读取
```

##### d

日志文件系统可以把位图、目录和 inode 的修改作为事务先写入日志，崩溃后通过重放或撤销事务恢复一致状态。

fsck 可以在重启后扫描 inode、目录、数据块引用和位图，发现并修复位图与实际文件引用不一致、重复分配、孤立 inode、丢失块等问题，从而降低断电导致的不一致风险。
