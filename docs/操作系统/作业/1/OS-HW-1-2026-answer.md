## 二、简答题

### 1. 操作系统的三个主要目的是什么？

操作系统的主要目的可以概括为三点：

1. **方便用户使用计算机**：操作系统为用户和应用程序提供统一、易用的接口，例如命令行、图形界面、系统调用和文件系统。
2. **高效利用硬件资源**：操作系统负责调度 CPU、分配内存、管理 I/O 设备和存储设备，使硬件资源得到合理利用。
3. **为程序执行提供环境并保证系统可控**：操作系统为程序提供运行所需的抽象和服务，同时通过保护、同步、调度等机制控制程序执行，避免程序之间相互破坏。

也可以从教材角度表述为：操作系统是资源分配者、控制程序，并为用户程序提供执行环境。

### 2. 内核模式和用户模式如何作为保护系统的基本形式发挥作用？模式转换的触发条件是什么？

内核模式和用户模式通过 **模式位** 区分当前 CPU 正在执行的是操作系统内核代码还是普通用户程序代码。

- **内核模式**：操作系统内核运行在该模式下，可以执行全部机器指令，包括特权指令，例如 I/O 指令、修改页表、关闭中断、设置定时器等。
- **用户模式**：普通应用程序运行在该模式下，只能执行非特权指令，不能直接访问关键硬件资源，也不能直接修改操作系统核心数据结构。

这种区分形成了基本保护机制：用户程序如果试图直接执行特权指令，硬件会触发异常，由操作系统接管处理。这样可以防止用户程序破坏内核、其他进程或硬件设备状态。

常见的模式转换包括：

- **用户模式到内核模式**：由系统调用、陷阱、异常或外部中断触发。例如用户程序执行系统调用请求文件读写，CPU 会通过陷阱进入内核模式。
- **内核模式到用户模式**：内核完成服务后，通过从中断或陷阱返回的指令恢复用户程序上下文，并把模式位切回用户模式。

此外，定时器中断也是重要的保护机制。操作系统可以在用户程序运行前设置定时器，防止某个程序长期占用 CPU。

### 3. DMA 的工作原理及其相比中断驱动 I/O 的优势

DMA，即直接内存访问，是一种让设备控制器在较少 CPU 干预下直接在 I/O 设备和主存之间传输数据的机制。

基本过程如下：

1. CPU 向设备控制器发出 I/O 请求，并设置源地址、目标地址、传输长度、传输方向等参数。
2. 设备控制器获得总线控制权后，直接把数据块从设备传到内存，或从内存传到设备。
3. 数据传输期间，CPU 可以去执行其他任务。
4. 整个数据块传输完成后，设备控制器再通过一次中断通知 CPU。

相比普通中断驱动 I/O，DMA 的主要优势是：

- CPU 不需要为每个字节或每个字反复参与传输，开销更低。
- 中断次数大幅减少，通常一个数据块传输完成后才中断一次。
- 对磁盘、网卡、显卡等大块数据传输设备更高效。
- CPU 与 I/O 设备可以更好地并行工作，提高系统吞吐量。

## 三、分析题

### 1. 判断哪些指令是特权指令

| 指令 | 是否为特权指令 | 说明 |
| :--- | :--- | :--- |
| a. 设定定时器的值 | 是 | 定时器用于保证操作系统能够重新获得 CPU 控制权。如果用户程序可随意设置或关闭定时器，就可能长期霸占 CPU。 |
| b. 读取时钟 | 否 | 读取时钟通常不会改变系统状态，也不会破坏保护机制，因此一般可以允许用户程序通过普通指令或系统接口读取。 |
| c. 清除内存 | 是 | 如果这里指清除任意物理内存或系统内存，则必须是特权操作，否则用户程序可能破坏内核或其他进程的数据。若只是清除本进程地址空间内的普通变量，则不属于特权操作。 |
| d. 产生陷阱指令 | 否 | 陷阱指令是用户程序请求操作系统服务的受控入口，例如系统调用。它本身应允许用户执行，但陷入后只能跳转到预先规定的内核入口。 |
| e. 关闭中断 | 是 | 关闭中断会影响操作系统对设备和调度的控制，若用户程序可随意关闭中断，系统可能无法响应外部事件。 |
| f. 修改设备状态表中的条目 | 是 | 设备状态表属于操作系统维护的核心数据结构，直接修改会破坏 I/O 管理和设备一致性。 |
| g. 从用户模式切换到内核模式 | 是 | 用户程序不能直接修改模式位进入内核模式，只能通过陷阱、中断或异常等硬件控制路径进入。 |
| h. 访问 I/O 设备 | 是 | 直接访问 I/O 设备可能破坏设备状态或绕过文件权限、缓冲区和同步机制，因此通常必须由内核统一管理。 |

总结：特权指令的判断标准是该操作是否可能影响系统全局状态、硬件资源、内核数据结构或其他进程的安全。会造成这些影响的操作应限制在内核模式下执行。
