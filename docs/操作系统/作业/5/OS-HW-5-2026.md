# 操作系统原理 作业-5

## 第五章 Process Synchronization（进程同步）与第七章 Deadlocks（死锁）

*参考教材：Operating System Concepts, 9th Edition (Silberschatz et al.)*
*说明：以下题目依据进程同步、互斥与死锁核心知识点原创整理，适合作为课后作业。*

### 一、选择题（每题3分，共30分）

1\. 多个进程或线程并发访问共享数据，并且最终结果依赖于访问交错顺序，这种现象称为（ ）。

- A. 死锁（Deadlock）
- B. 竞争条件（Race Condition）
- C. 忙等待（Busy Waiting）
- D. 饥饿（Starvation）

2\. 临界区问题的正确解法通常必须满足三个要求。下列哪一项不属于这三个要求？（ ）。

- A. 互斥（Mutual Exclusion）
- B. 前进（Progress）
- C. 有界等待（Bounded Waiting）
- D. 最高 CPU 利用率（Maximum CPU Utilization）

3\. 关于 Peterson 算法，下列说法正确的是（ ）。

- A. 它是用于两个进程临界区互斥的软件算法
- B. 它可以直接推广到任意多个进程且无需修改
- C. 它不需要任何共享变量
- D. 它只能解决死锁检测问题

4\. 使用 test_and_set() 或 compare_and_swap() 等硬件原子指令实现自旋锁时，一个典型缺点是（ ）。

- A. 进程等待时仍持续占用 CPU 进行循环检查
- B. 无法保证任何形式的互斥
- C. 只能在单处理器系统中使用
- D. 一定会导致所有进程死锁

5\. 关于信号量 wait(S) 和 signal(S) 操作，下列说法正确的是（ ）。

- A. wait 和 signal 可以被普通赋值语句安全替代
- B. wait 和 signal 对信号量值的修改必须原子执行
- C. 计数信号量只能取 0 和 1 两个值
- D. 二值信号量不能用于互斥

6\. 读写器锁（Readers-Writers Lock）通常允许（ ）。

- A. 多个写者同时写共享数据
- B. 多个读者同时读共享数据，但写者写入时排斥其他读者和写者
- C. 读者和写者始终同时进入临界区
- D. 只允许一个读者读共享数据

7\. 下列哪一项不是死锁发生的四个必要条件之一？（ ）。

- A. 互斥（Mutual Exclusion）
- B. 占有并等待（Hold and Wait）
- C. 不可抢占（No Preemption）
- D. 最短作业优先（Shortest Job First）

8\. 在资源分配图中，如果每类资源只有一个实例，则图中存在环意味着（ ）。

- A. 系统一定处于死锁状态
- B. 系统一定处于安全状态
- C. 系统一定没有死锁
- D. 只能说明 CPU 调度算法错误

9\. 关于安全状态、安全序列和死锁的关系，下列说法正确的是（ ）。

- A. 安全状态一定不会导致死锁
- B. 不安全状态一定已经发生死锁
- C. 死锁状态一定是安全状态
- D. 安全序列只用于文件系统管理，与死锁无关

10\. 通过为所有资源类型规定全局申请顺序，并要求进程按该顺序申请资源，可以破坏死锁必要条件中的（ ）。

- A. 互斥条件
- B. 占有并等待条件
- C. 不可抢占条件

### 二、简答题（每题8分，共40分）

1. 什么是临界区（Critical Section）？请解释临界区问题的三个基本要求：互斥、前进和有界等待。

2. 为什么“关中断”不适合作为多处理器系统中的通用同步原语？它与硬件原子指令实现同步相比有什么局限？

3. 请比较二值信号量与计数信号量。wait(S) 与 signal(S) 操作如果没有原子性，可能造成什么问题？

4. 什么是读写器锁？如果一个多线程程序只使用读写器锁进行同步，当程序中存在多个读写器锁时，是否仍可能发生死锁？请说明理由。

5. 请解释安全状态（Safe State）、不安全状态（Unsafe State）和死锁状态（Deadlock State）之间的关系，并列出死锁发生的四个必要条件。

### 三、分析题（每题10分，共30分）

1. 考虑两个进程 P0 和 P1 使用如下 Peterson 算法进入临界区，其中 i 为当前进程编号，j = 1 - i：

   ```c
   do {
       flag[i] = true;
       turn = j;
       while (flag[j] && turn == j)
           ;    /* busy wait */

       /* critical section */

       flag[i] = false;

       /* remainder section */
   } while (true);
   ```

   请回答：
   - （a）变量 flag[] 和 turn 分别起什么作用？
   - （b）在顺序一致的内存模型下，该算法如何满足互斥、前进和有界等待？
   - （c）为什么在现代多处理器系统上仍需考虑内存重排序或内存屏障问题？

2. 一座单线桥连接南北两个方向。任意时刻桥上可以有多个同方向车辆，但不能同时存在南行和北行车辆。请使用信号量和互斥锁设计一个防止死锁的伪代码方案。无需解决饥饿问题，但应说明为什么不会出现相反方向车辆同时上桥。

3. 某系统有 5 个进程 P0-P4 和 3 类资源 A、B、C。当前 Allocation、Max 与 Available 如下：

   | 进程 | Allocation (A B C) | Max (A B C) |
   | :--- | :----------------- | :---------- |
   | P0   | 0 1 0              | 7 5 3       |
   | P1   | 2 0 0              | 3 2 2       |
   | P2   | 3 0 2              | 9 0 2       |
   | P3   | 2 1 1              | 2 2 2       |
   | P4   | 0 0 2              | 4 3 3       |

   Available = (3, 3, 2)。

   请回答：
   - （a）计算 Need 矩阵。
   - （b）判断当前系统是否处于安全状态；若安全，请给出一个安全序列。
   - （c）若 P1 请求 Request1 = (1, 0, 2)，系统能否立即分配？说明理由。
   - （d）若 P4 请求 Request4 = (3, 3, 0)，系统能否立即分配？说明理由。
