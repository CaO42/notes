# 概念速查表

## 操作系统基础

| 概念 | 速查说明 |
| :--- | :--- |
| 操作系统 | 系统软件；管理硬件资源、控制程序执行、提供用户和程序接口。 |
| 资源分配者 | 在多个进程/用户之间分配 CPU、内存、I/O、文件等资源。 |
| 控制程序 | 控制用户程序执行，防止错误和不当使用计算机。 |
| 分时系统 | 通过时间片让多个用户交互式共享计算机。 |
| SMP | 对称多处理，各处理器地位对等，通常共享内存。 |
| 中断向量 | 保存或指向中断服务例程入口地址。 |
| Bootstrap | 开机时运行的引导程序，通常存于 ROM/EEPROM。 |
| 存储层次 | 从寄存器到磁带，容量增大、速度降低、每位成本降低。 |
| Dual-mode | 用模式位区分用户模式和内核模式。 |
| 特权指令 | 只能内核态执行的敏感操作，如 I/O、关中断、改页表、设定时器。 |
| DMA | 设备控制器直接在设备和内存间传输数据，完成后中断 CPU。 |

## 操作系统结构

| 概念 | 速查说明 |
| :--- | :--- |
| Shell | 命令解释器，用户和 OS 的接口。 |
| 系统调用 | 用户程序请求内核服务的接口。 |
| API | 程序员调用的高级接口，由运行时库封装底层系统调用。 |
| 单体内核 | 多数服务在同一内核地址空间中，性能高但隔离弱。 |
| 微内核 | 内核只保留最小机制，服务尽量在用户态，隔离强但通信开销可能大。 |
| 分层结构 | 上层建立在下层服务上，结构清晰但划分困难。 |
| 可加载模块 | 动态扩展内核功能，兼顾效率和模块化。 |
| 虚拟机 | 多个 OS 实例共享并抽象同一底层硬件。 |
| eBPF | 在内核中安全运行的可验证字节码机制。 |
| XDP | 网络驱动层附近的 eBPF 钩子，位于协议栈之前。 |

## 进程

| 概念 | 速查说明 |
| :--- | :--- |
| 程序 | 存放在磁盘上的被动实体。 |
| 进程 | 正在执行的程序，是主动实体。 |
| PCB | 保存进程状态、PC、寄存器、调度信息、内存信息、打开文件等。 |
| 就绪态 | 已具备运行条件，等待 CPU。 |
| 等待态 | 等待某事件，如 I/O 完成。 |
| 长期调度器 | 控制哪些作业进入内存，控制多道程序度。 |
| 短期调度器 | 从就绪队列选择下一个运行进程。 |
| 僵尸进程 | 子进程已终止，父进程尚未 wait 回收退出状态。 |
| 孤儿进程 | 父进程已终止，子进程仍运行，由 init/systemd 收养。 |
| 共享内存 | 建立后通信快，但需要同步。 |
| 消息传递 | 抽象简单、隔离好、适合分布式，但可能有内核介入开销。 |

## CPU 调度

| 概念 | 速查说明 |
| :--- | :--- |
| Dispatcher | 执行上下文切换、切到用户态、跳转到目标进程。 |
| 抢占式调度 | OS 可强制收回 CPU，响应好但同步复杂。 |
| 非抢占式调度 | 进程主动放弃 CPU，简单但响应差。 |
| FCFS | 先到先服务，简单公平，可能有护航效应。 |
| SJF | 最短作业优先，平均等待时间好，但需预测 burst。 |
| SRTF | 抢占式 SJF，按最短剩余时间运行。 |
| RR | 时间片轮转，适合分时系统。 |
| 优先级反转 | 高优先级进程等待低优先级进程持有的资源。 |
| MLFQ | 多级反馈队列，进程可在队列间迁移。 |
| 硬实时 | 必须在截止期前完成，否则后果严重。 |
| 软实时 | 希望按时完成，偶尔错过可容忍。 |
| RMS | 周期越短，静态优先级越高。 |
| 处理器亲和性 | 尽量在同一 CPU 运行以复用缓存。 |

## 同步与死锁

| 概念 | 速查说明 |
| :--- | :--- |
| 竞争条件 | 结果依赖并发访问交错顺序。 |
| 临界区 | 访问共享资源的代码段。 |
| 互斥 | 同一时刻最多一个进程进入相关临界区。 |
| 前进 | 无人位于临界区且有人请求时，选择不能无限推迟。 |
| 有界等待 | 请求进入后，其他进程插队次数有上界。 |
| Peterson | 两进程软件互斥算法，依赖 `flag[]` 和 `turn`。 |
| 自旋锁 | 等待时忙循环，占用 CPU。 |
| 二值信号量 | 值为 0/1，常用于互斥。 |
| 计数信号量 | 值表示多个资源实例的可用数量。 |
| 读写器锁 | 多读可并发，写必须独占。 |
| 死锁四条件 | 互斥、占有并等待、不可抢占、循环等待。 |
| 安全状态 | 存在安全序列，一定可避免死锁。 |
| 不安全状态 | 无安全序列，可能导致死锁。 |
| 银行家算法 | 通过试探分配和安全性检查避免死锁。 |

## 内存管理

| 概念 | 速查说明 |
| :--- | :--- |
| 逻辑地址 | CPU 生成的地址。 |
| 物理地址 | 内存单元实际地址。 |
| MMU | 执行地址转换的硬件。 |
| base/limit | 逻辑地址合法条件为 `0 <= x < limit`，物理地址为 `base + x`。 |
| First Fit | 找第一个足够大的空闲区，速度快。 |
| Best Fit | 找最小的足够空闲区，易产生小碎片。 |
| Worst Fit | 找最大空闲区，可能快速消耗大块空间。 |
| 外部碎片 | 总空闲足够但不连续，无法满足连续分配。 |
| Compaction | 移动已分配块，把空闲空间合并。 |
| 页 | 逻辑地址空间的固定大小块。 |
| 页框 | 物理内存的固定大小块。 |
| 页表 | 页号到页框号的映射表。 |
| PTE | 页表项，含页框号、有效位、保护位、修改位等。 |
| TLB | 缓存最近页表项，加快地址转换。 |
| 分段 | 程序员可见的逻辑单位，段长可变。 |
| 请求分页 | 页面按需调入内存。 |
| COW | 父子先共享只读页，写入时才复制。 |
| 抖动 | 频繁缺页，系统大量时间用于换页。 |

## 文件、磁盘与 I/O

| 概念 | 速查说明 |
| :--- | :--- |
| 寻道时间 | 磁头移动到目标磁道的时间。 |
| 旋转延迟 | 等待目标扇区旋转到磁头下方的时间。 |
| 传输时间 | 实际读写数据的时间。 |
| SSTF | 选择最近请求，吞吐好但可能饥饿。 |
| SCAN | 电梯算法，向一侧扫描后反向。 |
| C-SCAN | 单方向扫描，到端点后回到另一端。 |
| RAID 1 | 镜像存储，提高可靠性。 |
| 设备驱动 | 把设备无关 I/O 转成具体设备命令。 |
| 中断驱动 I/O | 设备完成后中断通知 CPU，减少忙等。 |
| inode/FCB | 文件元数据结构，保存权限、大小、时间戳、块指针等。 |
| 目录项 | 文件名到 inode/FCB 的映射。 |
| 连续分配 | 随机访问快，但有外部碎片。 |
| 链式分配 | 无外部碎片，但随机访问差。 |
| 索引分配 | 随机访问较好，无外部碎片，但有索引开销。 |
| 位图 | 用每一位表示磁盘块空闲或已分配。 |
| 日志文件系统 | 用事务日志恢复元数据一致性。 |
| fsck | 重启后扫描和修复文件系统不一致。 |
