# 题型与计算模板

## 1. 特权指令判断模板

判断标准：该操作是否可能影响系统全局状态、硬件资源、内核数据结构或其他进程安全。

通常是特权指令：

- 设置定时器。
- 修改模式位或直接切换到内核态。
- 关闭中断。
- 访问 I/O 设备。
- 修改设备状态表。
- 修改页表或访问任意物理内存。

通常不是特权指令：

- 读取时钟。
- 发出 trap/system call 指令。
- 操作本进程普通变量。

答题句式：

```text
该操作会影响 ______，如果允许用户程序直接执行，可能导致 ______，
因此应限制在内核模式下执行，属于特权指令。
```

## 2. 系统调用路径模板

```text
应用程序
  -> 调用 API，如 read()/fork()/CreateFile()
  -> 运行时库准备参数和系统调用号
  -> 执行 trap/syscall 指令，从用户态进入内核态
  -> 内核 system-call interface 根据编号分派到具体服务例程
  -> 内核完成操作，返回结果或错误码
  -> 运行时库返回给应用程序
```

关键词：API 是程序员看到的高级接口；system-call interface 是内核实际入口；运行时库连接二者。

## 3. fork 程序分析模板

### 变量问题

`fork()` 后父子进程地址空间独立。fork 前的变量值被复制，之后父子各改各的。

示例判断：

```text
fork 前 x = 5。
子进程执行 x += 10，所以子进程 x = 15。
父进程执行 x -= 3，所以父进程 x = 2。
若父进程调用 wait(NULL)，则子进程输出先于父进程。
```

### 进程数问题

若没有条件限制，循环执行 n 次 `fork()`：

```text
总进程数 = 2^n
后续 printf 执行次数 = 2^n
```

如果 fork 在 if/else 或父子不同分支中，要画进程树。

## 4. 循环缓冲区模板

只使用 `in` 和 `out`：

```c
empty: in == out
full:  (in + 1) % BUFFER_SIZE == out
```

这样最多使用 `BUFFER_SIZE - 1` 个槽，因为必须空出一个位置区分空和满。

加入 `count` 后：

```c
empty: count == 0
full:  count == BUFFER_SIZE
```

生产者：

```c
buffer[in] = item;
in = (in + 1) % BUFFER_SIZE;
count++;
```

消费者：

```c
item = buffer[out];
out = (out + 1) % BUFFER_SIZE;
count--;
```

并发实现时，`in/out/count` 必须受互斥锁或信号量保护。

## 5. CPU 调度计算模板

### 通用公式

```text
完成时间 CT = 进程执行完的时刻
周转时间 TAT = CT - 到达时间 AT
等待时间 WT = TAT - CPU 突发时间 BT
平均等待时间 = 所有 WT 之和 / 进程数
平均周转时间 = 所有 TAT 之和 / 进程数
```

### 解题步骤

1. 按时间轴画甘特图。
2. 标出每段运行进程和起止时间。
3. 从甘特图读出每个进程完成时间。
4. 用公式计算周转时间和等待时间。
5. 求平均值。

### 算法规则

FCFS：

```text
按到达顺序执行，不抢占。
```

非抢占式 SJF：

```text
CPU 空闲时，从已到达进程中选择 CPU burst 最短者。
一旦开始运行，不因新进程到达而抢占。
```

SRTF：

```text
始终选择剩余 CPU 时间最短的进程。
新进程到达时，如果它的 burst 小于当前进程剩余时间，就抢占。
```

RR：

```text
每个进程最多运行一个时间片 q。
未完成则回到就绪队列尾部。
题目若规定“新到达进程先于被打断进程入队”，必须按题目规则处理。
```

优先级调度：

```text
先确认数值越小还是越大优先级越高。
抢占式版本中，新到达的更高优先级进程会抢占当前进程。
```

多级反馈队列：

```text
新进程进入最高优先级队列。
用完整个时间片仍未结束 -> 降级。
高优先级队列非空 -> 低优先级队列不能运行。
```

## 6. 银行家算法模板

### 矩阵关系

```text
Need = Max - Allocation
```

### 判断当前是否安全

1. 初始化 `Work = Available`，所有 `Finish[i] = false`。
2. 找一个未完成进程 `i`，满足 `Need[i] <= Work`。
3. 假设它完成，释放资源：

```text
Work = Work + Allocation[i]
Finish[i] = true
```

4. 重复直到所有进程完成。
5. 若能全部完成，则安全；完成顺序就是安全序列。

### 判断请求是否立即分配

对进程 Pi 的请求 `Request[i]`：

1. 检查 `Request[i] <= Need[i]`，否则请求超过最大声明。
2. 检查 `Request[i] <= Available`，否则资源暂时不足。
3. 试探分配：

```text
Available = Available - Request[i]
Allocation[i] = Allocation[i] + Request[i]
Need[i] = Need[i] - Request[i]
```

4. 运行安全性算法。
5. 试探后仍安全则可分配，否则必须等待。

## 7. 地址转换模板

页面大小为 `page_size`，逻辑地址为 `LA`：

```text
页号 p = LA / page_size
页内偏移 d = LA % page_size
```

查页表：

- 有效位为 0：缺页异常。
- 有效位为 1，页框号为 f：

```text
物理地址 PA = f * page_size + d
```

如果页面大小是 1KB，计算时用 1024B。

## 8. TLB 有效访问时间模板

设：

```text
TLB 查询时间 = epsilon
内存访问时间 = m
TLB 命中率 = alpha
```

忽略缺页开销时：

```text
命中时间 = epsilon + m
未命中时间 = epsilon + m + m
EAT = alpha * (epsilon + m) + (1 - alpha) * (epsilon + 2m)
```

例如 `epsilon = 10ns`，`m = 100ns`，`alpha = 0.8`：

```text
EAT = 0.8 * 110 + 0.2 * 210 = 130ns
```

## 9. 缺页异常处理模板

```text
CPU 访问页面
  -> MMU 发现有效位为 0，触发 page fault
  -> OS 检查地址是否合法
  -> 非法：终止进程
  -> 合法：寻找空闲页框
  -> 无空闲页框：运行页面置换算法
  -> 若牺牲页 dirty，先写回磁盘
  -> 从外存读入所需页面
  -> 更新页表/TLB
  -> 重新执行导致缺页的指令
```

## 10. 页面置换模拟模板

### FIFO

维护进入内存的先后队列，缺页且页框满时淘汰最早进入的页面。

### LRU

淘汰最长时间没有被访问的页面。模拟时记录每页最近一次访问时间。

### Optimal

淘汰未来最长时间不会再被访问的页面。若某页未来不再访问，优先淘汰。

答题时建议画表：

```text
访问页 | 页框1 | 页框2 | 页框3 | 是否缺页 | 淘汰页
```

## 11. 磁盘调度模板

总磁头移动距离：

```text
sum(abs(下一位置 - 当前位置))
```

FCFS：

```text
按请求到达顺序服务。
```

SSTF：

```text
每次选择距离当前磁头最近的请求。
```

SCAN：

```text
沿一个方向服务沿途请求。
若题目要求移动到端点，则即使端点没有请求也要计入移动距离。
到端点后反向。
```

C-SCAN：

```text
只在一个方向服务请求。
到端点后回到另一端，再继续同方向服务。
题目若规定到端点 199 后回到 0，则 199 -> 0 的距离要计入。
```

## 12. inode 容量计算模板

设：

```text
块大小 = B
指针大小 = P
直接指针个数 = D
```

一个一级间接块可保存：

```text
B / P 个块指针
```

直接 + 一级间接最大文件大小：

```text
(D + B/P) * B
```

若有二级间接：

```text
(D + B/P + (B/P)^2) * B
```

文件需要的数据块数：

```text
ceil(文件大小 / 块大小)
```

读取某数据块时，若 inode 已在内存：

- 直接块：至少读 1 次数据块。
- 一级间接块：至少读 1 次一级索引块 + 1 次数据块。
- 二级间接块：至少读 1 次二级索引块 + 1 次一级索引块 + 1 次数据块。

## 13. 位图分配模板

规则：题目中 0 表示空闲，1 表示已分配。

步骤：

1. 按块号从 0 开始逐位扫描。
2. 记录值为 0 的块号。
3. 连续分配要找连续 0。
4. 索引分配只需找足够数量的空闲块，其中 1 个作为索引块，其余作为数据块。

注意：位图字符串按块号顺序给出时，第一位就是当前分组的最低块号。
