# 按作业章节考点

## 作业 1：操作系统概述

### 范围

- 操作系统的定义、角色和目的。
- 分时系统、多道程序、多处理器系统、SMP。
- 存储管理、文件管理、处理器管理等基本职责。
- 中断、中断向量、陷阱、系统调用。
- 引导程序和系统启动。
- 存储层次结构。
- 双模式操作、模式位、特权指令。
- DMA 与中断驱动 I/O。

### 重点考察

- 能说明操作系统的三个主要目的：方便用户、有效利用硬件、为程序执行提供受控环境。
- 能解释用户模式与内核模式如何提供保护。
- 能判断某条指令是否应为特权指令：是否影响全局资源、硬件状态、内核数据结构或其他进程。
- 能描述 DMA 工作过程：CPU 初始化传输，设备控制器直接搬运数据，完成后中断通知 CPU。

### 易错点

- 产生 trap 指令本身通常不是特权指令，它是用户程序进入内核的受控入口。
- 读取时钟通常不是特权指令，但设置定时器一般是特权指令。
- 清除“任意物理内存”是特权操作，清除本进程普通变量不是。

## 作业 2：操作系统结构

### 范围

- 操作系统提供的服务：程序执行、I/O、文件系统、通信、错误检测、资源分配、记账、保护与安全。
- Shell/命令解释器的作用。
- 系统调用及其分类：进程控制、文件管理、设备管理、信息维护、通信、保护。
- API、运行时库、系统调用接口、内核之间的关系。
- 单体内核、微内核、分层系统、可加载内核模块。
- 虚拟机与系统启动。
- eBPF/XDP 网络过滤与观测实验。

### 重点考察

- 能画出或描述系统调用路径：应用程序调用 API，运行时库准备参数和系统调用号，trap 进入内核，内核返回结果。
- 能比较内核结构：
  - 单体内核：性能高，边界弱。
  - 微内核：内核小，隔离强，但 IPC 开销可能更大。
  - 分层结构：接口清晰，教学友好，但严格分层可能牺牲灵活性。
  - 可加载模块：效率和扩展性折中。
- 能从安全性、性能、可维护性分析某功能应放在内核态还是用户态。

### eBPF/XDP 实验考点

- eBPF 字节码在内核中运行，先由 verifier 检查安全性。
- XDP 处在网络协议栈之前，适合低延迟包过滤。
- map 用于内核态程序和用户态加载器共享统计数据。
- 常见实验流程：安装工具、编写 eBPF C 程序、clang 编译为 BPF object、加载到网卡接口、产生流量、用 bpftool/perf/loader 读取统计。

## 作业 3：进程

### 范围

- 进程与程序的区别。
- 进程内存布局：代码段、数据段、堆、栈。
- 进程状态转换：new、ready、running、waiting、terminated。
- PCB 内容：状态、程序计数器、寄存器、调度信息、内存管理信息、打开文件等。
- 长期/短期/中期调度器。
- `fork()`、`wait()`、僵尸进程、孤儿进程。
- IPC：共享内存与消息传递。
- 生产者-消费者循环缓冲区。

### 重点考察

- `fork()` 后父子进程拥有独立地址空间，变量修改互不影响。
- 子进程中 `fork()` 返回 0，父进程中返回子进程 PID，失败返回 -1。
- 等待态不能直接到运行态，事件发生后通常先回到就绪态。
- 阻塞 send + 阻塞 receive 称为 rendezvous。
- 消息传递适合分布式系统，共享内存适合同机大量数据交换。

### 程序分析重点

- 有 `wait()` 时，父进程通常等待子进程结束后再输出。
- 三次无条件 `fork()` 后总进程数为 `2^3 = 8`，每个进程都会执行后续语句。
- 循环缓冲区只用 `in` 和 `out` 时，`in == out` 既可能表示空，也可能表示满，所以通常牺牲一个槽位；若加入 `count`，就能使用全部槽位。

## 作业 4：CPU 调度

### 范围

- 调度发生时机。
- Dispatcher/分派程序。
- 抢占式与非抢占式调度。
- FCFS、SJF、SRTF、RR、优先级调度、多级反馈队列。
- 时间片大小影响。
- 优先级反转。
- 硬实时/软实时、RMS。
- 多处理器调度与处理器亲和性。

### 重点考察

- 会画甘特图。
- 会算完成时间、周转时间、等待时间。
- 会比较算法目标：
  - FCFS 简单公平，但可能有护航效应。
  - SJF/SRTF 平均等待时间好，但需要预测 CPU burst。
  - RR 响应性和公平性好，但受时间片影响。
  - 优先级调度照顾高优先级任务，但可能饥饿。
  - MLFQ 动态区分交互型和 CPU 密集型任务。

### 易错点

- RR 时间片趋近无穷大时退化为 FCFS，不是 SJF。
- SRTF 是抢占式 SJF，要根据剩余时间判断是否抢占。
- 优先级数值越小越高是题目给定规则，不要反过来。
- 多级反馈队列中“用完整个时间片未结束”才降级。

## 作业 5：同步与死锁

### 范围

- 竞争条件。
- 临界区问题：互斥、前进、有界等待。
- Peterson 算法。
- 关中断、硬件原子指令、自旋锁。
- 信号量：二值、计数、wait/signal 原子性。
- 读写器锁。
- 死锁四个必要条件。
- 资源分配图、安全状态、银行家算法。
- 单线桥同步问题。

### 重点考察

- 能解释为什么 `wait()` 和 `signal()` 必须原子执行。
- 能说明关中断不适合作为多处理器通用同步原语。
- 能分析 Peterson 算法中 `flag[]` 和 `turn` 如何保证互斥、前进和有界等待。
- 能列出死锁四条件：互斥、占有并等待、不可抢占、循环等待。
- 能用银行家算法判断请求是否可立即分配。

### 易错点

- 安全状态一定不会死锁；不安全状态不等于已经死锁；死锁状态一定不安全。
- 每类资源只有一个实例时，资源分配图有环意味着死锁；多实例时有环不一定死锁。
- 读写器锁也可能死锁，尤其是多个锁顺序不一致或读锁升级写锁。

## 作业 6：主存与虚拟内存

### 范围

- 逻辑地址、物理地址、MMU。
- base/limit 内存保护。
- 连续内存分配：First Fit、Best Fit、Worst Fit。
- 外部碎片与 compaction。
- 分页、页框、页表、PTE、TLB。
- 分段。
- 请求分页、缺页异常、写时复制。
- 页面置换：FIFO、LRU、Optimal。
- Belady 异常与抖动。

### 重点考察

- 地址转换：
  - 页号 = 逻辑地址 / 页面大小。
  - 页内偏移 = 逻辑地址 % 页面大小。
  - 物理地址 = 页框号 * 页面大小 + 偏移。
- TLB 有效访问时间：
  - 命中：TLB 查询 + 一次内存访问。
  - 未命中：TLB 查询 + 页表访问 + 数据访问。
- 缺页处理流程：判断合法性、找空闲页框或置换、必要时写回、读入页面、更新页表、重新执行指令。
- 页面置换模拟要逐步记录页框内容和缺页次数。

### 易错点

- 分页消除外部碎片，但仍可能有内部碎片。
- 分段是程序员可见的逻辑单位，段长可不同。
- FIFO 可能出现 Belady 异常，LRU 和 Optimal 属于栈性质算法，通常不会。
- COW 不是 fork 后立即复制全部页面，而是写入时才复制。

## 作业 7：存储、文件系统与 I/O

### 范围

- 磁盘随机访问时间：寻道、旋转延迟、传输。
- 磁盘调度：FCFS、SSTF、SCAN、C-SCAN。
- RAID 1。
- 设备驱动、中断驱动 I/O、DMA。
- FCB/inode、目录项、路径名解析。
- 文件分配：连续、链式、索引。
- 空闲空间位图。
- 日志文件系统与 fsck。

### 重点考察

- 磁盘调度要给出服务顺序和总磁头移动距离。
- SCAN 和 C-SCAN 要特别注意题目是否要求移动到端点。
- inode 计算要先算一个索引块能保存多少指针。
- 位图题要按块号顺序逐位扫描，0 表示空闲，1 表示已分配。
- 日志文件系统通过事务和重放/撤销降低崩溃不一致风险；fsck 通过重启后扫描修复位图、inode、目录不一致。

### 易错点

- SSTF 总移动距离通常较小，但可能导致远处请求饥饿。
- 连续分配随机访问快，但有外部碎片且文件增长困难。
- 链式分配无外部碎片，但随机访问差，指针损坏影响后续块。
- 索引分配随机访问较好且无外部碎片，但需要额外索引块。
