## 二、整体知识主线

这七次作业基本沿着操作系统课程的主线展开：

1. 操作系统为什么存在：资源管理、程序控制、用户便利性、保护机制。
2. 操作系统如何组织：系统调用、内核结构、启动流程、虚拟化与实验观测工具。
3. 程序如何变成并发执行实体：进程、fork、PCB、进程状态、IPC。
4. CPU 如何分配给进程：调度时机、调度算法、响应时间、公平性和实时性。
5. 并发访问如何保持正确：临界区、锁、信号量、读写器锁、死锁和避免策略。
6. 内存如何抽象与扩展：地址转换、分页、TLB、缺页异常、页面置换。
7. 外存和文件如何管理：磁盘调度、文件元数据、文件块分配、空闲空间、一致性恢复。

## 三、考察内容分类

### 1. 概念识别类

常见形式是选择题或短答，要求准确区分术语：

- 操作系统是系统软件，既管理资源也控制程序执行。
- 中断向量保存中断服务例程入口地址。
- Bootstrap 程序通常在 ROM/EEPROM 中。
- SMP 中处理器地位对等并共享资源。
- 进程是主动实体，程序是被动实体。
- 僵尸进程与孤儿进程的产生条件不同。
- 死锁状态一定不安全，不安全状态不一定死锁。
- 目录项通常保存文件名到 inode/FCB 的映射。

### 2. 比较分析类

这类题要求不仅知道定义，还要说出优缺点和适用场景：

- 内核模式 vs 用户模式。
- 单体内核 vs 微内核 vs 分层结构 vs 可加载模块。
- 共享内存 vs 消息传递。
- 抢占式调度 vs 非抢占式调度。
- FCFS、SJF、RR、优先级、多级反馈队列。
- 二值信号量 vs 计数信号量。
- 安全状态、不安全状态、死锁状态。
- 分页 vs 分段。
- FIFO、LRU、Optimal。
- 连续分配、链式分配、索引分配。
- FCFS、SSTF、SCAN、C-SCAN 磁盘调度。

### 3. 手算模拟类

这是复习时最应该优先练习的部分：

- CPU 调度甘特图、等待时间、周转时间。
- 多级反馈队列的队列迁移和完成时间。
- 银行家算法 Need 矩阵、安全序列、请求试探分配。
- 分页地址转换、缺页判断、TLB 有效访问时间。
- FIFO/LRU/Optimal 页面置换缺页次数。
- 磁盘调度服务顺序和总磁头移动距离。
- inode 直接/间接索引容量计算。
- 位图空闲块扫描和分配方案。

### 4. 程序/伪代码分析类

重点是能从代码推导并发行为：

- `fork()` 后父子进程地址空间独立。
- `wait()` 会影响父子输出顺序。
- 连续执行 n 次无条件 `fork()`，总进程数通常为 `2^n`。
- 循环缓冲区中 `in == out` 的空/满歧义。
- Peterson 算法中 `flag[]` 和 `turn` 的作用。
- 单线桥问题可用“同方向计数 + 桥互斥信号量”建模。

### 5. 实验与工具类

作业 2 的 eBPF/XDP 是独立实验型内容，复习时掌握框架即可：

- eBPF 程序运行在内核中，先经过 verifier 验证。
- XDP 位于网络驱动收包后、协议栈前，适合高性能包处理。
- 常见动作包括 `XDP_PASS`、`XDP_DROP`、`XDP_TX`。
- eBPF maps 用于内核态和用户态共享统计数据或配置。
- 工具链包括 clang/LLVM、libbpf、bpftool、perf、tcpdump。

## 四、复习优先级

### A 级：必须熟练

- CPU 调度计算：FCFS、SJF/SRTF、RR、优先级、多级反馈队列。
- 同步与死锁：临界区三条件、信号量、Peterson、银行家算法。
- 内存计算：地址转换、TLB EAT、页面置换。
- 文件/磁盘计算：磁盘调度、inode 容量、位图分配。

### B 级：必须理解

- 系统调用路径：应用程序 -> API/运行时库 -> trap -> 内核服务。
- 进程状态、PCB、fork、僵尸/孤儿进程、IPC 模型。
- 内核结构：单体、微内核、分层、模块、虚拟机。
- 请求分页、缺页异常处理流程、写时复制。
- 文件系统元数据、目录项、日志文件系统与 fsck。

### C 级：快速背诵

- OS 目标、存储层次、bootstrap、中断向量。
- 设备驱动、中断驱动 I/O、DMA。
- RAID 1、处理器亲和性、实时系统、RMS。
- eBPF/XDP 基本概念和实验步骤。
