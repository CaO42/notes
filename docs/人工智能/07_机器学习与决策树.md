### 机器学习原理：确定"评价函数好坏"的标准

**监督式学习（Supervised Learning）**

输入 $x_i$ → $f_1$ → $f(x_i)$，与真实标签 $y_i$ 比较：

| 输入 $x_i$ | 预测 $f(x_i)$ | 真实 $y_i$ | 误差 |
|-------------|----------------|-------------|------|
| 生物1       | 101 kg         | 103 kg      | 2    |
| 生物2       | 88 kg          | 87 kg       | 1    |
| 生物3       | 60 kg          | 72 kg       | 12   |

$f_1$ 的输出距离正确答案总和为 15 → $L(f_1) = 15$

$$L(f_1) := \sum_{i}^{N} \|f_1(x_i) - y_i\|$$

误差函数（Loss function）

**半监督式学习（Semi-Supervised Learning）**

$$L(f_1) = \text{输出距离正确答案} + \text{相似的样本间距离}$$

$$L(f_1) := \sum_{i}^{N} \|f_1(x_i) - y_i\| + \sum_{k \sim j} \|f_1(x_k) - f_1(x_j)\|$$

其中"相似的生物体重也相似"。

### 机器学习原理：Training 与 Testing

**训练阶段（Training）**：

$$f^* = \arg\min_{f \in \mathcal{H}} L(f)$$

$L(f)$ 小 → Training

**测试阶段（Testing）**：测试不一定好

$$L(f_1) = \text{输出距离正确答案} + \text{相似的样本间距离} + \text{函数泛化性}$$

$$L(f_1) := \sum_{i}^{N} \|f_1(x_i) - y_i\| + \sum_{k \sim j} \|f_1(x_k) - f_1(x_j)\| + Regular(f_1)$$

## 决策树

### CLS算法

**步骤：**

1. 生成一颗空决策树和一个训练样本属性表；
2. 若训练样本集 $T$ 中所有的样本都属于同一类，则生成结点 $T$，并终止学习算法；否则
3. 根据**某种策略**从训练样本属性表中选择属性 $A$ 作为测试属性，生成测试结点 $A$；
4. 若 $A$ 的取值为 $v_1, v_2, \ldots, v_m$，则根据 $A$ 的取值的不同，将 $T$ 划分成 $m$ 个子集 $T_1, T_2, \ldots, T_m$；
5. 从训练样本属性表中删除属性 $A$；
6. 对每个子集递归调用CLS

> **CLS算法有什么问题？**

在步骤3中，**根据某种策略从训练样本属性表中选择属性A作为测试属性。没有规定采用何种测试属性。实践表明，测试属性集的组成以及测试属性的先后对决策树的学习具有举足轻重的影响。**

### ID3算法

- ID3算法是一种经典的决策树学习算法，由Quinlan于1979年提出。
- ID3算法主要针对属性选择问题。是决策树学习方法中最具影响和最为典型的算法。
- 该方法使用**信息增益度**选择**测试属性**。
- 当获取信息时，将不确定的内容转为确定的内容，因此信息伴着不确定性。
- 从直觉上讲，小概率事件比大概率事件包含的信息量大。如果某件事情是"百年一见"则肯定比"习以为常"的事件包含的信息量大。

> **如何度量信息量的大小？也就是测试属性对应的信息量大小？**

### 信息论：Entropy（熵）

- **Entropy（熵）**：represent the expectation of uncertainty for a random variable（可以用来衡量离散变量的不确定性，如抛硬币、掷骰子）

$$H(X) = -\sum_{x \in X} p(x) \log_2 p(x) = \sum_{x \in X} p(x) \log_2 \frac{1}{p(x)} = E\left(\log_2 \frac{1}{p(X)}\right)$$

### 信息论：二值变量的熵

- $P(X=1) = p,\; P(X=0) = 1-p$
- 假设抛一枚硬币，正面朝上的概率为 $p$，反面朝上的概率为 $1-p$，则抛这枚硬币所得结果的不确定性（熵值）是 $p$ 的下述函数：

$$H(X) = -p \log_2 p - (1-p) \log_2(1-p)$$

[截图待插入: H(X)关于p的函数曲线图，呈倒U形，在p=0.5时达到最大值1]

### 信息论：条件熵与联合熵

**条件熵：**

$$H(Y|X) = \sum_{x \in X} p(x) H(Y|X=x) = \sum_{x \in X} p(x)\left[-\sum_{y \in Y} p(y|x) \log_2 p(y|x)\right] = -\sum_{x \in X}\sum_{y \in Y} p(x)p(y|x) \log_2 p(y|x) = -\sum_{x \in X}\sum_{y \in Y} p(x,y) \log_2 p(y|x)$$

- **条件熵**：在已知随机变量 $X$ 的结果后，随机变量 $Y$ 仍然保留的不确定性
- **联合熵**：两个随机变量 $X$ 和 $Y$ 作为一个联合系统时的总不确定性

**联合熵：**

$$H(X,Y) = -\sum_{x \in X}\sum_{y \in Y} p(x,y) \log_2 p(x,y)$$

### 信息论：联合的分解

$$H(X,Y) = -E_{p(x,y)} \log_2 p(x,y) = -E_{p(x,y)}(\log_2(p(x)p(y|x))) = -E_{p(x,y)}(\log_2 p(x) + \log_2 p(y|x)) = -E_{p(x)} \log_2 p(x) - E_{p(x,y)} \log_2 p(y|x) = H(X) + H(Y|X)$$

两个离散变量 $X$ 和 $Y$ 的联合熵（即，联合出现的不确定性）
= $X$ 的熵 + 给定 $X$，出现 $Y$ 的条件熵
= $X$ 的不确定性 + 给定 $X$，出现 $Y$ 的不确定性

### 信息论：互信息（Mutual Information）

因为：$H(X,Y) = H(X) + H(Y|X) = H(Y) + H(X|Y)$

所以：$H(Y) - H(Y|X) = H(X) - H(X|Y) = I(X;Y)$

> **两个离散变量X和Y的互信息I(X;Y)衡量的是这两个变量之间的相关度**

- 一个连续变量 $X$ 的不确定性，用方差 $Var(X)$ 来度量
- 一个离散变量 $X$ 的不确定性，用熵 $H(X)$ 来度量
- 两个连续变量 $X$ 和 $Y$ 的相关度，用协方差或相关系数来度量
- 两个离散变量 $X$ 和 $Y$ 的相关度，用互信息 $I(X;Y)$ 来度量

**互信息的含义：**

"因X而有Y事件"的熵（基于已知随机变量的不确定性）在"Y事件"的之中具有多少影响地位（"Y事件所具有的不确定性"其中包含了多少"Y|X事件所具有的不确定性"），意即"Y具有的不确定性"有多少程度是起因于X事件。

举例来说，当 $I(X;Y)=0$ 时，也就是 $H(Y)=H(Y|X)$ 时，即代表此时"Y的不确定性"即为"Y|X的不确定性"，这说明了互信息的具体意义是在度量两个事件彼此之间的关联性。所以具体的解释就是：互信息越小，两个来自不同事件空间的随机变量彼此之间的关联性越低；相互信息越高，关联性则越高。

### 基于Gini指数的CART模型

如果一个数据集 $D$ 包含来自 $n$ 个类的样本，那么基尼指数 $gini(D)$ 定义如下：

$$gini(D) = \sum_{j=1}^{n} p_j(1-p_j) = 1 - \sum_{j=1}^{n} p_j^2$$

$p_j$ 是类 $j$ 在 $D$ 中的相对频率。

如果 $n=2$，那么 $gini(D) = 2p(1-p)$

### 基于Gini指数的CART模型：Gini_split

如果一个数据集 $D$ 被分成两个子集 $D_1$ 和 $D_2$，大小分别为 $N_1$ 和 $N_2$，数据包含来自 $n$ 个类的样本，则基尼指数 $gini_{split}(D)$ 定义如下：

$$gini_{split}(D) = \frac{N_1}{N} gini(D_1) + \frac{N_2}{N} gini(D_2)$$

- **具有最小 $gini_{split}(D)$ 的属性被选为分裂节点的属性**（对每个属性，需要遍历所有可能的分裂位置点）。

### 基于Gini指数的CART模型：计算示例

在"是否买电脑"中，$D$ 有9个样本"是"，5个样本"否"：

$$gini(D) = 1 - \left(\frac{9}{14}\right)^2 - \left(\frac{5}{14}\right)^2 = 0.459$$

属性"收入"将 $D$ 分成：10个在 $D_1$：{medium, high} 以及4个在 $D_2$：{low}：

$$gini_{\{\text{medium,high/low}\}}(D) = \frac{10}{14} gini(D_1) + \frac{4}{14} gini(D_2) = \frac{10}{14}\left(1-\left(\frac{6}{10}\right)^2-\left(\frac{4}{10}\right)^2\right) + \frac{4}{14}\left(1-\left(\frac{1}{4}\right)^2-\left(\frac{3}{4}\right)^2\right) = 0.450$$

### 基于Gini指数的CART模型：三分类计算

算下三个属性收入划分为4个在 $D_1$：{high}，6个在 $D_2$：{medium}，以及4个在 $D_3$：{low} 的基尼指数。

**(1) income = high**

- 对应行：1, 2, 3, 13
- 类别分布：
  - no：行 1, 2 → 2 条
  - yes：行 3, 13 → 2 条
- 总数：4
- 基尼：

$$Gini_{\text{high}} = 1 - \left(\frac{2}{4}\right)^2 - \left(\frac{2}{4}\right)^2 = 1 - 0.25 - 0.25 = 0.5$$

$$Gini_{\text{medium}} = 1 - \left(\frac{2}{6}\right)^2 - \left(\frac{4}{6}\right)^2 = 1 - \frac{4}{36} - \frac{16}{36} = 1 - \frac{20}{36} = \frac{16}{36} \approx 0.444$$

$$Gini_{\text{low}} = 1 - \left(\frac{1}{4}\right)^2 - \left(\frac{3}{4}\right)^2 = 1 - \frac{1}{16} - \frac{9}{16} = \frac{6}{16} = 0.375$$

$$Gini_{\text{split}}(income) = \frac{4}{14} \times 0.5 + \frac{6}{14} \times 0.444 + \frac{4}{14} \times 0.375 = \frac{2}{14} + \frac{2.664}{14} + \frac{1.5}{14} = \frac{6.164}{14} \approx 0.4403$$

### 基于Gini指数的CART模型：CART分类树生成算法

**CART（分类树）的生成算法**

- **输入**：训练数据集 $D$，停止计算条件
- **输出**：CART分类树
- 从根节点开始，递归对每个结点操作
  1. 设结点数据集为 $D$，对每个特征 $A$，对其每个值 $a$，根据样本点对 $A=a$ 的测试为是或否，将 $D$ 分为 $D_1$，$D_2$，计算 $A=a$ 的基尼指数
  2. 在所有的特征 $A$ 以及所有可能的切分点 $a$ 中，**选择基尼指数最小的特征和切分点**，将数据集分配到两个子结点中
  3. 对两个子结点递归调用1和2步骤
  4. 生成CART树

### 机器学习原理: 找到最好的函数

## 无监督 (聚类) 算法

### 什么是聚类分析?

Finding groups of objects such that the objects in a group will be similar (or related) to one another and different from (or unrelated to) the objects in other groups.

寻找一组对象，使得同一组内的对象彼此相似（或相关），而与不同组内的对象不同（或不相关）。

- **Intra-cluster distances are minimized** (簇内距离最小化)
- **Inter-cluster distances are maximized** (簇间距离最大化)

层次聚类从将每个观测值视为一个独立簇开始。

随后，它循环执行以下两个步骤:

1. 识别距离最近的两个簇
2. 合并最相似的两个簇

此迭代过程持续进行，直到所有簇合并为一个整体。

### K-means聚类

- **Partitional clustering approach 分割式聚类方法**
  - Each cluster is associated with a **centroid** (center point) 每个簇有一个中心点
  - Each point is assigned to the cluster with the **closest centroid** 将各数据点划分到最近中心点对应的簇
- Number of clusters, K, must be specified
- The basic algorithm is very simple

中文说明：
- 必须预先指定簇的数量 $K$。
- 基本算法非常简单。

**算法 1 基础 K-means 算法。**

1. 选择 K 个点作为初始中心。
2. **重复**
3. 通过将所有点分配给最近的中心来形成 K 个簇。
4. 重新计算每个簇的中心。
5. **直到** 中心不再发生变化

### 标准K-means流程

[截图待插入: K-means迭代过程图，Iteration 1到Iteration 6，展示数据点从初始随机分配到逐步收敛到三个稳定簇的过程，颜色分别为红、绿、蓝]

### K-mean聚类细节

- 初始中心点通常**随机选择**。
  - 不同次运行得到的聚类结果可能不同。
- 中心点通常是簇内各点的**均值**。
- "相近度"可通过欧氏距离、余弦相似度、相关系数等度量。
- 使用上述常见相似性度量时，**K-means算法能够保证收敛**。
- 大部分收敛发生在最初几次迭代中。Often the stopping condition is changed to 'Until relatively few points change clusters'
- Complexity is $O(n \times K \times I \times d)$
  - $n$ = number of points, $K$ = number of clusters, $I$ = number of iterations, $d$ = number of attributes

中文说明：
- 大部分收敛发生在最初几次迭代中，因此停止条件常改为："直到只有相对较少的点改变所属簇"。
- 复杂度为 $O(n \times K \times I \times d)$。
- $n$ 表示点的数量，$K$ 表示簇数，$I$ 表示迭代次数，$d$ 表示属性/维度数量。

### K-means簇的评估

- 最常用的度量是误差平方和 **Sum of Squared Error (SSE)**
  - 计算每个点到其最近聚类中心的距离作为误差
  - 将所有这些误差平方后求和，即得到SSE

$$SSE = \sum_{i=1}^{K} \sum_{x \in C_i} dist^2(m_i, x)$$

- $x$表示簇$C_i$中的数据，$m_i$代表簇$C_i$中心点
- 给定两个聚类方案时，我们选择误差更小的那个
- 降低SSE最直接的方法是增加聚类数量K
  - 一个K值较小的优质聚类，其SSE可能低于K值更大的劣质聚类

### K的选择

- **Elbow method肘部法则**: 绘制不同K值对应的SSE折线图，若图形呈手臂状，则"肘关节"处对应的K值即为最优选择。
- 手臂曲线的肘关节

[截图待插入: Elbow method折线图，横轴为Number of clusters k(1-10)，纵轴为Total Within Sum of Square，曲线在k=3处出现肘部拐点，标注Optimal number of clusters]

### 选择初始点的问题

- 若存在K个真实簇群，则从每个簇中各选出一个中心点的概率很小。
  - 当K值较大时，这种概率会显著降低
  - 假设各簇规模相同（均为$n$个数据点），则:

$$P = \frac{\text{number of ways to select one centroid from each cluster}}{\text{number of ways to select } K \text{ centroids}} = \frac{K! n^K}{(Kn)^K} = \frac{K!}{K^K}$$

- For example, if $K = 10$, then probability = $10!/10^{10} = 0.00036$
- 初始中心点有时会自我调整到正确位置，有时则不能
- 以10个簇群的情况为例

### 如何选择初始质心?

- **多次运行**
  - 虽有效果，但成功概率依然有限
- **二分K均值法**
  - 采用分裂式聚类策略: 每次选择一个簇进行二分，最终形成K个簇
- **采样结合层次聚类**
  - 先对数据采样，再用层次聚类确定初始中心点
- **优选中心点**
  - 选取多于K个初始中心点，从中筛选分布最分散的K个点

### Bisecting(二分) K-means

- **Bisecting K-means algorithm**
  - Variant of K-means that can produce a partitional or a hierarchical clustering

中文说明：
- **二分 K-means 算法**
- 它是 K-means 的一种变体，可以产生划分式聚类结果，也可以形成层次式聚类结构。

**算法 3 二分 K-means 算法。**

1. 初始化簇列表，使其包含涵盖所有点的簇。
2. **重复**
3. 从簇列表中选择一个簇
4. **对于** $i = 1$ 到 $number\_of\_iterations$ **执行**
5. 使用基础 K-means 对所选簇进行二分
6. **结束 for**
7. 将具有最低 SSE 的二分结果中的两个簇添加到簇列表中。
8. **直到** 簇列表包含 $K$ 个簇

### K-means++ [Arthur et al. '07]

**算法执行步骤：**（核心思想：Spreads out the centers 分散选择中心点）

1. 从数据集中随机均匀选取第一个中心点 $x_1$
2. 当 $2 \le i \le k$ 时重复以下过程：
   - 按照特定概率分布采样得到新中心点 $x'$
   - 该分布与 $D(x)^2$ 成正比，其中 $D(x)$ 表示数据点 $x$ 到已选中心点的最小距离

$$\frac{D(x')^2}{\sum_{x \in \mathcal{X}} D(x)^2}$$

用通俗的话来讲，这个算法其实就是想让初始的几个中心点尽可能"离得越远越好"，不要扎堆。

[截图待插入: K-means++示意图，展示数据空间中四个簇，每个簇中用黄色星形标记选出的中心点，中心点分散在不同簇中]

### K-means++ 的问题

**What's wrong with K-means++?**

- 需要执行K次数据遍历
- 在大规模数据场景中，不仅数据体量庞大，K值通常也很大（例如轻易达到1000量级）
- 可扩展性差！

### 可伸缩K-means: K-means||

**Intuition of**

- K-means++ 每轮迭代只采样一个中心点并立即更新概率分布
- 如果我们通过独立采样扩大单次采样量会怎样？
- 这相当于大幅降低分布更新的频率
- 采用更粗粒度的采样方式
- 实践证明这种方式完全可行：这就是 **K-means||** 算法的核心思想

**主要思路**在于改变每次遍历时的取样策略，并非按照 k-means++ 那样每次遍历只取样一个样本，而是每次遍历取样A个样本，重复该取样过程大约 B次，共得到 A*B个样本点组成的集合

- 然后再聚类这 A*B个点成 k 个点，最后将这 k 个点作为初始聚类中心
- 实际实验证明**一般5次重复取样就可以得到一个较好的聚类初始中心**

### K-means 练习

假设有如下八个点：(1, 2)，(2, 4)，(1, 9)，(6, 5)，(4, 2)，(7, 2)，(8, 2)，(4, 3)，使用k-Means算法对其进行聚类。设K=2，初始聚类两个中心点坐标分别为(1, 2)，(8, 2)，算法使用曼哈顿距离（L1 距离）作为距离度量。请计算一次迭代后的聚类中心坐标，并写出上述八个点分别属于哪个簇（cluster）。

对于点(1, 2)：

$$d_{11} = |1-1| + |2-2| = 0, \quad d_{12} = |1-8| + |2-2| = 7$$

因此，(1, 2)应当被聚为第 1 类；

对于点(2, 4)：

$$d_{21} = |2-1| + |4-2| = 3, \quad d_{22} = |2-8| + |4-2| = 8$$

因此，(2, 4)应当被聚为第 1 类；

对于点(1, 9)：

$$d_{31} = |1-1| + |9-2| = 7, \quad d_{32} = |1-8| + |9-2| = 14$$

因此，(1, 9)应当被聚类为第 1 类；

对于点(6, 5)：

$$d_{41} = |6-1| + |5-2| = 8, \quad d_{42} = |6-8| + |5-2| = 5$$

因此，(6, 5)应当被聚类为第 2 类；

对于点(4, 2)：

$$d_{51} = |4-1| + |2-2| = 3, \quad d_{52} = |4-8| + |2-2| = 4$$

因此，点(4, 2)应当被聚类为第 1 类；

对于点(7, 2)：

$$d_{61} = |7-1| + |2-2| = 6, \quad d_{62} = |7-8| + |2-2| = 1$$

因此，点(7, 2)应当被聚类为第 2 类；

对于点(8, 2)：

$$d_{71} = |8-1| + |2-2| = 7, \quad d_{72} = |8-8| + |2-2| = 0$$

因此，点(8, 2)应当被聚类为第 2 类；

对于点(4, 3)：

$$d_{81} = |4-1| + |3-2| = 4, \quad d_{82} = |4-8| + |3-2| = 5$$

因此，点(4, 3)应当被聚类为第 1 类。

综上，类 1 的点有(1,2) (2,4) (1,9) (4,2) (4,3)，则对应的聚类中心为：(2.4, 4)。

### DBSCAN

DBSCAN是一种基于密度（Density-based）的聚类算法。

- **密度Density**：指定半径范围内包含的数据点数量
- **核心点core point**：在半径范围内包含至少MinPts个点的点
  - 这类点位于簇的内部核心区域
- **边界点border point**：在半径范围内包含少于MinPts个点，但位于某个核心点的邻域内
  - 处于簇的边界区域
- **噪声点noise point**：既不是核心点也不是边界点的点
  - 属于异常数据或离散点

**密度** = 指定半径（Eps）范围内包含的数据点数量。

- A point is a **core point** if it has more than a specified number of points (MinPts) within Eps. These are points that are at the interior of a cluster — **核心点**
- A **border point** has fewer than MinPts within Eps, but is in the neighborhood of a core point — **边界点**
- A **noise point** is any point that is not a core point or a border point — **噪声点**

### DBSCAN 算法

**Center-based approach**

**基于中心的处理方法**

**算法 S.4 DBSCAN 算法步骤**

1. 将所有点标记为**核心点**、**边界点**或**噪声点**。
2. **剔除噪声点**（使其不参与后续聚类）。
3. 在所有**相互距离在 Eps 范围内的核心点之间建立连接边**。
4. 将每个**连通的核心点**组形成一个独立的**簇**。
5. 将每个**边界点分配**到与其关联的**核心点所在的簇**中。

**Algorithm 8.4 DBSCAN algorithm.**

1. Label all points as core, border, or noise points.
2. Eliminate noise points.
3. Put an edge between all core points that are within $Eps$ of each other.
4. Make each group of connected core points into a separate cluster.
5. Assign each border point to one of the clusters of its associated core points.

**算法 8.4 DBSCAN 算法：**
1. 将所有点标记为核心点、边界点或噪声点。
2. 剔除噪声点。
3. 在任意两个距离不超过 $Eps$ 的核心点之间连边。
4. 将每个连通的核心点集合作为一个独立簇。
5. 将每个边界点分配到与它关联的核心点所在的簇。
