## 一句话总结

本讲介绍决策树分类模型和经典算法 CLS、ID3、C4.5、CART，重点是属性选择指标：信息增益、增益率和基尼指数。

## 决策树模型

决策树是一种树状分类模型：

- 内部结点: 对某个属性做测试。
- 分支: 测试结果或属性取值。
- 叶结点: 预测类别。

学习过程是从训练数据中确定划分属性；预测过程是从根结点开始，按测试结果沿分支走到叶结点。

建模阶段包括：

- Tree construction: 递归选择属性划分样本。
- Tree pruning: 删除受噪声和异常值影响较大的分支，缓解过拟合。

## CLS

CLS 是早期决策树学习算法，基本流程：

1. 从空树开始。
2. 若样本全属同一类，则生成叶结点。
3. 否则选择某属性作为测试属性。
4. 按属性取值划分子集。
5. 对子集递归建树。

问题：不同属性选择顺序会产生不同决策树，因此需要有原则地选择最优划分属性。

## 熵与信息增益

熵衡量离散变量的不确定性：

$H(X)=-\sum_x p(x)\log_2 p(x)$

二分类中，当 $p=0.5$ 时熵最大；当某类概率为 0 或 1 时熵为 0。

条件熵：

$H(Y|X)=\sum_x p(x)H(Y|X=x)$

联合熵：

$H(X,Y)=H(X)+H(Y|X)$

互信息：

$I(X;Y)=H(Y)-H(Y|X)=H(X)-H(X|Y)$

## ID3 与信息增益

ID3 使用信息增益选择测试属性。对数据集 $D$ 和属性 $A$：

$Gain(D,A)=H(D)-H(D|A)$

直观理解：选择能最大幅度降低类别不确定性的属性。

例子中“是否买电脑”数据集可通过计算各属性与类别标签的互信息，选择信息增益最大的属性作为根节点。

缺点：信息增益偏向取值很多的属性，例如用户 ID 这类几乎唯一的属性可能产生高增益但泛化差。

## C4.5 与增益率

C4.5 用增益率缓解 ID3 偏好多取值属性的问题：

$GainRatio_A(D)=Gain_A(D)/SplitInfo_A(D)$

$SplitInfo_A(D)$ 衡量属性自身划分数据的“分裂信息”。增益率相当于对信息增益进行正则化。

## CART 与基尼指数

基尼指数衡量数据集不纯度：

$gini(D)=1-\sum_j p_j^2$

二分类时：

$gini(D)=2p(1-p)$

若数据集被划分为 $D_1,D_2$：

$gini_{split}(D)=\frac{N_1}{N}gini(D_1)+\frac{N_2}{N}gini(D_2)$

CART 选择使 $gini_{split}$ 最小的属性和切分点。

## 连续属性处理

对连续属性 $A$：

1. 将样本按属性值排序。
2. 在相邻值之间取候选切分点 $(a_i+a_{i+1})/2$。
3. 对每个切分点计算信息增益或基尼指数。
4. 选择最优切分点。

CART 通常使用二叉划分，即对 $A\le a$ 和 $A>a$ 递归建树。
