## 强化学习

### 动态决策场景

在大量序列决策场景中，智能体需要与环境进行动态交互，通过交互获得的数据来学习决策策略。

典型应用场景包括：博弈游戏（如AlphaGo）、无人机空战、机器人控制、交通灯控制、无人驾驶、智能电网。

### 什么是强化学习？

强化学习讨论的问题是：在一个复杂不确定的环境中，智能体如何通过与环境进行大量试错来学习一个最优决策策略。

### 智能体和环境

强化学习（Reinforcement Learning, RL）：

- 受动物的学习过程启发，在试错中学习
- 不断与环境交互，获取学习所需的经验

1. 环境模型

### 奖励

- **奖励（reward）**：一般对应着相关任务需求

**正向奖励：**

- 收集苹果：+50

- ...

强化学习的目标就是最大化在环境中所能获得的奖励。

### 马尔科夫决策过程

**马尔可夫决策过程（Markov Decision Process, MDP）**：对于一个马尔可夫决策过程，可以用一个五元组 $\langle \mathcal{S}, \mathcal{A}, p, R, \gamma \rangle$ 来表示：

- **状态空间** $\mathcal{S}$ 是所有状态组成的集合
- **动作空间** $\mathcal{A}$ 是所有动作组成的集合
- **转移函数** $p$ 刻画了状态之间的转移概率：

$$p(s'|s,a) = Pr\{S_{t+1} = s' | S_t = s, A_t = a\}$$

- $R$ 表示奖励函数，不止取决于当前的状态，还受到动作的影响：

$$R_t = R(S_t, A_t, S_{t+1})$$

- $\gamma$ 表示折扣因子，$\gamma \in [0, 1]$

### 策略函数

- **策略（policy）**：从状态空间到动作空间的映射
- **随机性策略** $\pi(a|s)$：输出在状态 $s$ 下选择动作 $a$ 的概率

$$\sum_{a \in \mathcal{A}} \pi(a|s) = 1$$

- **确定性策略** $\pi(s)$：输出在状态 $s$ 下选择的确定动作

$$a = \pi(s)$$

确定性策略可以在《冒险岛》中取得不错的表现；石头剪刀布中则更适合使用随机性策略。

**策略的具体形式：**

- **表格策略**：用表格直接记录每个状态对应的动作方向（如网格迷宫中每个格子的箭头方向）
- **参数化策略**：用神经网络参数化策略函数

$$\pi_\theta(s) \triangleq \pi(s; \theta)$$

$$\pi_\theta(a|s) \triangleq \pi(a|s; \theta)$$

确定性策略：$a = \pi_\theta(s)$

随机性策略：$\pi_\theta(a|s) \sim \mathcal{N}(\mu, \sigma^2)$

### 回报

- **回报（return）**：回报 $G_t$ 为从时间步 $t$ 开始的折扣奖励总和

$$G_t = R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \ldots = \sum_{k=0}^{\infty} \gamma^k R_{t+k+1}$$

- 折扣因子 $\gamma \in [0,1]$ 用于衡量未来奖励在当前所具备的价值
- $k+1$ 步后的奖励 $R$ 在当前的价值被定义为 $\gamma^k R$

[截图待插入: 回报的时间序列图，展示从 $(S_t, A_t)$ 到 $(S_{t+3}, A_{t+3})$ 的状态-动作序列，以及对应的 $R_{t+1}$、$\gamma R_{t+2}$、$\gamma^2 R_{t+3}$ 对 $G_t$ 的贡献]

强化学习着眼于**最大化累积奖励**，也即上述定义的回报（return）。

### 价值函数

**价值函数用于评估一个策略的好坏**

- **状态价值函数（State-value function）**：以状态 $s$ 为起始状态且按照策略 $\pi$ 行动的期望回报。

$$V_\pi(s) \triangleq \mathbb{E}_\pi[G_t | S_t = s]$$

- **动作价值函数（Action-value function）**：以状态 $s$ 为起始状态且采取动作 $a$，然后按照策略 $\pi$ 行动得到的回报。

$$Q_\pi(s,a) \triangleq \mathbb{E}_\pi[G_t | S_t = s, A_t = a]$$

两者之间的关系：

$$V_\pi(s) = \sum_{a \in \mathcal{A}} \pi(a|s)\, Q_\pi(s, a)$$

$$Q_\pi(s,a) = \mathcal{R}_s^a + \gamma \sum_{s' \in \mathcal{S}} \mathcal{P}_{ss'}^a\, V_\pi(s')$$

### 最优价值函数

- **最优价值函数：**

$$V_*(s) = \max_\pi V_\pi(s),\ \forall s \in \mathcal{S}$$

$$Q_*(s,a) = \max_\pi Q_\pi(s,a)$$

- $V_*(s)$ 与 $Q_*(s,a)$ 之间的关系：

$$V_*(s) = \max_a Q_*(s,a)$$

$$Q_*(s,a) = \mathcal{R}_s^a + \gamma \sum_{s' \in \mathcal{S}} \mathcal{P}_{ss'}^a\, V_*(s')$$

**最优策略：可以通过 $Q_*(s,a)$ 得到一个确定性的最优策略**

$$\pi_*(a|s) = \begin{cases} 1 & \text{if } a = \underset{a \in \mathcal{A}}{arg\,max}\, Q_*(s,a) \\ 0 & \text{otherwise} \end{cases}$$

### 贝尔曼方程

状态价值函数可以被拆分为即时奖励与后续状态的折扣价值：

$$V_\pi(s) = \mathbb{E}_\pi[R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \ldots | S_t = s]$$

$$= \mathbb{E}_\pi[R_{t+1} + \gamma(R_{t+2} + \gamma R_{t+3} + \ldots) | S_t = s]$$

$$= \mathbb{E}_\pi[R_{t+1} + \gamma G_{t+1} | S_t = s]$$

$$= \mathbb{E}_\pi[R_{t+1} + \gamma V_\pi(S_{t+1}) | S_t = s]$$

$$= \sum_{a \in \mathcal{A}} \pi(a|s) \left( \mathcal{R}_s^a + \gamma \sum_{s' \in \mathcal{S}} \mathcal{P}_{ss'}^a\, V_\pi(s') \right)$$

动作状态价值函数也可以被拆分为即时奖励与后续状态的折扣价值：

$$Q_\pi(s,a) = \mathbb{E}_\pi[R_{t+1} + \gamma Q_\pi(S_{t+1}, A_{t+1}) | S_t = s, A_t = a]$$

$$= \mathcal{R}_s^a + \gamma \sum_{s' \in \mathcal{S}} \mathcal{P}_{ss'}^a \sum_{a \in \mathcal{A}} \pi(a|s')\, Q_\pi(s', a)$$

### 贝尔曼最优方程

- **最优状态价值函数：**

$$V_*(s) = \max_\pi V_\pi(s)$$

$$= \max_\pi \mathbb{E}_\pi[\mathcal{R}_s^a + \gamma V_\pi(s')]$$

$$= \max_\pi \mathbb{E}_\pi[\mathcal{R}_s^a + \gamma V_*(s')]$$

$$= \max_a \left( \mathcal{R}_s^a + \gamma \sum_{s' \in \mathcal{S}} \mathcal{P}_{ss'}^a\, V_*(s') \right)$$

$$\boxed{V_*(s) = \max_a Q_*(s,a)}$$

- **最优动作价值函数：**

$$Q_*(s,a) = \max_\pi Q_\pi(s,a)$$

$$= \max_\pi \mathbb{E}_\pi[\mathcal{R}_s^a + \gamma Q_\pi(s',a')]$$

$$= \mathbb{E}_\pi\left[\mathcal{R}_s^a + \gamma \max_{a'} Q_*(s',a')\right]$$

$$= \mathcal{R}_s^a + \gamma \sum_{s' \in \mathcal{S}} \mathcal{P}_{ss'}^a \max_{a'} Q_*(s',a')$$

$$\boxed{Q_*(s,a) = \mathcal{R}_s^a + \gamma \sum_{s' \in \mathcal{S}} \mathcal{P}_{ss'}^a\, V_*(s')}$$

### 表格型强化学习场景（Q表）

冰湖环境较为简单，我们可以很容易地构建所有状态动作对的Q表。

| 状态\动作 | 上 | 下 | 左 | 右 |
|---------|-----|-----|-----|-----|
| $s_0$ | 0.0 | 1.0 | 0.0 | 0.0 |
| $s_1$ | 0.0 | 1.0 | 0.0 | 1.0 |
| $s_2$ | 0.0 | 0.5 | 0.0 | 0.5 |
| $s_3$ | 0.5 | 2.0 | 0.0 | 0.5 |
| $s_4$ | 0.0 | 0.0 | 0.0 | 0.5 |
| ... | ... | ... | ... | ... |

根据 Q 表，我们可以知道此时每个状态对应的最优策略，即通过每个动作对应的 Q 值大小判断。对于状态 $s_0$，最优策略是采取动作"下"，状态 $s_1$ 有两个最优动作"下"和"右"。

### 表格型强化学习算法

通常，Q 表会被初始化为全 0，通过智能体和环境交互的结果不断更新 Q 表。当Q 表中的每个值函数更新幅度小于某个阈值时，表示着 Q 表已经收敛，那么每个状态对应的策略就是强化学习问题的最优策略。

Q 表的更新方法主要分为两大类：

- **有环境模型的求解方法**：需要知道环境的状态转移函数以及奖励函数，通过动态规划方法递归求解最优策略，常用求解方法主要包括：**策略迭代算法、值迭代算法**；
- **无环境模型的求解方法**：无需显式知道环境的状态转移函数和奖励函数，而是通过智能体与环境交互的数据更新 Q 值，常用求解方法主要包括：**蒙特卡洛方法，时序差分方法**

### 动态规划算法

- 动态规划（Dynamic Programming, DP）是一种适用于解决具有最优子结构（Optimal Substructure）和重叠子问题（overlapping subproblem）两个性质的算法思想。
- 20 世纪 50 年代初，美国数学家贝尔曼（R.Bellman）等人在研究多阶段决策过程的优化问题时，提出了著名的最优化原理，从而创立了动态规划。
- 该方法应用十分广泛，包括但不限于工程技术、经济、工业生产、军事等。

**性质一：最优子结构**

- 原问题可以拆分成若干个子问题
- 全局最优解可以拆分成子问题的最优解

**性质二：重叠子问题**

- 子问题可能会出现多次
- 子问题的解可以被记录并重复利用

贝尔曼方程含有递归分解 -> 马尔可夫决策过程 <- 价值函数存储了状态价值并被重复利用

策略迭代、价值迭代 -> 马尔可夫决策过程

### 动态规划算法：策略评估

**策略评估（Policy Evaluation）**：评估给定的策略 $\pi$（求解该策略对应的价值函数）

- 迭代调用贝尔曼期望方程进行更新
- 使用同步更新（每次迭代更新所有状态的价值）

$$V_\pi(s) = \sum_{a \in \mathcal{A}} \pi(a|s)\left(\mathcal{R}_s^a + \gamma \sum_{s' \in \mathcal{S}} \mathcal{P}_{ss'}^a\, V_\pi(s')\right)$$

对于 $k = 1 : H$（$H$ 是让价值函数收敛所需的迭代次数）对于所有状态 $s \in \mathcal{S}$

$$v_{k+1}(s) = \sum_{a \in \mathcal{A}} \pi(a|s)\left(\mathcal{R}_s^a + \gamma \sum_{s' \in \mathcal{S}} \mathcal{P}_{ss'}^a\, v_k(s')\right)$$

**收敛性证明：**

利用 $\infty$ 范数度量两个状态价值函数 $u$ 与 $v$ 之间的距离，即状态价值的最大差异：

$$\|u - v\|_\infty = \max_{s \in \mathcal{S}} |u(s) - v(s)|$$

定义贝尔曼期望算子 $B_\pi$：

$$B_\pi v_k = R_\pi + \gamma P_\pi v_k$$

该算子是一个 $\gamma$-压缩：

$$\|B_\pi u - B_\pi v\|_\infty = \|(R_\pi + \gamma P_\pi u) - (R_\pi + \gamma P_\pi v)\|_\infty$$

$$= \|\gamma P_\pi(u - v)\|_\infty \leq \|\gamma P_\pi\| \|u - v\|_\infty \|_\infty$$

$$\leq \gamma \|u - v\|_\infty$$

由压缩映射定理可知 $v_\pi$ 是 $B_\pi$ 的唯不动点，因此策略评估能够收敛到 $v_\pi$。

**Algorithm 1.1 策略评估**

```
Input: 策略 π(s)，状态转移 P(s'|s,a)，奖励函数 R(s,a)，阈值 ε，衰减因子 γ，值函数 V(s)
Output: 值函数 V_π(s)

1: k ← 0, V_k(s) ← V(s)
2: for k = 0, 1, 2, ... do
3:   for 每个状态 s ∈ S do
4:     利用公式计算 V_{k+1}(s)
5:   end for
6:   if |V_{k+1} - V_k| < ε then
7:     V_π ← V_{k+1}
8:     return V_π
9:   end if
10: end for
```

### 动态规划算法 - 策略提升 (Policy Improvement)

策略提升：依据策略评估得到的值函数，改进策略 $\pi$

- 根据价值函数得到一个更优的策略
- 可以依据价值函数定义策略之间的偏序关系

$$\pi \geq \pi' \text{ if } v_\pi(s) \geq v_{\pi'}(s), \forall s \in \mathcal{S}$$

- 对于一个确定性策略 $(a = \pi(s))$，可以通过贪心的最大化动作价值函数得到一个新策略

$$\pi'(s) = \arg\max_{a \in \mathcal{A}} Q_\pi(s, a)$$

而策略 $\pi'$ 比策略 $\pi$ 更好或至少一样好

[截图待插入: 游戏画面示意图 - 高手玩家对应 $v_{\pi_{\text{高手}}}(s)$ 较大，新手玩家对应 $v_{\pi_{\text{新手}}}(s)$ 较小]

$$Q_\pi(s, \pi'(s)) = \max_{a \in \mathcal{A}} Q_\pi(s, a) \geq Q_\pi(s, \pi(s)) = v_\pi(s)$$

$$v_\pi(s) \leq Q_\pi(s, \pi'(s)) = \mathbb{E}_{\pi'}[R_{t+1} + \gamma v_\pi(S_{t+1}) | S_t = s]$$

$$\leq \mathbb{E}_{\pi'}[R_{t+1} + \gamma Q_\pi(S_{t+1}, \pi'(S_{t+1})) | S_t = s]$$

$$\leq \mathbb{E}_{\pi'}[R_{t+1} + \gamma R_{t+2} + \gamma^2 Q_\pi(S_{t+2}, \pi'(S_{t+2})) | S_t = s]$$

$$\leq \mathbb{E}_{\pi'}[R_{t+1} + \gamma R_{t+2} + \ldots | S_t = s] = v_{\pi'}(s)$$

### 动态规划算法 - 策略迭代 (Policy Iteration)

- **策略评估** 估计原策略价值函数 $v_\pi$
  - 迭代策略评估 $v_{k+1}(s) = \sum_{a \in \mathcal{A}} \pi(a|s) \left( \mathcal{R}_s^a + \gamma \sum_{s' \in \mathcal{S}} \mathcal{P}_{ss'}^a v_k(s') \right)$
- **策略提升** 生成新策略 $\pi' \geq \pi$
  - 贪心策略提升

[截图待插入: 策略迭代收敛示意图 - 初始化 $v, \pi$，交替进行策略评估 $v \leftarrow v_\pi$ 和策略提升 $\pi \leftarrow greedy(v)$，最终收敛到 $v_*, \pi_*$]

**Algorithm 1.2 策略迭代**

**Input:** 策略 $\pi(s)$，状态转移 $P(s'|s,a)$，奖励函数 $R(s,a)$，阈值 $\epsilon$，衰减因子 $\gamma$，值函数 $V(s)$

**Output:** 最优策略 $\pi_*(s)$，最优值函数 $V_{\pi_*}(s)$

1: $k \leftarrow 0$，$\pi_k(s) \leftarrow \pi(s)$，$V_k(s) \leftarrow V(s)$。

2: **for** $k = 0, 1, 2, \cdots$ **do**

3: 利用策略评估得到当前策略 $\pi_k(s)$ 的值函数 $V_k(s)$

4: 利用策略改进公式得到新策略 $\pi_{k+1}(s)$

5: **if** $\pi_{k+1}(s) = \pi_k(s)$ **then**

6: $\pi_*(s) \leftarrow \pi_k(s)$，$V_{\pi_*}(s) \leftarrow V_k(s)$

7: **return** $\pi_*(s)$，$V_{\pi_*}(s)$

8: **end if**

9: **end for**

### 动态规划算法 - 值迭代 (Value Iteration)

任意的最优策略都可以分解为两个部分：

- 最优的第一个动作 $A_*$
- 对于后继状态 $S'$ 的最优策略 $\pi_*$

**定理：** 策略 $\pi(a|s)$ 能够达到状态 $s$ 的最优价值，$v_\pi(s) = v_*(s)$，当且仅当对于状态 $s$ 的任意后继状态 $s'$，策略 $\pi$ 均能达到其最优价值，$v_\pi(s') = v_*(s')$

如果知道子问题的解 $v_*(s')$，则 $v_*(s)$ 可以通过下式得到：

$$v_*(s) \leftarrow \max_a \left( \mathcal{R}_s^a + \gamma \sum_{s' \in \mathcal{S}} \mathcal{P}_{ss'}^a v_*(s') \right)$$

价值迭代的思路即是迭代应用上式直至收敛。

### 动态规划算法 - 值迭代 (续)

策略迭代：$v_{k+1}(s) = \sum_{a \in \mathcal{A}} \pi(a|s) \left( \mathcal{R}_s^a + \gamma \sum_{s' \in \mathcal{S}} \mathcal{P}_{ss'}^a v_k(s') \right)$

**不同于策略迭代，价值迭代中没有显式的策略**

- 迭代的调用贝尔曼最优方程进行更新
- 使用同步更新（每次迭代更新所有状态的价值）

对于 $k = 1:H$（$H$ 是迭代次数）

对于所有状态 $s \in \mathcal{S}$

$$v_{k+1}(s) = \max_a \mathcal{R}_s^a + \gamma \sum_{s' \in \mathcal{S}} \mathcal{P}_{ss'}^a v_k(s')$$

**Algorithm 1.3 值迭代**

**Input:** 状态转移 $P(s'|s,a)$，奖励函数 $R(s,a)$，阈值 $\epsilon$，衰减因子 $\gamma$，值函数 $V(s)$

**Output:** 最优策略 $\pi_*(s)$，最优值函数 $V_{\pi_*}(s)$

1: $k \leftarrow 0$，$V_k(s) \leftarrow V(s)$。

2: **for** $k = 0, 1, 2, \cdots$ **do**

3: **for** 每个状态 $s \in S$ **do**

4: 利用公式计算 $V_{k+1}(s)$

5: **end for**

6: **if** $|V_{k+1} - V_k| < \epsilon$ **then**

7: $V_{\pi_*}(s) \leftarrow V_{k+1}$

8: 利用公式计算最优策略 $\pi_*(s)$

9: **return** $\pi_*(s)$，$V_{\pi_*}(s)$

10: **end if**

11: **end for**

### 动态规划算法 - 总结

| 求解问题 | 贝尔曼方程 | 算法 |
|---------|-----------|------|
| 预测（求解价值函数） | 贝尔曼期望方程 | 策略评估 |
| 控制（求解最优策略） | 贝尔曼期望方程 + 贪心策略提升 | 策略迭代 |
| 控制（求解最优策略） | 贝尔曼最优方程 | 价值迭代 |

- 以上算法都基于状态价值函数 $v_*(s)$ 或 $v_\pi(s)$，对于 $n$ 个状态、$m$ 个动作的MDP，每次迭代的复杂度为 $O(mn^2)$
- 也可以应用于动作价值函数 $q_*(s)$ 或 $q_\pi(s)$，每次迭代复杂度为 $O(m^2n^2)$

- 20世纪40年代，在科学家冯·诺伊曼、斯塔尼斯拉夫·乌拉姆和尼古拉斯·梅特罗波利斯于洛斯阿拉莫斯国家实验室为核武器计划工作时，发明了蒙特卡洛估计方法。

- 蒙特卡洛估计方法是一种以概率统计理论为指导的数值计算方法，与之相对的是确定性算法。

### 蒙特卡洛方法中的策略评估

- 记 $G^{(i)(j)}(s)$ 为在第 $i$ 个回合中第 $j$ 次访问状态 $s$ 对应的累积回报
- 估计**状态价值函数**：

$$v_\pi(s) = \mathbb{E}_\pi[G_t | S_t = s] \approx \frac{1}{N} \sum_{i=1}^{N} G^{(i)(j)}(s)$$

- 记 $G^{(i)(j)}(s, a)$ 为在第 $i$ 个回合中第 $j$ 次访问状态动作对 $(s, a)$ 对应的累积回报
- 估计**动作状态价值函数**：

$$q_\pi(s, a) = \mathbb{E}_\pi[G_t | S_t = s, A_t = a] \approx \frac{1}{N} \sum_{i=1}^{N} G^{(i)(j)}(s, a)$$

可以发现，蒙特卡洛策略评估只需要采样数据，而不需要环境模型 $\mathcal{P}_{ss'}^a$！

[截图待插入: 网格世界中多条采样轨迹示意图 - 展示4条不同的从起点到终止状态T的路径]

- 通过采样多条样本轨迹，利用其累积回报的均值来估计状态价值函数

[截图待插入: 多回合采样轨迹示意图 - 第1、2、...、k个回合中状态s的多次访问及其对应的回报 $G^{(i)(j)}(s)$]

$G^{(i)(j)}(s)$ 表示在第 $i$ 个回合中第 $j$ 次访问状态 $s$ 的回报

### 蒙特卡洛方法中的策略提升

$$\pi'(s) = \arg\max_{a \in \mathcal{A}} Q_\pi(s, a) \qquad Q_\pi(s, a) = \mathcal{R}_s^a + \gamma \sum_{s' \in \mathcal{S}} \mathcal{P}_{ss'}^a V_\pi(s')$$

- 已经估计了状态价值函数 $V_N(s)$，策略提升部分为：

$$\arg\max_\pi R(s, a) + \gamma \sum_{s'} p(s'|s, a) V_N(s')$$

- 已经估计了动作状态价值函数 $Q_N(s, a)$，策略提升部分为：

$$\arg\max_\pi Q_N(s, a)$$

可以发现，估计动作状态价值函数不需要知道环境模型，计算更为方便（但也需要更多的数据样本）

### 蒙特卡洛估计方法 - 优缺点

**策略评估：** $Q_N(s, a) = \frac{1}{N} \sum_{i=1}^{N} G^{(i)(j)}(s, a)$

**策略提升：** $\arg\max_\pi Q_N(s, a)$

| 优点 | 缺点 |
|------|------|
| **无需环境模型**，只需与环境交互采样样本数据即可学习策略 | 若采样的回合个数不够多，容易收敛到次优策略 |
| 评估某个状态的价值函数与其它状态无关，从而可以只评估部分重要的状态 | 由于需要多个回合的回报，通常只适用有限步长的场景 |
| 由于不使用贝尔曼方程更新，适用于非马尔可夫场景 | |

### 蒙特卡洛策略评估收敛需要两个很强的假设

- **第一假设：** 需要**无限多回合**产生的数据
- **第二假设：** 需要**试探性出发假设**产生数据
  - 也即，对于每一对状态动作对 $(s, a)$ 都有非零概率被选为回合的起点

这两个假设保证了每一个状态动作对 $(s, a)$ 都能被访问到无数次，从而蒙特卡洛策略评估可以收敛到精确的动作状态价值函数。

然而，这两个假设在实践中无法实现。为了得到一个可行的算法，我们需要去除这两个假设。

### 蒙特卡洛策略评估 - 放松假设

有两类方法可以对第一个假设进行放松：

- 在每次策略评估时，通过**有限多个回合**产生的数据，对精确值计算一个具有一定误差的近似值
  - 需要做一些假设，来分析逼近误差的幅度和出现概率的上下界，并采取足够多的步数来保证这些界足够小
  - **误差控制型：** 用数学担保替代无限数据，如同保险公司用精算模型替代全量数据
  - 这类方法虽然可以保证收敛到较好的近似水平，但在实践中仍然需要大量的回合数据来用于计算
- 在每次策略评估时，采用**广义策略迭代**的思想，不再要求策略改进前就完成策略评估
  - 例如，在每一回合结束时，就使用观测到的回报进行策略评估，然后在该回合访问到的每一个状态上进行策略的改进
  - **迭代进化型：** 用增量更新替代完整评估，如同拼图时边找边拼而非先找齐所有碎片

### 蒙特卡洛策略评估 - 通过"软性"策略放松第二假设

通过"软性"策略对第二个假设进行放松

- 一个策略称为**软性**的，若满足任意 $s, a$，都有 $\pi(a|s) > 0$
- $\epsilon$-贪心策略是保证充分探索的一个简单软性策略
  - 以 $1 - \epsilon$ 的概率贪心地选择当前最优动作
  - 以较小的 $\epsilon$ 概率随机选择一个动作

$$\pi'(a|s) = \begin{cases} \epsilon/|\mathcal{A}| + 1 - \epsilon & \text{if } a = \arg\max_{a \in \mathcal{A}} Q_N(s, a) \\ \epsilon/|\mathcal{A}| & \text{otherwise} \end{cases}$$

[截图待插入: $\epsilon$-贪心策略示意图 - 状态s以概率 $1-\epsilon$ 选择最优动作 $a^* = \arg\max_{a \in \mathcal{A}} Q_N(s,a)$，以概率 $\epsilon/|\mathcal{A}|$ 选择其他动作]

### 时序差分方法

| 蒙特卡洛方法：从完整轨迹中估计值 | 动态规划方法：用下一步值计算当前值 |
|--------------------------------|--------------------------------|
| $V(s) \leftarrow Mean\{G_{t:T} | S_t = s\}$ | $v(s) = \max_a \mathcal{R}_s^a + \gamma \sum_{s' \in \mathcal{S}} \mathcal{P}_{ss'}^a v(s')$ |
| **缺点：** | **缺点：** |
| - 只能在**整个episode结束**时更新策略 | - 要求**下一步状态准确值**已知 |
| - 策略收敛缓慢 | - 要求转移函数 $\mathcal{P}_{ss'}^a$ 已知 |
| - 值估计方差较大 | |

**时序差分方法：用下一步估计值估计当前值**

$$V(s) \leftarrow V(s) + \alpha(R_{t+1} + \gamma V(s') - V(s))$$

综合以上两者思想，实现优势互补：

- 可以在episode中更新策略
- 无需已知转移函数 $\mathcal{P}_{ss'}^a$ 或准确值函数
- 值估计方差较小

### 时序差分方法 - 推导

蒙特卡洛方法：用整个episode的累积回报 $G_t$ 更新 $V(S_t)$

$$V(S_t) \leftarrow V(S_t) + \alpha(G_t - V(S_t))$$

更新目标：$G_t = R_{t+1} + \gamma R_{t+2} + \ldots + \gamma^T R_{t+T}$

回顾贝尔曼等式的推导：

$$G_t = R_{t+1} + \gamma R_{t+2} + \ldots + \gamma^T R_{t+T} = R_{t+1} + \gamma(R_{t+2} + \gamma R_{t+3} + \ldots) \approx R_{t+1} + \gamma V(S_{t+1})$$

从而：$G_t \approx R_{t+1} + \gamma V(S_{t+1})$

**时序差分方法：** 将 $R_{t+2} + \gamma R_{t+3} + \ldots$ 替换为 $V(S_{t+1})$

$$V(S_t) \leftarrow V(S_t) + \alpha(R_{t+1} + \gamma V(S_{t+1}) - V(S_t))$$

- $R_{t+1} + \gamma V(S_{t+1})$ 被称为 **TD 目标**
- $\delta_t = R_{t+1} + \gamma V(S_{t+1}) - V(S_t)$ 被称为 **TD 误差**

[截图待插入: TD方法示意图 - 用估计值 $V(S_{t+1})$ 来作为自身 $V(S_t)$ 的学习目标，利用现实与预期的误差，来微调当前状态的价值]

### 时序差分方法 - 三种方法对比

| 动态规划 | 蒙特卡洛 | 时序差分 |
|---------|---------|---------|
| $V(S_t) \leftarrow \mathbb{E}_\pi[R_{t+1} + \gamma V(S_{t+1})]$ | $V(S_t) \leftarrow V(S_t) + \alpha(G_t - V(S_t))$ | $V(S_t) \leftarrow V(S_t) + \alpha(R_{t+1} + \gamma V(S_{t+1}) - V(S_t))$ |

[截图待插入: 三种方法的树形展开对比图 - 动态规划展开到下一层，蒙特卡洛展开到叶子节点，时序差分只展开到下一层的一个节点]

### 时序差分方法 - 例子

两个状态 A、B；不考虑折扣 $\gamma = 1$

**已知：** 现有 8 条轨迹 $(S_1, R_2, S_2, R_3, \ldots, S_T)$：

(A, 0, B, 0), (B, 1), (B, 1), (B, 1), (B, 1), (B, 1), (B, 1), (B, 0)

显然 $V(B) = 0.75$

**问题：** 状态 A 的价值 $V(A)$？

$$V(S_t) \leftarrow V(S_t) + \alpha(R_{t+1} + \gamma V(S_{t+1}) - V(S_t))$$

### 时序差分方法 - SARSA

**使用 TD 方法作为策略迭代中的策略评估方法**

- 模型未知，因此直接估计动作价值函数

$$Q(S_t, A_t) \leftarrow Q(S_t, A_t) + \alpha(R_{t+1} + \gamma Q(S_{t+1}, A_{t+1}) - Q(S_t, A_t)) \quad (*)$$

策略迭代两步走：

- **策略评估：** 使用 (*)式估计动作价值函数 $Q = q_\pi$
- **策略提升：** 使用 $\epsilon$-greedy 策略提升方法

[截图待插入: SARSA策略迭代收敛示意图 - 初始化 $Q, \pi$，交替进行策略评估 $Q = q_\pi$ 和策略提升 $\pi = \epsilon\text{-greedy}(Q)$，最终收敛到 $Q_*, \pi_*$]

由于(*)式使用 $<S_t, A_t, R_{t+1}, S_{t+1}, A_{t+1}>$ 进行更新，因此称为 **SARSA** 算法。

**Algorithm 1.6 SARSA**

**Input:** 迭代轮数 T，更新步长 $\alpha$，衰减因子 $\gamma$，最大探索率 $\epsilon_{start}$，最小探索率 $\epsilon_{min}$，最大交互回合数 $N$，最大时间步长 $T$

**Output:** 最优值函数 $Q_*$

1: 随机初始化所有的状态动作对应的值函数 $Q$。

2: **for** 每个交互回合 $k = 0, 1, \cdots, N$ **do**

3: 初始化初始状态 $s_0$

4: 在状态 $s_0$ 利用 $\epsilon$-greedy 策略选择动作 $a_0$

5: **for** $t = 0, 1, \cdots, T$ **do**

6: 在状态 $s_t$ 执行当前动作 $a_t$，得到新状态 $s_{t+1}$ 和奖励 $r_{t+1}$

7: 在新状态 $s_{t+1}$ 利用 $\epsilon$-greedy 策略选择动作 $a_{t+1}$

8: 利用公式更新值函数 $Q(s_t, a_t)$

9: 更新探索率 $\epsilon$

10: **end for**

11: **end for**

**收敛性：** 由于 SARSA 是 TD 方法的一种形式，因此 SARSA 具有和 TD 方法相同的收敛性。

### 时序差分方法 - 同策略与异策略

$$Q(S_t, A_t) \leftarrow Q(S_t, A_t) + \alpha(R_{t+1} + \gamma Q(S_{t+1}, A_{t+1}) - Q(S_t, A_t))$$

**同策略 (on-policy)**

- 边采样边学习
- 行为策略与目标策略**一致**

[截图待插入: 同策略示意图 - 运动员在球场上边打球边学习]

**异策略 (off-policy)**

- 行为策略与目标策略**不一致**
- 利用行为策略 $\mu(a|s)$ 采集经验 $S_1, A_1, R_2, \ldots, S_T \sim \mu$
- 随后利用经验 $S_1, A_1, R_2, \ldots, S_T$ 更新目标策略 $\pi$

**优势：**

- 保持充分探索环境，并同步学习最优策略
- 可以通过观测其他智能体的行为进行学习
- 重用旧策略 $\pi_1, \pi_2, \ldots, \pi_{t-1}$ 的经验

**考虑动作价值函数 $Q(s, a)$ 的异策略学习**

- 根据贝尔曼方程，我们可以得到如下公式：

$$Q_\pi(S_t, A_t) = \mathbb{E}_\pi[R_{t+1} + \gamma Q(S_{t+1}, A_{t+1})]$$

其中 $R_{t+1}$ 为即时奖励，$\gamma$ 为折扣因子，$S_{t+1}$ 为下一状态，$A_{t+1}$ 为策略 $\pi$ 在状态 $S_{t+1}$ 下所要采取的动作。

- 如果我们令策略 $\pi$ 为依据动作价值函数 $Q(s, a)$ 的贪婪策略，那么有

$$\pi(s) = \arg\max_a Q(s, a)$$

$$Q_\pi(S_t, A_t) = \mathbb{E}_\pi[R_{t+1} + \gamma \max_{a'} Q(S_{t+1}, a')]$$

当给定 $S_t$ 以及 $A_t$ 后，$R_{t+1}$ 与 $S_{t+1}$ 仅由环境决定，而与所采取的策略无关。因此，当我们通过某个策略采集到样本 $<S_t, A_t, R_{t+1}, S_{t+1}>$ 后，可直接以 $R_{t+1} + \gamma \max_{a'} Q(S_{t+1}, a')$ 作为目标更新贪婪策略 $\pi$ 的动作价值函数 $Q_\pi(S_t, A_t)$，而无需进行重要性采样。

$$Q(S_t, A_t) \leftarrow Q(S_t, A_t) + \alpha(R_{t+1} + \gamma \max_{a'} Q(S_{t+1}, a') - Q(S_t, A_t))$$

### 时序差分方法 - Q Learning (续)

$$Q(S_t, A_t) \leftarrow Q(S_t, A_t) + \alpha(R_{t+1} + \gamma \max_a Q(S_{t+1}, a) - Q(S_t, A_t))$$

**Algorithm 1.7 Q-Learning**

**Input:** 迭代轮数 T，更新步长 $\alpha$，衰减因子 $\gamma$，最大探索率 $\epsilon_{start}$，最小探索率 $\epsilon_{min}$，最大交互回合数 $N$，最大时间步长 $T$

**Output:** 最优值函数 $Q_*$

1: 随机初始化所有的状态动作对应的值函数 $Q$。

2: **for** 每个交互回合 $k = 0, 1, \cdots, N$ **do**

3: 初始化初始状态 $s_0$

4: **for** $t = 0, 1, \cdots, T$ **do**

5: 在状态 $s_t$ 利用 $\epsilon$-greedy 策略选择动作 $a_t$

6: 在状态 $s_t$ 执行当前动作 $a_t$，得到新状态 $s_{t+1}$ 和奖励 $r_t$

7: 利用公式更新值函数 $Q(s_t, a_t)$

8: 更新探索率 $\epsilon$

9: **end for**

10: **end for**

在实际应用时，通常使用 $\epsilon$-贪婪策略与环境交互，以充分探索环境。

### 深度强化学习

然而，表格型强化学习方法面临着大量的问题：

- **扩展性差：** 状态和动作空间较大时，表格的大小会急剧增长（"维度灾难"），导致存储和计算成本过高，难以应用于高维或连续空间。
- **对复杂环境不适用：** 表格型方法无法直接处理复杂的、具有连续状态或动作空间的问题，例如机器人控制和自动驾驶中的应用场景。
- **缺乏泛化能力：** 表格型方法对未探索到的状态无法进行泛化处理，表现为对未见状态完全缺乏经验，导致策略表现差。
- **存储开销大：** 随着状态-动作空间增大，存储需求成指数增长，资源消耗显著增加。
- **采样效率低：** 在高维空间中，穷举式的探索会导致算法采样效率极低，难以收敛。

### 基于值的RL：Naïve DQN

Q-Learning 公式：

$$Q_\pi(S_t, A_t) = \mathbb{E}_\pi\left[R_{t+1} + \gamma \max_{a'} Q_\pi(S_{t+1}, a')\right]$$

- **基于值函数的深度强化学习算法**是由 Q-Learning 算法结合深度神经网络衍生的一类算法，主要包括 DQN 算法及其改进算法 Double DQN、Dueling DQN、Rainbow DQN。
- Q-Learning 算法直接结合神经网络的算法称为 **Naïve DQN**。（由于策略由神经网络参数 $w$ 决定，因此 $Q_\pi$ 一般写成 $Q_w$）

#### Naïve DQN

通过拟合数据集得到最优动作价值函数：

- 使用某个策略采样数据集 $\{(s_1, a_1, s_1', r_1), \ldots, (s_n, a_n, s_n', r_n)\}$
- $y_i = r_i + \gamma \max_{a'} Q_w(s_i', a')$
- $w \leftarrow \arg\min_w \frac{1}{2} \sum_{i=1}^{n} \|Q_w(s_i, a_i) - y_i\|^2$

误差 $\varepsilon$：

$$\varepsilon = \frac{1}{2} \mathbb{E}_{(s,a,s',r)\sim B}\left[\left(Q_w(s,a) - \left[r + \gamma \max_{a'} Q_w(s',a')\right]\right)^2\right]$$

当 $\varepsilon = 0$ 时，有：

$$Q_w(s,a) = r + \gamma \max_{a'} Q_w(s', a')$$

此时为最优动作价值函数 $Q_w(s,a) = Q_*(s,a)$，可由此求得最优策略 $\pi_*$：

$$\pi_*(a|s) = \begin{cases} 1 & \text{if } a = \arg\max_{a \in A} Q_*(s,a) \\ 0 & \text{otherwise} \end{cases}$$

- $w \leftarrow w - \alpha(Q_w(s_i, a_i) - y_i)\nabla_w Q(s_i, a_i)$

策略为：

$$\pi(a|s) = \begin{cases} 1 & \text{if } a = \arg\max_{a \in A} Q_w(s,a) \\ 0 & \text{otherwise} \end{cases}$$

### 使用经验回放池打破样本相关性

#### 基于经验回放池的 Q-learning

- 与环境交互获取数据 $\{(s, a, s', r)\}$，并存入缓存 $B$
- 从缓存 $B$ 中采样一批数据 $\{(s_1, a_1, s_1', r_1), \ldots, (s_n, a_n, s_n', r_n)\}$
- $w \leftarrow w - \sum_{i=1}^{n} \alpha(Q_w(s', a') - y)\nabla_w Q(s, a)$

**优点**：样本之间不再具有较高的关联性；批量中样本能够降低梯度的方差。

### 基于策略的强化学习算法

#### 基本思想

- 利用**目标函数**定义策略优劣性：$J(\theta) = J(\pi_\theta)$
- 对**目标函数**进行优化，以寻找最优策略

- **目标函数** $J(\theta)$ 如何设计？

### 基于策略的强化学习算法——梯度优化

#### 策略梯度计算

- 当**目标函数可微分**时，可以使用一些基于梯度的算法：
  - 梯度下降法、拟牛顿法等
- 但在此之前，需要先计算出目标函数关于策略参数的梯度。

[截图待插入: 策略梯度计算流程图 - 状态s → 神经网络 π_θ(a|s) → 动作a → 环境 p(s'|s,a) → 状态完全可观测]

#### 最大化平均轨迹回报目标

$$\max_\theta J(\theta) = \max_\theta \mathbb{E}_{\tau \sim p_\theta(\tau)}\left[\sum_t r(s_t, a_t)\right]$$

- $\tau$ 为策略 $\pi_\theta$ 采样而来的轨迹 $\{s_1, a_1, r_1, s_2, a_2, r_2, \ldots, s_T\}$

$$\theta^* = \arg\max_\theta \mathbb{E}_{\tau \sim p_\theta(\tau)}\left[\sum_t r(s_t, a_t)\right]$$

目标函数 $J(\theta)$

轨迹概率：

$$p_\theta(s_1, a_1, \ldots, s_T) = p(s_1) \prod_{t=1}^{T-1} \pi_\theta(a_t|s_t) p(s_{t+1}|s_t, a_t)$$

### 策略梯度计算（续）

记 $G(\tau) = \sum_t r(s_t, a_t)$，轨迹回报目标的策略梯度为：

$$\nabla_\theta J(\theta) = \nabla_\theta \int p_\theta(\tau) G(\tau) d\tau = \mathbb{E}_{\tau \sim p_\theta(\tau)}[\nabla_\theta \log p_\theta(\tau) \, G(\tau)]$$

利用恒等式：

$$\nabla_\theta \log p_\theta(\tau) = \frac{1}{p_\theta(\tau)} \cdot \nabla_\theta p_\theta(\tau)$$

#### 关于策略参数梯度的推导：

$$\nabla_\theta J(\theta) = \int \nabla_\theta p_\theta(\tau) G(\tau) d\tau$$

$$= \int \frac{p_\theta(\tau)}{p_\theta(\tau)} \nabla_\theta p_\theta(\tau) \, G(\tau) d\tau$$

$$= \int p_\theta(\tau) \frac{\nabla_\theta p_\theta(\tau)}{p_\theta(\tau)} G(\tau) d\tau$$

$$= \int p_\theta(\tau) \nabla_\theta \log p_\theta(\tau) \, G(\tau) d\tau$$

$$= \mathbb{E}_{\tau \sim p_\theta(\tau)}[\nabla_\theta \log p_\theta(\tau) \, G(\tau)]$$

轨迹的概率：$p_\theta(s_1, a_1, \ldots, s_T) = p(s_1) \prod_{t=1}^{T-1} \pi_\theta(a_t|s_t) p(s_{t+1}|s_t, a_t)$

### 策略梯度计算（续）

$$\nabla_\theta J(\theta) = \nabla_\theta \int p_\theta(\tau) G(\tau) d\tau = \mathbb{E}_{\tau \sim p_\theta(\tau)}[\nabla_\theta \log p_\theta(\tau) \, G(\tau)] = \mathbb{E}_{\tau \sim p_\theta(\tau)}\left[\sum_{t=1}^{T} \nabla_\theta \log \pi_\theta(a_t|s_t) \, G(\tau)\right]$$

$$p_\theta(s_1, a_1, \ldots, s_T) = p(s_1) \prod_{t=1}^{T-1} \pi_\theta(a_t|s_t) p(s_{t+1}|s_t, a_t)$$

#### 关于 $\nabla_\theta \log p_\theta(\tau)$ 的简化：

$$\nabla_\theta \log p_\theta(\tau)$$

$$= \nabla_\theta\left(\log p(s_1) + \sum_{t=1}^{T} \log \pi_\theta(a_t|s_t) + \sum_{t=1}^{T} \log p(s_{t+1}|s_t, a_t)\right)$$

$$= \nabla_\theta \log p(s_1) + \nabla_\theta \sum_{t=1}^{T} \log \pi_\theta(a_t|s_t) + \nabla_\theta \sum_{t=1}^{T} \log p(s_{t+1}|s_t, a_t)$$

$$= \nabla_\theta \sum_{t=1}^{T} \log \pi_\theta(a_t|s_t)$$

$$= \sum_{t=1}^{T} \nabla_\theta \log \pi_\theta(a_t|s_t)$$

其中 $p(s_1)$，$p(s_{t+1}|s_t, a_t)$ 与参数 $\theta$ 无关，因此 $\nabla_\theta \log p(s_1) = 0$，$\nabla_\theta \sum_{t=1}^{T} \log p(s_{t+1}|s_t, a_t) = 0$

### REINFORCE 算法

#### 基于蒙特卡洛方法计算策略梯度

前面我们推导出了策略梯度：

$$\nabla_\theta J(\theta) = \mathbb{E}_{\tau \sim p_\theta(\tau)}\left[\sum_{t=1}^{T} \nabla_\theta \log \pi_\theta(a_t|s_t) \, G(\tau)\right]$$

在实践中，我们可以通过**蒙特卡洛方法**进行估计：

$$\nabla_\theta J(\theta) = \frac{1}{N} \sum_{n=1}^{N} \sum_{t=1}^{T^n} G(\tau^n) \nabla_\theta \log \pi_\theta(a_t^n|s_t^n)$$

据此，我们可以得到 REINFORCE 算法。

[截图待插入: REINFORCE算法循环图 - 数据采样（对于策略π_θ采样轨迹τ，计算G(τ)）→ 更新模型 θ ← θ + η∇R̄_θ → 数据采样，其中 ∇R̄_θ = (1/N) Σ Σ G(τ^n)∇log p_θ(a_t^n|s_t^n)，仅使用一次]

#### REINFORCE 算法：

1. 利用策略 $\pi_\theta(a|s)$ 采样轨迹 $\{\tau_i\}$
2. 计算梯度 $\nabla_\theta J(\theta) = \frac{1}{N} \sum_{n=1}^{N} \sum_{t=1}^{T_n} G(\tau^n) \nabla_\theta \log \pi_\theta(a_t^n|s_t^n)$
3. 更新参数 $\theta \leftarrow \theta + \alpha \nabla_\theta J(\theta)$

### 基于Actor-Critic的RL

- **基于值的强化学习算法**（如 DQN 算法）难以直接应用于**高维离散动作空间**、**连续动作空间**、**离散连续混合的动作空间**。这些场景中，动作空间的维度可能极高甚至是无限大，因此直接枚举或计算每个动作的值函数所带来的计算成本十分高昂，同时在连续动作空间中也无法实现枚举所有可能的动作。

- **基于策略的强化学习算法（Policy-based）** 使用神经网络直接拟合策略函数。策略网络用于在给定状态 $s$ 下生成对应的策略 $\pi(a|s)$。**对于离散动作空间的场景**，策略网络通常输出所有离散动作的概率分布，通过概率采样得到最终和环境交互的动作。**对于连续动作空间的场景**，策略网络通常输出一个多维高斯分布的均值和方差，通过在高斯分布中采样得到最终和环境交互的动作。

- **基于 Actor-Critic 的强化学习算法（或者称为基于演员-评论家算法）** 使用两个不同的网络，一个**策略网络（Actor）** 和一个**值网络（Critic）**，分别用于**策略生成**和**策略评估**。

---

### 基于值 vs 基于策略的对比

| 基于值的强化学习算法 | 基于策略的强化学习算法 |
|---|---|
| 学习值函数 | 不需要值函数 |
| 利用值函数导出策略 | 直接学习策略 |
| 更高的样本训练效率 | 在高维或连续动作空间场景中更加高效 |
| 通常仅适用于具有离散动作的环境 | 适用任何动作类型的场景 |
| | 容易收敛到次优解 |

**基于Actor-Critic的强化学习算法**：结合两者优势

### REINFORCE 算法主循环

1. 使用当前策略 $\pi_\theta$ 采样 $N$ 条经验轨迹 $\{\tau_i\}$
2. 计算梯度 $\nabla_\theta J(\theta) \approx \frac{1}{N} \sum_i \left(\sum_{t=1}^{T} \nabla_\theta \log \pi_\theta(a_t^i|s_t^i)\left(\sum_{t'=t}^{T} r(s_{t'}^i, a_{t'}^i)\right)\right)$
   - 对第 $t$ 步做对数极大似然估计
   - 对 $N$ 条轨迹进行平均
   - 在每一条轨迹内累计回报
3. 更新策略参数 $\theta \leftarrow \theta + \alpha \nabla_\theta J(\theta)$

### 使用动作价值估计改进策略梯度

$$\nabla_\theta J(\theta) \approx \frac{1}{N} \sum_{i=1}^{N}\left(\sum_{t=1}^{T} \nabla_\theta \log \pi_\theta(a_t^i|s_t^i) \underbrace{\sum_{t'=t}^{T} r(s_{t'}^i, a_{t'}^i)}_{\text{方差较大}}\right)$$

↓ 改进

使用动作价值估计 $\widehat{Q}^\pi$ 近似轨迹回报期望 $\mathbb{E}_{\pi_\theta} \sum_{t'=t}^{T} r(s_{t'}^i, a_{t'}^i)$：

$$\nabla_\theta J(\theta) \approx \frac{1}{N} \sum_{i=1}^{N}\left(\sum_{t=1}^{T} \nabla_\theta \log \pi_\theta(a_t^i|s_t^i) \widehat{Q}^\pi(s_t^i, a_t^i)\right)$$

$$\widehat{Q}^\pi(s_t^i, a_t^i) \approx \sum_{t'=t}^{T} \mathbb{E}_{\pi_\theta}[r(s_{t'}, a_{t'})|s_t, a_t]$$

### Actor-Critic 算法

- **Actor**: $\pi_\theta$
- **Critic**: $\widehat{Q}^\pi$

[截图待插入: Actor-Critic示意图 - Actor说"向左移动"，Critic说"价值很低"]

1. 使用当前策略 $\pi_\theta$ 在环境中进行采样
2. 策略提升：$\theta \leftarrow \theta + \alpha \nabla_\theta J(\theta) \approx \frac{1}{N} \sum_{i=1}^{N}\left(\sum_{t=1}^{T} \nabla_\theta \log \pi_\theta(a_t^i|s_t^i) \widehat{Q}^\pi(s_t^i, a_t^i)\right)$
   - 对第 $t$ 步做对数极大似然估计
   - 对 $N$ 条轨迹进行平均
   - 动作值函数估计（Critic）→ **Actor Learning**
3. 拟合当前策略的动作值函数：$\widehat{Q}^\pi(s_{t'}^i, a_{t'}^i) \approx \sum_{t'=t}^{T} r(s_{t'}^i, a_{t'}^i)$ → **Critic Learning**

### Actor-Critic算法的策略梯度

$$\nabla_\theta J(\theta) \approx \frac{1}{N} \sum_{i=1}^{N}\left(\sum_{t=1}^{T} \nabla_\theta \log \pi_\theta(a_t^i|s_t^i) \widehat{Q}^\pi(s_t^i, a_t^i)\right) \quad (*)$$

#### 进一步对策略梯度进行改进

$$\nabla_\theta J(\theta) \approx \frac{1}{N} \sum_{i=1}^{N}\left(\sum_{t=1}^{T} \nabla_\theta \log \pi_\theta(a_t^i|s_t^i) \widehat{A}^\pi(s_t^i, a_t^i)\right) \quad (**)$$

**Advantage Actor-Critic (A2C) 算法**：减去基线函数 $\widehat{V}^\pi(s_t^i)$

"优势函数"：$\widehat{A}^\pi(s_t^i, a_t^i) = \widehat{Q}^\pi(s_t^i, a_t^i) - \widehat{V}^\pi(s_t^i)$

---

### 关于优势函数 $\widehat{A}^\pi(s_t^i, a_t^i)$ 的说明

- **含义**：衡量当前动作 $a_t^i$ 相对于平均情况 $\widehat{V}^\pi(s_t^i)$ 的好坏程度
- **等价性**：用 $\widehat{A}^\pi$ 代替 $\widehat{Q}^\pi$ 后策略梯度**期望值不变**，证明：
- **作用**：对**单个动作**的梯度有正负之分，减小方差

#### 证明：

$$\nabla_\theta \log \pi_\theta(a_t^i|s_t^i) \widehat{Q}^\pi(s_t^i, a_t^i) = \nabla_\theta \log \pi_\theta(a_t^i|s_t^i)\left(\widehat{Q}^\pi(s_t^i, a_t^i) - \widehat{V}^\pi(s_t^i)\right) + \nabla_\theta \log \pi_\theta(a_t^i|s_t^i) \, \widehat{V}^\pi(s_t^i)$$

而 $\mathbb{E}_{a_t \sim \pi_\theta(a_t|s_t)}\left[\nabla_\theta \log \pi_\theta(a_t^i|s_t^i) \, \widehat{V}^\pi(s_t^i)\right] =$

$\sum_{a_t^i \in A} \pi_\theta(a_t^i|s_t^i) \nabla_\theta \log \pi_\theta(a_t^i|s_t^i) \, \widehat{V}^\pi(s_t^i) =$

$\sum_{a_t^i \in A} \nabla_\theta \pi_\theta(a_t^i|s_t^i) \, \widehat{V}^\pi(s_t^i) =$

$\widehat{V}^\pi(s_t^i) \nabla_\theta \sum_{a_t^i \in A} \pi_\theta(a_t^i|s_t^i) = \widehat{V}^\pi(s_t^i) \nabla_\theta 1 = 0$

可知使用 $\widehat{A}^\pi(s_t^i, a_t^i) = \widehat{Q}^\pi(s_t^i, a_t^i) - \widehat{V}^\pi(s_t^i)$ 代替 $\widehat{Q}^\pi(s_t^i, a_t^i)$ 不会影响策略梯度的期望值。

---

### 优势函数展开

$$\nabla_\theta J(\theta) \approx \frac{1}{N} \sum_{i=1}^{N}\left(\sum_{t=1}^{T} \nabla_\theta \log \pi_\theta(a_t^i|s_t^i) \widehat{A}^\pi(s_t^i, a_t^i)\right) \quad (**)$$

展开为：

$$\nabla_\theta J(\theta) \approx \frac{1}{N} \sum_{i=1}^{N}\left(\sum_{t=1}^{T} \nabla_\theta \log \pi_\theta(a_t^i|s_t^i)\left[\widehat{Q}^\pi(s_t^i, a_t^i) - \widehat{V}^\pi(s_t^i)\right]\right)$$

$$= \frac{1}{N} \sum_{i=1}^{N}\left(\sum_{t=1}^{T} \nabla_\theta \log \pi_\theta(a_t^i|s_t^i)\left[R(s_t^i, a_t^i) + \gamma \widehat{V}^\pi(s_{t+1}^i) - \widehat{V}^\pi(s_t^i)\right]\right)$$

只需用**一个**神经网络拟合 $\widehat{V}^\pi$！

### AC 算法

利用单个样本进行更新：更新方差较大，训练稳定性差。

**解决方案**：获得一个批次的数据后再进行更新。

[截图待插入: 同步更新示意图 - 采样数据(s,a,s',r) → 更新参数θ → 采样数据(s,a,s',r) → 更新参数θ，随时间t推进]

#### Algorithm 1.12 AC 算法

- **Input**: 迭代轮数 $T$，学习率 $\alpha$，衰减因子 $\gamma$，最大交互回合数 $N$，最大时间步长 $T$，存储队列 $B$
- **Output**: 最优策略 $\pi_*$

1. 随机初始化策略网络 $\pi_\theta$，值网络 $\phi$
2. **for** 每个交互回合 $k = 0, 1, \cdots, N$ **do**
3. 重置环境并获取环境的初始状态 $s_0$
4. **for** $t = 0, 1, \cdots, T$ **do**
5. 在状态 $s_t$，通过策略网络 $\pi_\theta$ 选择动作 $a_t$
6. 在状态 $s_t$ 执行当前动作 $a_t$，得到新状态 $s_{t+1}$ 和奖励 $r_t$
7. 计算 $\log(\pi_\theta(a_t|s_t))$
8. 将 $\log(\pi_\theta(a_t|s_t))$、$s_t$、$a_t$、$r_t$、$s_{t+1}$ 存入存储队列 $B$
9. **end for**
10. 从存储队列 $B$ 取出交互回合 $k$ 的轨迹数据
11. 计算每个状态的 $\Psi_t$
12. 利用策略更新公式和值函数更新公式分别更新 $\theta$ 和 $\phi$
13. 清空存储队列 $B$
14. **end for**
