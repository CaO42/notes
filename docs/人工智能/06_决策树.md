## 决策树

### Bayesian Classifiers 贝叶斯分类器

将每个属性和类别标签视为随机变量。例如给定一位学生人工智能、明清文学、古文字学、常微分方程几门课的成绩（attribute），判别他是计算机学院还是历史学院（class label）。

给定一个记录带有属性 $(A_1, A_2, \ldots, A_n)$：

- 目标是预测类别 C
- 具体地，我们想找到使 $P(C \mid A_1, A_2, \ldots, A_n)$ 最大化的 C 值

能否直接从数据中估计 $P(C \mid A_1, A_2, \ldots, A_n)$？

### 贝叶斯分类器方法

方法：

- 利用贝叶斯定理，对所有 C 值计算后验概率 $P(C \mid A_1, A_2, \ldots, A_n)$

$$P(C \mid A_1 A_2 \ldots A_n) = \frac{P(A_1 A_2 \ldots A_n \mid C) P(C)}{P(A_1 A_2 \ldots A_n)}$$

- 选择使 $P(C \mid A_1, A_2, \ldots, A_n)$ 最大化的 C 值
- 等价于选择使 $P(A_1, A_2, \ldots, A_n \mid C) \cdot P(C)$ 最大化的 C 值

如何估计 $P(A_1, A_2, \ldots, A_n \mid C)$？

### Naive Bayes Classifier 朴素贝叶斯分类器

假设在给定类别时属性 $A_i$ 之间相互独立：

- $P(A_1, A_2, \ldots, A_n \mid C_j) = P(A_1 \mid C_j) \cdot P(A_2 \mid C_j) \cdots P(A_n \mid C_j)$
- 可以估计所有 $A_i$ 和 $C_j$ 的 $P(A_i \mid C_j)$
- 新样本点被分类到 $C_j$，如果 $P(C_j) \cdot P(A_1 \mid C_j) \cdot P(A_2 \mid C_j) \cdots P(A_n \mid C_j)$ 取最大值

### 朴素贝叶斯基本原理

**问题设定：** 假设预测患者是否感染疾病 $C \in \{\text{感染}, \text{未感染}\}$，预测因子包括：

- $A_1$ 性别（男/女）
- $A_2$ 地区（高风险/低风险）
- $A_3$ 核酸检测结果（阳性/阴性）
- $A_4$ 体重（kg）

**训练数据：**

|------|---------|---------|----------------|-------------|---------------|
| 1    | 男      | 低风险   | 阴性            | 70          | 未感染         |
| 2    | 女      | 高风险   | 阳性            | 55          | 感染           |
| 3    | 男      | 高风险   | 阴性            | 64          | 未感染         |
| 4    | 男      | 低风险   | 阴性            | 85          | 未感染         |
| 5    | 女      | 低风险   | 阴性            | 50          | 未感染         |
| 6    | 女      | 低风险   | 阳性            | 62          | 感染           |
| 7    | 男      | 高风险   | 阳性            | 67          | 未感染         |
| 8    | 女      | 低风险   | 阴性            | 60          | 未感染         |
| 9    | 女      | 高风险   | 阴性            | 53          | 感染           |
| 10   | 男      | 低风险   | 阴性            | 74          | 未感染         |

**问题形式化：** 给定一个样本带有 $(A_1, A_2, \ldots, A_n)$ 等属性，我们要预测样本的类别 C，即：

> 找到 C 使得 $P(C \mid A_1, A_2, \ldots, A_n)$ 最大化

我们可以从数据中直接得到 $P(C \mid A_1, A_2, \ldots, A_n)$ 吗？

**解决步骤：**

1. 使用贝叶斯公式：

$$P(C \mid A_1 A_2 \ldots A_n) = \frac{P(A_1 A_2 \ldots A_n \mid C) P(C)}{P(A_1 A_2 \ldots A_n)}$$

2. 选择合适的数值 C 最大化 $P(C \mid A_1, A_2, \ldots, A_n)$，等价于：

$$\text{Max} \; P(A_1, A_2, \ldots, A_n \mid C) \cdot P(C)$$

$P(C)$ 可以从数据样本中得到，如何估计其中的 $P(A_1, A_2, \ldots, A_n \mid C)$？

### 朴素贝叶斯基本原理：独立性假设

如何估计其中的 $P(A_1, A_2, \ldots, A_n \mid C)$？

假设 $A_1, A_2, \ldots, A_n$ 在给定样本的类别 $C_j$ 时具有独立性，即：

$$P(A_1, A_2, \ldots, A_n \mid C_j) = P(A_1 \mid C_j) \cdot P(A_2 \mid C_j) \cdots P(A_n \mid C_j)$$

可通过统计所有 $C_j$ 样本中 $A_i$ 出现频率来估计。

因此，一个样本点划分到 $C_j$ 类别等价于：

$$\text{Max} \; P(A_1, A_2, \ldots, A_n \mid C_j) \cdot P(C_j) = \text{Max} \; P(A_1 \mid C_j) \cdot P(A_2 \mid C_j) \cdots P(A_n \mid C_j) \cdot P(C_j)$$

### 类别先验概率

$$P(C) = N_C / N$$

例如：$P(\text{未感染}) = 7/10$，$P(\text{感染}) = 3/10$

### 离散属性

$$P(A_i \mid C_k) = |A_{ik}| / N_C$$

其中 $|A_{ik}|$ 是具有属性 $A_i$ 且属于 $C_k$ 类别的比率。

例如：
- $P(A_1=\text{男} \mid \text{未感染}) = 5/7$
- $P(A_3=\text{阴性} \mid \text{感染}) = 1/3$

**问：如何处理连续数值？**

### 连续属性处理

**1. 离散化：将范围划分为区间**

- 大区间数量：训练记录太少，难以可靠估计
- 小区间数量：会将不同类别的记录合并在一起

**2. 概率密度估计：**

- 假设属性遵循正态分布
- 使用数据估计分布参数（例如，均值和标准差）
- 一旦概率分布已知，就可以用它来估计条件概率 $P(A_i \mid C_k)$

### 连续属性：正态分布概率密度

假设符合正态分布：

$$P(A_i \mid C_j) = \frac{1}{\sqrt{2\pi \sigma_{ij}^2}} e^{-\frac{(A_i - \mu_{ij})^2}{2\sigma_{ij}^2}}$$

比如，对于 $(A_4, C=\text{未感染})$，统计得到其均值与方差：

- 均值 mean = 67
- 方差 variance = 200

$$P(A_4 = 77 \mid \text{未感染}) = \frac{1}{\sqrt{2\pi \times 200}} e^{-\frac{(77-67)^2}{2 \times 200}} = 0.0022$$

### 练一练

给定一个数据样本，计算患病概率：

$$X = (\text{男性}, \text{低风险地区}, \text{阴性}, 77\text{kg})$$

**可事先计算：**

| 条件概率 | 值 |
|---------|-----|
| $P(A_1=\text{man} \mid \text{No})$ | $5/7$ |
| $P(A_1=\text{women} \mid \text{No})$ | $2/7$ |
| $P(A_1=\text{man} \mid \text{Yes})$ | $0$ |
| $P(A_1=\text{women} \mid \text{Yes})$ | $1$ |
| $P(A_2=\text{low} \mid \text{No})$ | $5/7$ |
| $P(A_2=\text{high} \mid \text{No})$ | $2/7$ |
| $P(A_2=\text{low} \mid \text{Yes})$ | $1/3$ |
| $P(A_2=\text{high} \mid \text{Yes})$ | $2/3$ |
| $P(A_3=\text{negative} \mid \text{No})$ | $6/7$ |
| $P(A_3=\text{positive} \mid \text{No})$ | $1/7$ |
| $P(A_3=\text{negative} \mid \text{Yes})$ | $1/3$ |
| $P(A_3=\text{positive} \mid \text{Yes})$ | $2/3$ |

**A4 连续属性：**
- If $C = \text{No}$: sample mean = 67, sample variance = 200
- If $C = \text{Yes}$: sample mean = 57, sample variance = 15

**计算过程：**

$$P(X \mid \text{未感染}) = P(A_1=\text{男} \mid \text{未感染}) \times P(A_2=\text{低风险} \mid \text{未感染}) \times P(A_3=\text{阴性} \mid \text{未感染}) \times P(A_4=77\text{kg} \mid \text{未感染})$$
$$= 5/7 \times 5/7 \times 6/7 \times 0.0022 = 0.001$$

$$P(X \mid \text{感染}) = P(A_1=\text{男} \mid \text{感染}) \times P(A_2=\text{低风险} \mid \text{感染}) \times P(A_3=\text{阴性} \mid \text{感染}) \times P(A_4=77\text{kg} \mid \text{感染})$$
$$= 0 \times 1/3 \times 1/3 \times 1.66 \times 10^{-7} = 0$$

由于 $P(X \mid \text{未感染}) \cdot P(\text{未感染}) > P(X \mid \text{感染}) \cdot P(\text{感染})$，因此样本 X 属于未感染类。

**注意：** $P(A_1=\text{man} \mid \text{Yes}) = 0$，导致整个概率为 0！

### 朴素贝叶斯平滑化

由于样本数量的缺失，部分条件概率会出现 0 值，此时可基于概率估计方法进行平滑化：

$$\text{Original:} \quad P(A_i \mid C) = \frac{N_{ic}}{N_C}$$

$$\text{Laplace:} \quad P(A_i \mid C) = \frac{N_{ic} + 1}{N_C + c}$$

$$\text{m-estimate:} \quad P(A_i \mid C) = \frac{N_{ic} + mp}{N_C + m}$$

其中：
- $c$：类别数量
- $p$：先验概率
- $m$：参数

平滑技术可以**避免模型过度依赖训练数据**，尤其在数据稀疏的情况下，通过给未见过的特征值分配适度的概率，**提升模型的鲁棒性**。

### 垃圾邮件分类

垃圾邮件困扰着互联网用户，正确识别技术难度大。

- **校验码法：** 计算邮件文本的效验码，与已知的垃圾邮件进行对比，但是识别效果不理想，且容易规避
- **Paul Graham 提出基于"贝叶斯推断"的检测方案：** 基于特定词语计算垃圾邮件概率

### 垃圾邮件分类：训练数据

- 假设预先提供两组已经识别好的邮件：正常邮件和垃圾邮件各 4000 封
- 解析所有的邮件，提取每一个词。计算每个词语在正常邮件和垃圾邮件中出现的频率。
- **prince** 在 4000 封垃圾邮件中，有 200 封包含，频率为 5%；4000 封正常邮件中，2 封包含，频率为 0.05%
- 而如果某个词只出现在垃圾邮件中，就假定它在正常邮件的出现频率是 1%（经验值，可调整），反之亦然，避免概率为 0。

### 垃圾邮件分类：单条件概率计算

收到一封新邮件，假定它是垃圾邮件的概率为 50%：

- S 表示垃圾邮件（spam），H 表示正常邮件（healthy）
- $P(S)$ 和 $P(H)$ 的先验概率，均为 50%

用 W 表示事件——包含 "prince" 这个词，问题变成如何计算 $P(S \mid W)$，即在某个词语 (W) 已经存在的情况下，垃圾邮件 (S) 的概率有多大：

$$P(S \mid W) = \frac{P(W \mid S) P(S)}{P(W \mid S) P(S) + P(W \mid H) P(H)} \approx 99.01\%$$

其中：
- $P(W \mid S) = 5\%$
- $P(W \mid H) = 0.05\%$
- $P(S) = 50\%$
- $P(H) = 50\%$

### 垃圾邮件分类：多词联合概率

能否得出结论，这封新邮件是垃圾邮件？

- 不能，一封邮件包含很多词语，一些词语（比如 prince）说是垃圾邮件，另一些（比如 computer science）说不是，以哪个词为准？

选出这封邮件中 $P(S \mid W)$ 最高的 15 个词，计算它们的**联合概率**。

先考虑只有 $W_1$ 和 $W_2$ 两个词语的情况，它们都出现在某封电子邮件中，这封邮件是垃圾邮件的概率，就是联合概率。

| 事件 | $W_1$ | $W_2$ | 垃圾邮件 |
|------|-------|-------|---------|
| $E_1$ | 出现  | 出现  | 是的     |
| $E_2$ | 出现  | 出现  | 不是     |

### 垃圾邮件分类：联合概率公式

假定所有事件都是**独立事件**（严格来说，这个假定不成立，但是这里先忽略）：

$$P(E_1) = P(S \mid W_1) P(S \mid W_2) P(S)$$

$$P = \frac{P(E_1)}{P(E_1) + P(E_2)}$$

$$P = \frac{P(S \mid W_1) P(S \mid W_2) P(S)}{P(S \mid W_1) P(S \mid W_2) P(S) + (1 - P(S \mid W_1))(1 - P(S \mid W_2))(1 - P(S))}$$

将 $P(S)$ 等于 0.5 代入，得到：

$$P = \frac{P(S \mid W_1) P(S \mid W_2)}{P(S \mid W_1) P(S \mid W_2) + (1 - P(S \mid W_1))(1 - P(S \mid W_2))}$$

$$= \frac{P_1 P_2}{P_1 P_2 + (1 - P_1)(1 - P_2)}$$

### 垃圾邮件分类：15 词联合

推广到 15 个单词：

$$P = \frac{P_1 P_2 \cdots P_{15}}{P_1 P_2 \cdots P_{15} + (1 - P_1)(1 - P_2) \cdots (1 - P_{15})}$$

这就是 2002 年，Paul Graham 提出使用"贝叶斯推断"过滤垃圾邮件方法。

- Paul Graham 的门槛值是 0.9，概率大于 0.9，表示 15 个词联合认定，这封邮件有 90% 以上的可能属于垃圾邮件；概率小于 0.9，就表示是正常邮件
- 有了这个方法以后，1000 封垃圾邮件可以过滤掉 995 封，没有误判

### Naive Bayes 朴素贝叶斯特点

- **Robust to isolated noise points / irrelevant attributes**：根据后验概率判断，对数据噪音有天然的鲁棒性
- **Sensitive to prior probability**：（垃圾邮件例子中的 $P(S)$ 对最终结果影响大）
  - Noninformative prior
- **Independence assumption may not hold for some attributes**：（独立性条件有时候并不成立，如邮件中有些单词一起出现概率大，prince 与 account）
  - Use other techniques such as Bayesian Belief Networks (BBN) 贝叶斯网络或信念网络...

### 独立性带给我们什么？

- Suppose Boolean variables $X_1, X_2, \ldots, X_n$ are mutually independent (i.e., every subset is variable independent of every other subset)
- We can specify full joint distribution (probability function over all vectors of values) using only $n$ parameters (linear) instead of $2^n - 1$ (exponential)
  - **只使用 n 个参数（线性）来计算完整的联合分布**
- Simply specify $Pr(X_1 = \text{true}), \ldots, Pr(X_n = \text{true})$ (i.e., $Pr(X_i = \text{true})$ for all $i$)
- We can easily recover probability of any primitive event, e.g.
- $Pr(X_1 \neg X_2 X_3 X_4) = Pr(X_1)(1 - Pr(X_2))Pr(X_3)Pr(X_4)$

### The Value of Independence 独立性的价值

- Complete independence reduces both representation and inference from $O(2^n)$ to $O(n)$!
- Unfortunately, such complete mutual independence is rare. **Most realistic domains do not exhibit this property.** 完全的相互独立性不存在了！
- Fortunately, most domains do exhibit a fair amount of conditional independence. **而条件独立性时常有！**
- And we can exploit conditional independence for representation and inference as well.
- **Bayesian networks do just this**

### Exploiting Conditional Independence

想象这么一个故事：

- If Craig woke up too early ($E$), Craig probably needs coffee ($C$);
- if $C$, he's likely angry ($A$).
- If $A$, there is an increased chance of a burst blood vessel ($B$).
- If $B$, Craig is quite likely to be hospitalized ($H$).

```mermaid
graph LR
    E[E: Craig woke too early] --> C[C: Craig needs coffee]
    C --> A[A: Craig is angry]
    A --> B[B: Craig burst a blood vessel]
    B --> H[H: Craig hospitalized]
```

- E - Craig woke too early
- C - Craig needs coffee
- A - Craig is angry
- B - Craig burst a blood vessel
- H - Craig hospitalized

### Exploiting Conditional Independence：条件独立性分析

如果了解了 E、C、A 或 B 中任何一个，你对 $Pr(H)$ 的评估会改变。对 E、C、A、B 信息的掌握影响对 $Pr(H)$ 的评估。

- e.g., if any of these are seen to be true, you would increase $Pr(h)$ and decrease $Pr(\neg h)$.
- So H is **not independent** of E, or C, or A, or B.

But if you knew value of B (true or false), learning value of E, C, or A, would not influence $Pr(H)$. Influence these factors have on H is mediated by their influence on B.

- Craig doesn't get sent to the hospital because he's angry, he gets sent because he's had a burst blood vessel.
- So H is **independent** of E, and C, and A, **given B**

### Exploiting Conditional Independence：进一步推导

Similarly:
- B is **independent** of E, and C, **given A**
- A is **independent** of E, **given C**

This means that:
- $Pr(H \mid B, \{A, C, E\}) = Pr(H \mid B)$
- $Pr(B \mid A, \{C, E\}) = Pr(B \mid A)$
- $Pr(A \mid C, \{E\}) = Pr(A \mid C)$
- $Pr(C \mid E)$ and $Pr(E)$ don't simplify


### 利用条件独立性 (Exploiting Conditional Independence)

考虑链式贝叶斯网络：E → C → A → B → H

根据链式法则（对 H...E 的任意实例化）：

$$P(H,B,A,C,E) = P(H|B,A,C,E) \cdot P(B|A,C,E) \cdot P(A|C,E) \cdot P(C|E) \cdot P(E)$$

完全联合分布需要 $2^4 + 2^3 + 2^2 + 2^1 + 2^0 = 2^5 - 1 = 31$ 个参数。

根据条件独立性假设，可简化为：

$$P(H,B,A,C,E) = P(H|B) \cdot P(B|A) \cdot P(A|C) \cdot P(C|E) \cdot P(E)$$

只需指定五个**局部条件分布**：$P(H|B)$, $P(B|A)$, $P(A|C)$, $P(C|E)$, $P(E)$。

### 量化示例 (Example Quantification)

链式网络 E → C → A → B → H 的条件概率表：

| 条件概率 | 值 |
|---------|-----|
| Pr(c\|e) | 0.9 |
| Pr(~c\|e) | 0.1 |
| Pr(c\|~e) | 0.5 |
| Pr(~c\|~e) | 0.5 |
| Pr(e) | 0.7 |
| Pr(~e) | 0.3 |
| Pr(a\|c) | 0.3 |
| Pr(~a\|c) | 0.7 |
| Pr(a\|~c) | 1.0 |
| Pr(~a\|~c) | 0.0 |
| Pr(b\|a) | 0.2 |
| Pr(~b\|a) | 0.8 |
| Pr(b\|~a) | 0.1 |
| Pr(~b\|~a) | 0.9 |
| Pr(h\|b) | 0.9 |
| Pr(~h\|b) | 0.1 |
| Pr(h\|~b) | 0.1 |
| Pr(~h\|~b) | 0.9 |

注意其中一半是另一半的 "1 减去" 值。

因此，指定完整联合分布仅需 $9(1+2+2+2+2)$ 个参数，而非显式表示所需的 31 个参数：

- 变量数量呈**线性**而非指数增长！
- 如果依赖关系具有链式结构，通常也是线性的。

### 推理很简单 (Inference is Easy)

想知道 P(a)？使用消去求和规则：

$$\Pr(a) = \sum_{c_i \in Dom(C)} \Pr(a \mid c_i) \Pr(c_i)$$

$$= \sum_{c_i \in Dom(C)} \Pr(a \mid c_i) \sum_{e_i \in Dom(E)} \Pr(c_i \mid e_i) \Pr(e_i)$$

这些项全部由局部条件分布指定。

**具体计算示例：**

$$P(C) = P(C|E)P(E) + P(C|\sim E)P(\sim E) = 0.9 \times 0.7 + 0.5 \times 0.3 = 0.78$$

$$P(\sim C) = 1 - P(C) = 0.22$$

$$P(A) = P(A|C)P(C) + P(A|\sim C)P(\sim C) = 0.7 \times 0.78 + 0.0 \times 0.22 = 0.54$$

$$P(\sim A) = 1 - P(A) = 0.454$$

- 贝叶斯网络是一组变量上直接依赖关系的**图形表示**，以及一组**条件概率表 (CPTs)** 来量化这些影响的强度。

贝叶斯网络以非常有趣的方式推广了上述思想，从而在不确定性下提供了有效的表示和推理手段。

- **一图**：有向无环图
- **一表**：条件概率表

### 贝叶斯网络定义

变量 $\{X_1, X_2, \ldots, X_n\}$ 上的贝叶斯网络由以下组成：

- 一个 **DAG**（有向无环图），其节点为变量
- 一组 **CPTs**（条件概率表）：$Pr(X_i | Par(X_i))$，对每个 $X_i$

**关键概念：**

- 节点的 **parents**：$Par(X_i)$
- 节点的 **children**
- 节点的 **descendents**
- 节点的 **ancestors**
- **family**：由 $X_i$ 及其 parents 组成的节点集合

### 贝叶斯网络（中文定义）

- 贝叶斯网络（Bayesian Network，简称BN）是不确定知识表示与推理的一种**有效方法**，它由一个**有向无环图**（Directed Acyclic Graph, **DAG**）和一系列**条件概率表**（衡量了上述关系的强度）组成。

> **有向无环图**指的是一个无回路的有向图，即从图中任意一个节点出发经过任意条边，均无法回到该节点。有向无环图刻画了图中所有节点之间的依赖关系。

- 1986年，Judea Pearl提出贝叶斯网络，用于描述不确定知识表示与推理问题。贝叶斯网络让机器可以回答问题——给出一个从非洲回来的发烧且身体疼痛的病人，最有可能的解释是疟疾。

令 $G$ 为定义在 $\{X_1, X_2, \ldots, X_M\}$ 上的一个贝叶斯网络，其联合概率分布可以表示为各个节点的条件概率分布的乘积：

$$p(X) = \prod_i p_i(X_i | Par_G(X_i))$$

### 贝叶斯网络的语义 (Semantics of Bayes Nets)

贝叶斯网络指定了网络中变量的联合分布可以写成如下乘积分解：

$$Pr(X_1, X_2, \ldots, X_n) = Pr(X_n | Par(X_n)) \cdot Pr(X_{n-1} | Par(X_{n-1})) \cdot \ldots \cdot Pr(X_1 | Par(X_1))$$

该等式对变量 $X_1, X_2, \ldots, X_n$ 的任意取值 $d_1, d_2, \ldots, d_n$ 都成立。

例如，$X_1, X_2, X_3$ 的定义域 $Dom[X_i] = \{a, b, c\}$：

$$Pr(X_1, X_2, X_3) = P(X_3 | X_1, X_2) P(X_2) P(X_1)$$

则：

$$Pr(X_1 = a, X_2 = b, X_3 = c) = P(X_3 = c | X_1 = a, X_2 = b) P(X_2 = b) P(X_1 = a)$$

### 从贝叶斯网络推导概率

**问题：P(T) = ?**

$$P(T) = P(T|L)P(L) + P(T|\sim L)P(\sim L)$$

其中：

$$P(L) = P(L|A,S)P(A)P(S) + P(L|\sim A,S)P(\sim A)P(S) + P(L|A,\sim S)P(A)P(\sim S) + P(L|\sim A,\sim S)P(\sim A)P(\sim S)$$

计算得 $P(L) = 0.1275$，$P(\sim L) = 1 - 0.1275 = 0.8725$

$$P(T) = (0.3 \times 0.1275) + (0.8 \times 0.8725) = 0.03825 + 0.698 = 0.73625$$

### 贝叶斯网络的构建

给定变量 $\{X_1, \ldots, X_n\}$，构建贝叶斯网络的步骤：

**步骤1：** 在某种变量顺序下，对所有变量的联合概率应用链式法则：

$$Pr(X_1, \ldots, X_n) = Pr(X_n | X_1, \ldots, X_{n-1}) Pr(X_{n-1} | X_1, \ldots, X_{n-2}) \ldots Pr(X_1)$$

**步骤2：** 对于每个变量 $X_i$，考虑该变量的条件集合 $X_1, \ldots, X_{i-1}$，采用如下方法递归地判断条件集合中的每个变量 $X_j$ 是否可以删除：如果给定其余变量的集合，$X_i$ 和 $X_j$ 是条件独立的，则将 $X_j$ 从 $X_i$ 的条件集合中删除。经过这一步骤，可以得到：

$$Pr(X_1, \ldots, X_n) = Pr(X_n | Par(X_n)) Pr(X_{n-1} | Par(X_{n-1})) \ldots Pr(X_1)$$

**步骤3：** 基于上述公式，构建一个有向无环图。其中，对于每个用节点表示的变量 $X_i$，其父节点为 $Par(X_i)$ 中的变量集合。

**步骤4：** 为每个家庭（即变量及其父节点集合）确定条件概率表的取值。

### 变量顺序的重要性 — 因果直觉 (Variable Ordering Matters - Causal Intuitions)

- 贝叶斯网络可以使用变量的任意顺序构建。**变量序列**
- 然而，某些顺序会产生父节点集合很大的贝叶斯网络，这需要指数级空间，以及（稍后将看到）指数级时间来进行推理。
- 经验和概念上，构建贝叶斯网络的好方法是使用基于因果关系的顺序。这通常会产生更自然、更紧凑的贝叶斯网络。**基于因果关系的变量序列可以获得更加自然紧致的贝叶斯网络结构。**

### 因果直觉 (Causal Intuitions)

疟疾、流感和感冒都会引起疼痛。因此使用原因先于结果的顺序：Malaria, Flu, Cold, Aches。

$$Pr(M, F, C, A) = Pr(A|M, F, C) Pr(C|M, F) Pr(F|M) Pr(M)$$

每种疾病都影响疼痛的概率，因此第一个条件概率不变。

合理假设这些疾病相互独立：患或不患一种疾病不会改变患其他疾病的概率。

所以 $Pr(C|M, F) = Pr(C)$，$Pr(F|M) = Pr(F)$

这产生了一个相当简单的贝叶斯网络，只需要一个涉及 "Aches" 家族的大 CPT。

网络结构：Malaria → Aches, Flu → Aches, Cold → Aches

### 因果直觉（反向顺序的问题）

假设使用相反的顺序构建贝叶斯网络：Aches, Cold, Flu, Malaria

$$Pr(A, C, F, M) = Pr(M|A, C, F) Pr(F|A, C) Pr(C|A) Pr(A)$$

**能否简化 $Pr(M|A, C, F)$？不能。**

- 知道疼痛明显影响患疟疾的概率。
- 知道疼痛和感冒，或疼痛、感冒和流感呢？
- 患疟疾的概率受这些额外知识的影响。
- 知道感冒和流感会降低疼痛的概率，从而表明是疟疾，因为它们 "explain away"（解释掉）了疼痛！

类似地，**不能**简化 $Pr(F|A, C)$。

$Pr(C|A) \neq Pr(C)$

反向顺序无法利用条件独立性来简化网络，导致更复杂的结构。


### Causal Intuitions

若变量排序不当，会得到更复杂的贝叶斯网络，实际上相比显式表示完整联合分布（即表示每个原子事件的概率）没有任何节省。

### Inference in Bayes Nets

**给定：**

1. 一个贝叶斯网络：
   $$\text{Pr}(X_1, X_2, \ldots, X_n) = \text{Pr}(X_n \mid \text{Par}(X_n)) \cdot \text{Pr}(X_{n-1} \mid \text{Par}(X_{n-1})) \cdot \ldots \cdot \text{Pr}(X_1 \mid \text{Par}(X_1))$$

2. 一些**证据** E：
   $$E = \{\text{某些变量的取值集合}\}$$

**目标：** 计算新的概率分布 $\text{Pr}(X_k \mid E)$

即，求出 $\text{Pr}(X_k = d \mid E)$，对所有 $d \in \text{Dom}[X_k]$

### 贝叶斯网络示例

网络结构：Difficulty → Grade ← Intelligence，Intelligence → SAT，Grade → Letter

$$
p(X) = \prod_i p_i(X_i \mid \text{Par}_G(X_i))
$$

**节点定义：**

- 试题难度 ($D$)：$d^0$（低），$d^1$（高）
- 智力 ($I$)：$i^0$（低），$i^1$（高）
- 考试成绩 ($G$)：$g^1$（A），$g^2$（B），$g^3$（C）
- 高考成绩 ($S$)：$s^0$（低），$s^1$（高）
- 是否得到推荐信 ($L$)：$l^0$（否），$l^1$（是）

各CPT表：

| $d^0$ | $d^1$ |
|-------|-------|
| 0.6 | 0.4 |

| $i^0$ | $i^1$ |

### Independence in BN

关于贝叶斯网络的重要问题：

- 给定某些证据，两个节点是否独立？
- 如果是，可以用代数证明（通常很繁琐）
- 如果不是，可以用反例证明

**示例：** X → Y → Z

**问题：** X 和 Z 是否必然独立？

- **答：不是。** 例如：低气压导致下雨，下雨导致交通拥堵。
- X 可以影响 Z，Z 也可以影响 X（通过 Y）

### D-separation / D分离

[截图待插入: 装饰性插图，多个球形机器人被不同颜色的闪电连接]

D分离（D-separation）：一种用于回答独立性查询的条件/算法。

- 研究三元组的独立性性质
- 用成员三元组分析复杂情况
- D分离：回答此类查询的条件/算法

### D-separation: Causal Chains（因果链）

配置：X → Y → Z（因果链）

- X: Low pressure（低气压）
- Y: Rain（下雨）
- Z: Traffic（交通）

$$
P(x, y, z) = P(x)P(y|x)P(z|y)
$$

**X 和 Z 是否必然独立？No!**

一组使X不独立于Z的CPT就足以证明独立性不保证。

**示例：** 低气压→下雨→交通拥堵，高气压→不下雨→不拥堵

数值示例：
$P(+y \mid +x) = 1$，$P(-y \mid -x) = 1$，$P(+z \mid +y) = 1$，$P(-z \mid -y) = 1$

**给定 Y 时，X 和 Z 是否独立？Yes!**

$$
P(z|x,y) = \frac{P(x,y,z)}{P(x,y)} = \frac{P(x)P(y|x)P(z|y)}{P(x)P(y|x)} = P(z|y)
$$

**沿链的证据"阻塞"了影响。**

### D-separation: Common Cause（共同原因）

配置：Y → X，Y → Z（共同原因 / common cause）

- Y: Project due（项目截止）
- X: Forums busy（论坛繁忙）
- Z: Lab full（实验室满员）

$$
P(x, y, z) = P(y)P(x|y)P(z|y)
$$

**X 和 Z 是否必然独立？No!**

**示例：** 项目截止导致论坛繁忙和实验室满员

数值：$P(+x \mid +y) = 1$，$P(-x \mid -y) = 1$，$P(+z \mid +y) = 1$，$P(-z \mid -y) = 1$

**给定 Y 时，X 和 Z 是否独立？Yes!**

$$
P(z|x,y) = \frac{P(x,y,z)}{P(x,y)} = \frac{P(y)P(x|y)P(z|y)}{P(y)P(x|y)} = P(z|y)
$$

**观察到原因会阻塞效应之间的影响。**

### D-separation: Common Effect（共同效应 / v-structure）

配置：X → Z ← Y（两个原因导致一个效应，v-结构）

- X: Raining（下雨）
- Y: Ballgame（球赛）
- Z: Traffic（交通）

**X 和 Y 是否独立？**

- **Yes：** 球赛和下雨都会导致交通拥堵，但它们之间不相关。

**给定 Z 时，X 和 Y 是否独立？**

- **No：** 看到交通拥堵会使下雨和球赛成为竞争性的解释。

**这与前两种情况相反：** 观察到效应会**激活**可能原因之间的影响。

再举些例子：
- dead battery → **car won't start** ← no gas
- 陈老师买了2张彩票，赚了1个亿

### Reachability 相关与独立：连通与阻塞 — Active / Inactive Paths

**问题：** 给定证据变量 {Z}，X 和 Y 是否条件独立？

- 是，如果 X 和 Y 被 Z "d分离"
- 考虑从 X 到 Y 的所有（无向）路径
- 没有活跃路径 = 独立！

**路径活跃的条件 — 每个三元组都活跃：**

| 结构类型 | 活跃条件 |
|----------|----------|
| 因果链 A → B → C（任一方向） | B 未被观测 |
| 共同原因 A ← B → C | B 未被观测 |
| 共同效应（v-结构）A → B ← C | B **或**其某个后代被观测 |

**只要有一个不活跃的段就能阻塞整条路径。**

[截图待插入: Active Triples 与 Inactive Triples 对比图，展示四种基本结构的活跃/不活跃状态，灰色节点表示被观测]

再举些例子：老陈买了2张彩票，赚了1个亿，买了法拉利

**相关 = 连通，独立 = 阻塞**


### D-separation

- **Query**: $X_i \perp\!\!\!\perp X_j \mid \{X_{k_1}, ..., X_{k_n}\}$ ?
- **Check all (undirected!) paths between** $X_i$ **and** $X_j$
  - If one or more active, then independence not guaranteed: $X_i \not\perp\!\!\!\perp X_j \mid \{X_{k_1}, ..., X_{k_n}\}$
  - Otherwise (i.e. if all paths are inactive), then independence is guaranteed: $X_i \perp\!\!\!\perp X_j \mid \{X_{k_1}, ..., X_{k_n}\}$

### D-separation Example

**网络结构**: $R \rightarrow T \leftarrow B$, $T \rightarrow T'$

- $R \perp\!\!\!\perp B$ — **Independent**
- $R \perp\!\!\!\perp B \mid T$
- $R \perp\!\!\!\perp B \mid T'$

(All paths inactive = independent)

- Alarm ($A$) $\rightarrow$ JohnCalls ($J$)
- Alarm ($A$) $\rightarrow$ MaryCalls ($M$)

|-----|---------|

### 间接的因果作用（Chain）

**路径**: Burglary $\rightarrow$ Alarm $\rightarrow$ MaryCalls

**结论**: 给定 A 的前提下，M 与 B 条件独立。

$$Pr(m|a) = Pr(m|a,b) = 0.7$$

### 共同的原因（Fork）

**路径**: JohnCalls $\leftarrow$ Alarm $\rightarrow$ MaryCalls

**结论**: 给定 A 的前提下，J 与 M 条件独立。

$$Pr(m|a) = Pr(m|a,j) = 0.7$$

**未给定 A 的前提下，J 与 M 不独立**:

$$Pr(m) = Pr(m|j) \quad \text{（需验证）}$$

计算过程：

$$p(m) = p(m, a=0) + p(m, a=1) = p(m|a=0)p(a=0) + p(m|a=1)p(a=1) = 0.01 \times 0.997484 + 0.7 \times 0.002516 = 0.01173604$$

$$p(a) = p(a|b=0,e=0)p(b=0)p(e=0) + p(a|b=0,e=1)p(b=0)p(e=1) + p(a|b=1,e=0)p(b=1)p(e=0) + p(a|b=1,e=1)p(b=1)p(e=1) = 0.001 \times 0.999 \times 0.998 + 0.29 \times 0.999 \times 0.002 + 0.94 \times 0.001 \times 0.998 + 0.95 \times 0.001 \times 0.002 = 0.002516$$

$$p(m,j) = p(m,j|a=0)p(a=0) + p(m,j|a=1)p(a=1) = p(m|a=0)p(j|a=0)p(a=0) + p(m|a=1)p(j|a=1)p(a=1) = 0.01 \times 0.05 \times 0.997484 + 0.7 \times 0.9 \times 0.002516 = 0.002083822$$

$$p(j) = p(j|a=0)p(a=0) + p(j|a=1)p(a=1) = 0.05 \times 0.997484 + 0.9 \times 0.002516 = 0.0521386$$

$$p(m|j) = \frac{p(m,j)}{p(j)} = \frac{0.002083822}{0.0521386} = 0.0399667$$

### 共同的作用（Collider）

**路径**: Burglary $\rightarrow$ Alarm $\leftarrow$ Earthquake

**结论**: 给定 A 的前提下，B 与 E **不是**条件独立的。

$$Pr(b|a) = Pr(b|a,e) \quad \text{（需验证）}$$

计算过程：

$$p(b|a) = \frac{p(ab)}{p(a)} = \frac{p(a,b,e=0) + p(a,b,e=1)}{p(a)} = \frac{p(a|b,e=0)p(b)p(e=0) + p(a|b,e=1)p(b)p(e=1)}{p(a)} = \frac{0.94 \times 0.001 \times 0.998 + 0.95 \times 0.001 \times 0.002}{0.002516} = \frac{0.00094002}{0.002561} = 0.3736$$

$$p(b|a,e) = \frac{p(a,b,e)}{p(a,e)} = \frac{p(a|b,e)p(b)p(e)}{p(a|e,b=0)p(b=0)p(e) + p(a|e,b=1)p(e)p(b=1)} = \frac{0.95 \times 0.001 \times 0.002}{0.29 \times 0.999 \times 0.002 + 0.95 \times 0.001 \times 0.002} = \frac{0.0000019}{0.00058132} = 0.003268423$$

**未给定 A 的前提下，B 与 E 是独立的**:

$$Pr(b) = Pr(b|e) = 0.001$$

### BN Inference 贝叶斯网络推断

- **Inference**: calculating some useful quantity from a joint probability distribution

**Examples**:
- **Posterior probability**: $P(Q | E_1 = e_1, ... E_k = e_k)$
- **Most likely explanation**: $\text{argmax}_q \, P(Q = q | E_1 = e_1 ...)$

### BN Inference Example by 枚举法

- Given unlimited time, inference in BNs is easy
- **Recipe**:
  - State the marginal probabilities you need
  - Figure out ALL atomic probabilities you need
  - Calculate and combine them

**示例**: 计算 $P(+b, +j, +m)$

$$P(+b, +j, +m) = P(+b)P(+e)P(+a|+b,+e)P(+j|+a)P(+m|+a) + P(+b)P(+e)P(-a|+b,+e)P(+j|-a)P(+m|-a) + P(+b)P(-e)P(+a|+b,-e)P(+j|+a)P(+m|+a) + P(+b)P(-e)P(-a|+b,-e)P(+j|-a)P(+m|-a)$$

对于大型网络（如汽车保险网络），枚举法的计算量呈指数型增长，变得不可行。

### BN Inference — Variable Elimination 变量消除

If the network has a lot of variables, making exact inferences can be computationally hairy.

**Variable elimination 变量消除** uses the product decomposition that defines a Bayes Net and the summing out rule to compute probabilities conditioned on evidence from information in the network (CPTs).

**VE** helps to reduce some of computation required to make exact inferences.

### Inference by 枚举法 vs Variable Elimination 变量消元

**Why is inference by enumeration so slow?**
- You join up the whole joint distribution before you sum out the hidden variables
- 枚举法：指数型增长的联合分布

**Idea: interleave joining and marginalizing!**
- Called "Variable Elimination"
- Still NP-hard, but usually much faster than inference by enumeration
- VE：通过交替合并和边缘化减少分布规模

First we'll need some new notation: **factors**


### Variable Elimination: Factors 1

- 联合分布: $P(X, Y)$
  - 条目 $P(x,y)$ 对所有 $x, y$
  - 求和为 1

- 大写字母数量 = 表格的维度

$P(\text{cold}, W)$

| T | W | P |
|---|---|---|
| cold | sun | 0.2 |
| cold | rain | 0.3 |

- 单一条件分布: $P(Y \mid x)$
  - 条目 $P(y \mid x)$ 对固定的 $x$，所有 $y$
  - 求和为 1

$P(W \mid \text{cold})$

| T | W | P |
|---|---|---|
| cold | sun | 0.4 |
| cold | rain | 0.6 |

- 条件分布族: $P(X \mid Y)$
  - 多个条件分布
  - 条目 $P(x \mid y)$ 对所有 $x, y$
  - 求和为 $|Y|$

$P(W \mid T)$

| T | W | P | |
|---|---|---|---|
| hot | sun | 0.8 | $P(W \mid \text{hot})$ |

### Variable Elimination: Factors 3

- 指定族: $P(y \mid X)$
  - 条目 $P(y \mid x)$ 对固定的 $y$，但所有 $x$
  - 求和为... 谁知道！

| T | W | P |
|---|---|---|

- 一般来说，当我们写 $P(Y_1 \ldots Y_N \mid X_1 \ldots X_M)$
  - 它是一个"因子"(factor)，一个多维数组
  - 其值为所有 $P(y_1 \ldots y_N \mid x_1 \ldots x_M)$
  - 任何已赋值的 $X$ 或 $Y$ 是从数组中缺失(选定)的一个维度

### Marginalizing Early (= Variable Elimination)

提前边缘化（即变量消除）: 在合并过程中尽早消除变量，避免构建过大的联合因子。

### Marginalizing Early (= Variable Elimination) 详细过程

逐步过程:

$P(R)$ 和 $P(T \mid R)$ 通过 **Join R** $\rightarrow P(R, T)$ 通过 **Sum out R** $\rightarrow P(T)$

$P(T)$ 和 $P(L \mid T)$ 通过 **Join T** $\rightarrow P(T, L)$ 通过 **Sum out T** $\rightarrow P(L)$

$P(R)$:

| R | P |
|---|---|
| +r | 0.1 |
| -r | 0.9 |

$P(R, T)$:

| R | T | P |
|---|---|---|
| +r | +t | 0.08 |
| +r | -t | 0.02 |
| -r | +t | 0.09 |
| -r | -t | 0.81 |

$P(T)$:

| T | P |
|---|---|
| +t | 0.17 |
| -t | 0.83 |

$P(T, L)$:

| T | L | P |
|---|---|---|
| +t | +l | 0.051 |
| +t | -l | 0.119 |
| -t | +l | 0.083 |
| -t | -l | 0.747 |

$P(L)$:

| L | P |
|---|---|
| +l | 0.134 |
| -l | 0.886 |

### Evidence (已知取值变量) and Query (最终判断变量)

- 如果有证据，从选定该证据的因子开始
  - 无证据时使用这些初始因子:

$P(R)$ | $P(T \mid R)$ | $P(L \mid T)$

- 计算 $P(L \mid +r)$ 时，初始因子变为:

$P(+r)$:

| R | P |
|---|---|
| +r | 0.1 |

$P(T \mid +r)$:

| R | T | P |
|---|---|---|
| +r | +t | 0.8 |
| +r | -t | 0.2 |

$P(L \mid T)$:

| T | L | P |
|---|---|---|
| +t | +l | 0.3 |
| +t | -l | 0.7 |
| -t | +l | 0.1 |
| -t | -l | 0.9 |

- 我们消除所有除 query + evidence 之外的变量
  - **只保留与前提 Evidence 和结论 Query 相关的变量**

### Evidence and Query (续)

- 结果将是 query 和 evidence 的选定联合分布
  - 例如，对于 $P(L \mid +r)$，最终得到:

$P(+r, L)$:

| R | L | P |
|---|---|---|
| +r | +l | 0.026 |
| +r | -l | 0.074 |

通过 **Normalize** $\rightarrow P(L \mid +r)$:

| L | P |
|---|---|
| +l | 0.26 |
| -l | 0.74 |

- 要得到答案，只需归一化即可！

### Marginalizing Early (= Variable Elimination) 带 Evidence 情况

带 Evidence 情况的变量消除:

- $P(R)$ 中选定 $+r$，$P(T \mid R)$ 中选定 $+r$
- Join R $\rightarrow P(R, T)$ (仅保留 $+r$ 行)
- 然后 Join T $\rightarrow P(+r, L)$
- 最后 Normalize $\rightarrow P(L \mid +r)$

### General Variable Elimination

**General Variable Elimination**

- Query: $P(Q \mid E_1 = e_1, \ldots E_k = e_k)$

- 从初始因子开始:
  - 局部 CPT（但由证据实例化）

- 当仍存在隐藏变量（不是 Q 或证据）时:
  - 选择一个隐藏变量 H
  - 合并所有提到 H 的因子
  - 消除（求和消去）H

- 合并所有剩余因子并归一化 $\times \frac{1}{Z}$

### 练习 4 (a) — 朴素贝叶斯分类器

假设给定如下训练数据集，其中 $A_1$、$A_2$、$A_3$ 为二值输入特征，$y$ 为二值类标签。

| 训练样例 | $A_1$ | $A_2$ | $A_3$ | $y$ |
|---|---|---|---|---|
| $x_1$ | T | F | F | F |
| $x_2$ | T | F | T | F |
| $x_3$ | F | T | F | F |
| $x_4$ | T | T | T | T |
| $x_5$ | T | T | F | T |
| $x_6$ | F | F | F | T |

(a) 对一个新的测试数据，其输入特征 $A_1 = \text{T}$，$A_2 = \text{F}$，$A_3 = \text{F}$，朴素贝叶斯分类器将会预测 $y = \underline{\quad}$？

解答: (a) 对于题目中给定的测试数据，朴素贝叶斯分类器将会预测 $y = \text{F}$。如下:

$$P(C \mid A_1 A_2 \ldots A_n) = \frac{P(A_1 A_2 \ldots A_n \mid C) P(C)}{P(A_1 A_2 \ldots A_n)}$$

$$P(y = \text{T} \mid A_1 = \text{T}, A_2 = \text{F}, A_3 = \text{F}) \propto P(y = \text{T}, A_1 = \text{T}, A_2 = \text{F}, A_3 = \text{F})$$

$$= P(y = \text{T}) \times P(A_1 = \text{T} \mid y = \text{T}) \times P(A_2 = \text{F} \mid y = \text{T}) \times P(A_3 = \text{F} \mid y = \text{T})$$

$$= \frac{1}{2} \times \frac{2}{3} \times \frac{1}{3} \times \frac{2}{3} = \frac{2}{27}$$

$$P(y = \text{F} \mid A_1 = \text{T}, A_2 = \text{F}, A_3 = \text{F}) \propto P(y = \text{F}, A_1 = \text{T}, A_2 = \text{F}, A_3 = \text{F})$$

$$= P(y = \text{F}) \times P(A_1 = \text{T} \mid y = \text{F}) \times P(A_2 = \text{F} \mid y = \text{F}) \times P(A_3 = \text{F} \mid y = \text{F})$$

$$= \frac{1}{2} \times \frac{2}{3} \times \frac{2}{3} \times \frac{2}{3} = \frac{4}{27}$$

因为 $\frac{4}{27} > \frac{2}{27}$，所以预测 $y = \text{F}$。

### 练习 4 — 贝叶斯网络推断

Compute $P(c \mid a, b, \neg d, \neg e, \neg f)$.

网络结构: A → C, B → C, B → D, C → E, C → F

已知概率:

| | |
|---|---|
| $P(a) = 0.9$ | $P(d \mid b) = 0.1$ |
| $P(b) = 0.2$ | $P(d \mid \neg b) = 0.8$ |
| $P(c \mid a, b) = 0.1$ | $P(e \mid c) = 0.7$ |
| $P(c \mid a, \neg b) = 0.8$ | $P(e \mid \neg c) = 0.2$ |
| $P(c \mid \neg a, b) = 0.7$ | $P(f \mid c) = 0.2$ |
| $P(c \mid \neg a, \neg b) = 0.4$ | $P(f \mid \neg c) = 0.9$ |

计算过程:

$$P(a, b, \neg d, \neg e, \neg f, c) = P(a)P(b)P(c \mid a,b)P(\neg d \mid b)P(\neg e \mid c)P(\neg f \mid c)$$

$$= 0.9 \times 0.2 \times 0.1 \times 0.9 \times 0.3 \times 0.8 = 0.003888$$

$$P(a, b, \neg d, \neg e, \neg f, \neg c) = P(a)P(b)P(\neg c \mid a,b)P(\neg d \mid b)P(\neg e \mid \neg c)P(\neg f \mid \neg c)$$

$$= 0.9 \times 0.2 \times 0.9 \times 0.9 \times 0.8 \times 0.1 = 0.011664$$

$$P(c \mid a, b, \neg d, \neg e, \neg f) = \frac{P(a, b, \neg d, \neg e, \neg f, c)}{P(a, b, \neg d, \neg e, \neg f)}$$

$$= \frac{P(a, b, \neg d, \neg e, \neg f, c)}{P(a, b, \neg d, \neg e, \neg f, c) + P(a, b, \neg d, \neg e, \neg f, \neg c)}$$

$$= \frac{0.003888}{0.003888 + 0.011664} = 0.25$$
