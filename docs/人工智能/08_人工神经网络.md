### 机器学习原理: 找到最好的函数

### 机器学习原理: 三大步骤

| 步骤 | 说明 | 方法 |
|------|------|------|
| 设定范围 | 考虑任务特性，确定候选函数的集合 | Deep Learning (CNN, Transformer ...), Decision Tree, etc. |
| 设定标准 | 确定「评价函数好坏」的标准 | Supervised Learning, Semi-supervised Learning, RL, etc. |
| 达成目标 | 找出最好的函数 | Gradient Descent (Adam ...), Genetic Algorithm, etc. |

### 机器学习类别: 学习范式

- **监督学习 (Supervised Learning)**: 训练数据(Data) + 全部标签(Label)
- **无监督学习 (Unsupervised Learning)**: 训练数据 (无需标签)
- **半监督学习 (Semi-supervised Learning)**: 训练数据 + 部分标签
- **强化学习 (Reinforcement Learning)**: 智能体在环境中采取行动时的状态(State)和奖励(Reward)

## 无监督 (聚类) 算法

### 什么是聚类分析?

Finding groups of objects such that the objects in a group will be similar (or related) to one another and different from (or unrelated to) the objects in other groups.

寻找一组对象，使得同一组内的对象彼此相似（或相关），而与不同组内的对象不同（或不相关）。

- **Intra-cluster distances are minimized** (簇内距离最小化)
- **Inter-cluster distances are maximized** (簇间距离最大化)

### 聚类算法

- Hierarchical clustering (Omit, because...)

### 阶层式聚类算法

**Hierarchical clustering**

[截图待插入: 左侧为A-F六个数据点的散点图及对应的Dendrogram树状图；右侧展示层次聚类迭代合并过程，从识别最近两个簇到合并最相似的两个簇，直到所有簇合并为一个整体]

层次聚类从将每个观测值视为一个独立簇开始。

随后，它循环执行以下两个步骤:

1. 识别距离最近的两个簇
2. 合并最相似的两个簇

此迭代过程持续进行，直到所有簇合并为一个整体。

### K-means聚类

- **Partitional clustering approach 分割式聚类方法**
  - Each cluster is associated with a **centroid** (center point) 每个簇有一个中心点
  - Each point is assigned to the cluster with the **closest centroid** 将各数据点划分到最近中心点对应的簇
- Number of clusters, K, must be specified
- The basic algorithm is very simple

**算法 1 基础 K-means 算法。**

1. 选择 K 个点作为初始中心。
2. **重复**
3. 通过将所有点分配给最近的中心来形成 K 个簇。
4. 重新计算每个簇的中心。
5. **直到** 中心不再发生变化

### 标准K-means流程

[截图待插入: K-means迭代过程图，Iteration 1到Iteration 6，展示数据点从初始随机分配到逐步收敛到三个稳定簇的过程，颜色分别为红、绿、蓝]

### K-mean聚类细节

- 初始中心点通常**随机选择**。
  - 不同次运行得到的聚类结果可能不同。
- 中心点通常是簇内各点的**均值**。
- "相近度"可通过欧氏距离、余弦相似度、相关系数等度量。
- 使用上述常见相似性度量时，**K-means算法能够保证收敛**。
- 大部分收敛发生在最初几次迭代中。Often the stopping condition is changed to 'Until relatively few points change clusters'
- Complexity is $O(n \times K \times I \times d)$
  - $n$ = number of points, $K$ = number of clusters, $I$ = number of iterations, $d$ = number of attributes

### 选择初始质心的重要性

[截图待插入: 两组不同的K-means迭代过程对比图，展示不同初始质心选择导致不同的聚类收敛结果]

### K-means簇的评估

- 最常用的度量是误差平方和 **Sum of Squared Error (SSE)**
  - 计算每个点到其最近聚类中心的距离作为误差
  - 将所有这些误差平方后求和，即得到SSE

$$SSE = \sum_{i=1}^{K} \sum_{x \in C_i} dist^2(m_i, x)$$

- $x$表示簇$C_i$中的数据，$m_i$代表簇$C_i$中心点
- 给定两个聚类方案时，我们选择误差更小的那个
- 降低SSE最直接的方法是增加聚类数量K
  - 一个K值较小的优质聚类，其SSE可能低于K值更大的劣质聚类

### K的选择

- **Elbow method肘部法则**: 绘制不同K值对应的SSE折线图，若图形呈手臂状，则"肘关节"处对应的K值即为最优选择。
- 手臂曲线的肘关节

[截图待插入: Elbow method折线图，横轴为Number of clusters k(1-10)，纵轴为Total Within Sum of Square，曲线在k=3处出现肘部拐点，标注Optimal number of clusters]

### 选择初始点的问题

- 若存在K个真实簇群，则从每个簇中各选出一个中心点的概率很小。
  - 当K值较大时，这种概率会显著降低
  - 假设各簇规模相同（均为$n$个数据点），则:

$$P = \frac{\text{number of ways to select one centroid from each cluster}}{\text{number of ways to select } K \text{ centroids}} = \frac{K! n^K}{(Kn)^K} = \frac{K!}{K^K}$$

- For example, if $K = 10$, then probability = $10!/10^{10} = 0.00036$
- 初始中心点有时会自我调整到正确位置，有时则不能
- 以10个簇群的情况为例

### 如何选择初始质心?

- **多次运行**
  - 虽有效果，但成功概率依然有限
- **二分K均值法**
  - 采用分裂式聚类策略: 每次选择一个簇进行二分，最终形成K个簇
- **采样结合层次聚类**
  - 先对数据采样，再用层次聚类确定初始中心点
- **优选中心点**
  - 选取多于K个初始中心点，从中筛选分布最分散的K个点

### Bisecting(二分) K-means

- **Bisecting K-means algorithm**
  - Variant of K-means that can produce a partitional or a hierarchical clustering

**算法 3 二分 K-means 算法。**

1. 初始化簇列表，使其包含涵盖所有点的簇。
2. **重复**
3. 从簇列表中选择一个簇
4. **对于** $i = 1$ 到 $number\_of\_iterations$ **执行**
5. 使用基础 K-means 对所选簇进行二分
6. **结束 for**
7. 将具有最低 SSE 的二分结果中的两个簇添加到簇列表中。
8. **直到** 簇列表包含 $K$ 个簇

### K-means++ [Arthur et al. '07]

**算法执行步骤：**（核心思想：Spreads out the centers 分散选择中心点）

1. 从数据集中随机均匀选取第一个中心点 $x_1$
2. 当 $2 \le i \le k$ 时重复以下过程：
   - 按照特定概率分布采样得到新中心点 $x'$
   - 该分布与 $D(x)^2$ 成正比，其中 $D(x)$ 表示数据点 $x$ 到已选中心点的最小距离

$$\frac{D(x')^2}{\sum_{x \in \mathcal{X}} D(x)^2}$$

用通俗的话来讲，这个算法其实就是想让初始的几个中心点尽可能"离得越远越好"，不要扎堆。

[截图待插入: K-means++示意图，展示数据空间中四个簇，每个簇中用黄色星形标记选出的中心点，中心点分散在不同簇中]

### K-means++ 的问题

**What's wrong with K-means++?**

- 需要执行K次数据遍历
- 在大规模数据场景中，不仅数据体量庞大，K值通常也很大（例如轻易达到1000量级）
- 可扩展性差！

### 可伸缩K-means: K-means||

**Intuition of**

- K-means++ 每轮迭代只采样一个中心点并立即更新概率分布
- 如果我们通过独立采样扩大单次采样量会怎样？
- 这相当于大幅降低分布更新的频率
- 采用更粗粒度的采样方式
- 实践证明这种方式完全可行：这就是 **K-means||** 算法的核心思想

**主要思路**在于改变每次遍历时的取样策略，并非按照 k-means++ 那样每次遍历只取样一个样本，而是每次遍历取样A个样本，重复该取样过程大约 B次，共得到 A*B个样本点组成的集合

- 然后再聚类这 A*B个点成 k 个点，最后将这 k 个点作为初始聚类中心
- 实际实验证明**一般5次重复取样就可以得到一个较好的聚类初始中心**

对于点(1, 2)：

$$d_{11} = |1-1| + |2-2| = 0, \quad d_{12} = |1-8| + |2-2| = 7$$

对于点(2, 4)：

因此，(2, 4)应当被聚为第 1 类；

对于点(1, 9)：

$$d_{31} = |1-1| + |9-2| = 7, \quad d_{32} = |1-8| + |9-2| = 14$$

因此，(1, 9)应当被聚类为第 1 类；

对于点(6, 5)：

$$d_{41} = |6-1| + |5-2| = 8, \quad d_{42} = |6-8| + |5-2| = 5$$

因此，(6, 5)应当被聚类为第 2 类；

对于点(4, 2)：

$$d_{51} = |4-1| + |2-2| = 3, \quad d_{52} = |4-8| + |2-2| = 4$$

因此，点(4, 2)应当被聚类为第 1 类；

对于点(7, 2)：

$$d_{61} = |7-1| + |2-2| = 6, \quad d_{62} = |7-8| + |2-2| = 1$$

### DBSCAN

DBSCAN是一种基于密度（Density-based）的聚类算法。

- **密度Density**：指定半径范围内包含的数据点数量
- **核心点core point**：在半径范围内包含至少MinPts个点的点
  - 这类点位于簇的内部核心区域
- **边界点border point**：在半径范围内包含少于MinPts个点，但位于某个核心点的邻域内
  - 处于簇的边界区域
- **噪声点noise point**：既不是核心点也不是边界点的点
  - 属于异常数据或离散点

[截图待插入: 基于中心的密度示意图，圆形区域内有7个点（Density = 7 points），中心点A到边界的距离为Eps]

**Density** = number of points within a specified radius (Eps)

### DBSCAN: 核心点、边界点、噪声点

[截图待插入: 左图为Figure 8.21，MinPts=4，展示核心点A（Eps范围内点>=4）、边界点B（Eps范围内点<4但在核心点邻域内）、噪声点C（既非核心也非边界）；右图为社交场景类比图，社牛(Core Point)为核心点，外围成员(Border Point)为边界点，独行侠(Noise Point)为噪声点]

- A point is a **core point** if it has more than a specified number of points (MinPts) within Eps. These are points that are at the interior of a cluster — **核心点**
- A **border point** has fewer than MinPts within Eps, but is in the neighborhood of a core point — **边界点**
- A **noise point** is any point that is not a core point or a border point — **噪声点**

[截图待插入: 散点图展示Eps=1, MinPts=4条件下，Core Point、Border Point、Noise Point的位置关系，用虚线圆圈表示Eps范围]

### DBSCAN 算法

**Center-based approach**

**算法 S.4 DBSCAN 算法步骤**

1. 将所有点标记为**核心点**、**边界点**或**噪声点**。
2. **剔除噪声点**（使其不参与后续聚类）。
3. 在所有**相互距离在 Eps 范围内的核心点之间建立连接边**。
4. 将每个**连通的核心点**组形成一个独立的**簇**。
5. 将每个**边界点分配**到与其关联的**核心点所在的簇**中。

**Algorithm 8.4 DBSCAN algorithm.**

1. Label all points as core, border, or noise points.
2. Eliminate noise points.
3. Put an edge between all core points that are within $Eps$ of each other.
4. Make each group of connected core points into a separate cluster.
5. Assign each border point to one of the clusters of its associated core points.

### 人工神经元模型MP

类比生物结构：输入可以类比为神经元的树突，而输出可以类比为神经元的轴突，计算则可以类比为细胞核。神经元模型是一个包含输入、输出与计算功能的模型。

下图是一个典型的神经元模型：包含有3个输入，1个输出，以及2个计算功能。

注意中间的箭头线。这些线称为"连接"。每个上有一个"权值"。

[截图待插入: 神经元模型示意图，左侧为输入1/2/3经权值1/2/3连接到求和节点，再经非线性函数到输出；右侧为生物神经元结构图，标注树突、轴突末梢、细胞核、轴突]

### 人工神经元模型MP（计算过程）

一个典型的神经元模型，包含有**N个输入**，**1个输出**，以及**2个计算功能**，每个连接上有一个权值。

| 输入 | 权值 | 计算过程 | 输出 |
|------|------|----------|------|
| $a_1$ | $w_1$ | 求和 sum | $z$ |
| $a_2$ | $w_2$ | 非线性函数 $g$ | |
| $a_3$ | $w_3$ | | |

$$z = g(a_1 w_1 + a_2 w_2 + a_3 w_3)$$

### 人工神经元模型MP（计算公式）

神经元的计算公式：用 $\boldsymbol{a} = [a_1, a_2, a_3]^T$ 表示输入，用 $\boldsymbol{W} = [w_1, w_2, w_3]$ 表示权值，经过加权计算末端信号为 $\boldsymbol{W} \times \boldsymbol{a}$，再叠加非线性函数 $g$，即是输出值 $z$。

在MP模型里，**非线性函数 $g$** 设置是**符号函数（sgn）**。

由于起到类似神经元的"激活"作用，也称作**激活函数**（Active Function）。

$$\text{sgn}(x) = \begin{cases} 1, & x \geq 0 \\ -1, & x < 0 \end{cases}$$

$$z = g(a_1 w_1 + a_2 w_2 + a_3 w_3) = g(\boldsymbol{W} \times \boldsymbol{a})$$

[截图待插入: 左侧为神经元计算流程图（输入a1/a2/a3经权值w1/w2/w3到求和sum再到非线性函数g到输出z）；右侧为sgn函数图像，阶梯状，x≥0时y=1，x<0时y=-1]

### 人工神经元模型MP（简化图）

对神经元模型的图进行简化：将求和函数sum与激活函数sgn合并到一个圆圈里，代表神经元的内部计算。此时，一个**"节点"**就是一个神经元。

[截图待插入: 左侧为详细神经元图（a1/a2/a3经w1/w2/w3到包含sum和sgn的圆圈，输出z）；右侧为简化节点图（a1/a2/a3经w1/w2/w3到Node圆圈，输出z）]

### MP神经元的理解

神经元模型可以这样理解：

- 根据3个**已知属性**预测**未知属性**
- 神经元模型中：3个已知属性的值是 $a_1$，$a_2$，$a_3$，未知属性的值是 $z$
- 已知的属性称为**特征**，未知属性称为**目标**

| 已知（特征） | 权值 | 计算过程 | 未知（目标） |
|-------------|------|----------|-------------|
| $a_1$ 细胞大小 | $w_1$ | 求和 sum → 激活函数 sgn | $z$ |
| $a_2$ 细胞形状 | $w_2$ | | 恶性肿瘤 / 良性肿瘤 |
| $a_3$ 分化程度 | $w_3$ | | |

### MP神经元的缺陷

1943年发布的MP模型，虽然简单，但已经建立了神经网络大厦的地基。然而，MP模型中，**权重的值都是预先设置的**，因此不能进行学习。

1949年心理学家Hebb发现人脑神经细胞的突触上的强度是可以变化的，提出了**Hebb学习率**。这启发计算机科学家们开始考虑用调整权值的方法来让**机器自我学习**，为后面的学习算法奠定了基础。

基于这个启发，才**诞生了感知机（Perceptron）**。

### 感知机模型

**感知机（Perceptron）**：由两层神经元构成

- **输入层**：在MP模型的输入位置添加神经元节点，**只传输数据**，不做计算
- **输出层**：对前面一层的输入**进行计算**

[截图待插入: 感知机结构图，输入层a1/a2/a3（红色节点）经权值w1,1/w1,2/w2,1/w2,2/w3,1/w3,2连接到输出层f（绿色节点），输出z1/z2]

感知机拥有一个计算层，也称为"单层神经网络"。

### 感知机模型数学化公式

输入向量 $\boldsymbol{a} = [a_1, a_2, a_3]^T$，输出向量 $\boldsymbol{z} = [z_1, z_2]^T$，系数矩阵 $\boldsymbol{W} \in \mathbb{R}^{2 \times 3}$。

则用矩阵乘法表达感知机的计算公式为：$\boldsymbol{z} = g(\boldsymbol{W} \times \boldsymbol{a})$。

该公式也是**神经网络中从前一层到后一层的运算过程**。

$$z_1 = g(a_1 w_{1,1} + a_2 w_{1,2} + a_3 w_{1,3})$$
$$z_2 = g(a_1 w_{2,1} + a_2 w_{2,2} + a_1 w_{2,3})$$

[截图待插入: 感知机网络图，输入a1/a2/a3经权值矩阵W连接到输出f节点，输出z1/z2]

### 感知机模型数学化公式（训练）

与神经元MP模型不同，感知机中的权重是通过**训练**得到的。

目标：$\min_{\boldsymbol{W}} |\boldsymbol{y} - \hat{\boldsymbol{y}}|$

[截图待插入: 感知机训练图，特征a1到an经权值w1/w2/wn连接到节点，输出 $\hat{y} = \text{sgn}(\boldsymbol{w} \times \boldsymbol{a})$，目标为y]

$$\text{sgn}(z) = \begin{cases} 1, & \text{if } z \geq 0 \\ -1, & \text{otherwise} \end{cases}$$

**感知机训练法则：**

$$W_t \leftarrow W_t + \Delta W_t$$
$$\Delta W_t = \eta (y - \hat{y}) \boldsymbol{a}$$

其中：
- $y$：是真实的目标
- $\hat{y}$：是感知机的输出
- $\eta$：学习速率（如 0.1）
- $\boldsymbol{a}$：训练数据

### 感知机的缺陷（实验结论）

**实验结论**：感知机只能做简单的线性分类任务。

### 多层感知机

单层神经网络无法解决异或问题。但是当增加一个计算层以后，两层神经网络不仅可以解决异或问题，而且具有非常好的非线性分类效果。不过两层神经网络的计算是一个问题，没有一个较好的解法。

### 多层感知机模型（多层神经网络/MLP）

- **多层神经网络**：即**多层感知机（Multi-Layer Perceptron，简称MLP）**
- 包含三个层次：一个**输入层**，一个或多个中间层（也叫**隐藏层**，hidden layer）和一个**输出层**

### 多层感知机模型数学化表达

使用向量和矩阵来表示神经网络中的变量。$\boldsymbol{x}$, $\boldsymbol{a}$, $\boldsymbol{z}$ 是网络中传输的向量数据。$\boldsymbol{W_1}$ 和 $\boldsymbol{W_2}$ 是网络的矩阵参数。

矩阵运算具有简洁性，节点数增多也不受影响。

$$\boldsymbol{a} = g(\boldsymbol{W_1} \times \boldsymbol{x})$$
$$\boldsymbol{z} = g(\boldsymbol{W_2} \times \boldsymbol{a}) = g(\boldsymbol{W_2} \times g(\boldsymbol{W_1} \times \boldsymbol{x}))$$

MLP通常还会引入**偏置单元**，它与后一层的所有节点都有连接。

### 多层感知机模型激活函数

两层神经网络**不再使用sgn函数**作为激活函数，而是使用**平滑函数sigmoid**。

**不同的激活函数：**

| 函数 | 公式 | 说明 |
|------|------|------|
| (a) sgn | $\text{sgn}(x) = \begin{cases} 1, & x > 0 \\ -1, & x < 0 \end{cases}$ | 用于MP神经元，感知机 |
| (b) sigmoid | $\text{sigmoid}(x) = \frac{1}{1 + e^{-x}}$ | 多用于多层神经网络(MLP) |
| (c) ReLU | $\text{ReLU}(x) = \max(0, x)$ | 多用于深层神经网络(DNN) |
| (d) tanh | $\text{tanh}(x) = \frac{e^x - e^{-x}}{e^x + e^{-x}}$ | 多用于深层神经网络(DNN) |

### 多层神经网络的拟合原理

**通用近似定理（Universal Approximation Theorem）**[Cybenko, 1989, Hornik et al., 1989]：

令 $\varphi(\cdot)$ 是一个非常数、有界、单调递增的连续函数，$\mathbb{I}_d$ 是一个d维的单位超立方体 $[0,1]^d$，$C(\mathbb{I}_d)$ 是定义在 $\mathbb{I}_d$ 上的连续函数集合。对于任何一个函数 $\boldsymbol{f(x)} \in C(\mathbb{I}_d)$，存在一个整数 $m$，和一组实数 $v_i, b_i \in \mathbb{R}$ 以及实数向量 $\boldsymbol{W_i} \in \mathbb{R}^d, i = 1, \cdots, m$，可以定义函数：

$$F(\boldsymbol{x}) = \sum_{i=1}^{m} v_i \varphi(\boldsymbol{W_i^T x} + b_i)$$

作为函数 $f$ 的近似实现，即：

$$|F(\boldsymbol{x}) - f(\boldsymbol{x})| < \epsilon, \forall \boldsymbol{x} \in L_d \quad \text{其中} \epsilon > 0 \text{是一个很小的正数。}$$

[截图待插入: 神经网络拟合示意图，输入x经W1到隐藏层φ节点（z1-z4），再经v1-v4加权求和到Σ输出F(x)，偏置b1]

- 多层神经网络的本质就是通过**矩阵参数**与**激活函数**来拟合特征与目标之间的真实函数关系
- 因此，**多层神经网络也能够解决XOR异或问题！**

### 多层感知机效果（总结）

两层神经网络可以做非线性分类的关键--隐藏层。

矩阵公式中矩阵和向量相乘以及非线性函数sigmod/tanh/...，本质上就是对向量的坐标空间进行一个变换。因此，隐藏层的作用就是使得数据的原始坐标空间**从线性不可分，转换成了线性可分**。

两层神经网络通过两层的线性模型模拟了数据内真实的非线性函数。因此，多层的神经网络的**本质就是复杂函数拟合**。

设计一个神经网络时，输入层的节点数需要与特征的维度匹配，输出层的节点数要与目标的维度匹配。如何决定这个自由层的节点数呢？一般根据经验来设置。

**Grid Search（网格搜索）**：预先设定几个可选值，通过切换这几个值来看整个模型的预测效果，选择效果最好的值作为最终选择。
