### 循环神经网络基本结构

### 循环神经网络计算示例

输入序列: $[1, 1]^T$, $[1, 1]^T$, $[2, 2]^T$, ......

输出序列: $[4, 4]^T$, $[12, 12]^T$, $[32, 32]^T$

假设所有权重为"1"，没有偏置，且所有激活函数都为线性。

给定初始化值为0，逐步计算：

- 第1步: 输入$[1,1]^T$，隐藏层值为2，输出为4
- 第2步: 输入$[1,1]^T$，隐藏层值为$1+1+2=4$（加上记忆），实际为6，输出为12
- 第3步: 输入$[2,2]^T$，隐藏层值为$2+2+6=10$，实际为16，输出为32

**改变输入序列的顺序将会对输出产生影响！**

尝试使用ReLU激活函数: $R(z) = \max(0, z)$，输入序列 $[2,2]^T$, $[-1,-1]^T$, $[1,1]^T$ ......

### 循环神经网络在NLP中的应用

相同的网络结构多次使用。以句子"arrive Guangzhou on May 31th"为例：

- "arrive"对应每个类别的概率 $y^1$
- "Guangzhou"对应每个类别的概率 $y^2$
- "on"对应每个类别的概率 $y^3$

每个时间步的隐藏状态$a^t$通过store传递给下一步，作为额外输入。

**概率分布不同**：同一词"Guangzhou"在不同上下文（"leave" vs "arrive"）中对应的类别概率不同，因为记忆中存储的信息是不同的。

## 长短记忆网络

### 长短记忆网络 (Long Short-term Memory)

LSTM包含特殊神经元，具有4项输入、1项输出。核心组件：

- **Memory Cell**: 记忆单元
- **Input Gate**: 输入门，控制信号
- **Forget Gate**: 遗忘门，控制信号
- **Output Gate**: 输出门，控制信号

[截图待插入: LSTM单元结构图，展示Memory Cell、Input Gate、Forget Gate、Output Gate的连接关系]

### LSTM内部计算

通常使用sigmoid激活函数，限制在0到1之间，模拟门的开关。

记忆单元更新公式:

$$c' = g(z)f(z_i) + c f(z_f)$$

输出公式:

$$a = h(c')f(z_o)$$

其中:
- $z_i$: 输入门信号
- $z_f$: 遗忘门信号
- $z_o$: 输出门信号
- $f$: sigmoid激活函数
- $g, h$: 其他激活函数

[截图待插入: LSTM Block详细计算图，展示各门控信号和乘法操作]

### LSTM示例

任务规则:
- 当$x_2 = 1$，把$x_1$的值累加到记忆中
- 当$x_2 = -1$，重置记忆
- 当$x_3 = 1$，输出记忆的值

输入序列:

| 时间步 | $x_1$ | $x_2$ | $x_3$ | $y$ |
|--------|-------|-------|-------|-----|
| 1 | 1 | 0 | 0 | 0 |
| 2 | 3 | 1 | 0 | 0 |
| 3 | 2 | 0 | 0 | 0 |
| 4 | 4 | 1 | 0 | 0 |
| 5 | 2 | 0 | 0 | 0 |
| 6 | 1 | 0 | 1 | 7 |
| 7 | 3 | -1 | 0 | 0 |
| 8 | 6 | 1 | 0 | 0 |
| 9 | 3 | -1 | 0 | 0 |
| 10 | 6 | 0 | 1 | 6 |

逐步计算过程:

**第1步** (输入$[0,0,0,1]^T$): Memory Cell = 0, Output Gate $\approx 0$, $y \approx 0$

**第2步** (输入$[3,1,0,1]^T$): Input Gate $\approx 1$, Forget Gate $\approx 1$, Memory Cell = $3+0=3$, $y \approx 0$

**第3步** (输入$[4,1,0,1]^T$): Input Gate $\approx 1$, Forget Gate $\approx 1$, Memory Cell = $4+3=7$, $y \approx 0$

**第4步** (输入$[2,0,0,1]^T$): Input Gate $\approx 0$, Forget Gate $\approx 1$, Memory Cell = $0+7=7$, $y \approx 0$

**第5步** (输入$[1,0,1,1]^T$): Input Gate $\approx 0$, Forget Gate $\approx 1$, Output Gate $\approx 1$, Memory Cell = 7, $y \approx 7$

**第6步** (输入$[3,-1,0,1]^T$): Input Gate $\approx 0$, Forget Gate $\approx 0$, Memory Cell = $3+0=0$（重置）, $y \approx 0$

**第7步** (输入$[6,1,0,1]^T$): Memory Cell = 6

**第8步** (输入$[3,-1,0,1]^T$): Memory Cell重置为0

**第9步** (输入$[6,0,1,1]^T$): Output Gate $\approx 1$, $y \approx 6$

### LSTM展开结构

LSTM展开后，每个时间步使用相同的Block结构，参数量为普通RNN的4倍（因为有4个门控/计算模块）。

## 深度学习II: Transformer

### 利用上下文语境表示单词

**分布语义学**: 单词的含义由经常出现在附近的单词给出

> "You shall know a word by the company it keeps" (J. R. Firth 1957: 11)

- 是现代统计NLP最成功的想法之一
- 当单词$w$出现在文本中时，其**上下文（context）**是出现在附近的单词集（在一个固定大小的窗口内）
- 我们利用$w$的丰富上下文来构建$w$的表示

示例:
- ...government debt problems turning into **banking** crises as happened in 2009...
- ...saying that Europe needs unified **banking** regulation to replace the hodgepodge...
- ...India has just given its **banking** system a shot in the arm...

These context words will represent **banking**

### 词向量

**简短但密集**的实数向量，大概50-300维。

为每个单词构建一个密集向量，使其类似于出现在相似上下文中的单词向量，使用**点（数量）积**衡量相似性。

$$\text{banking} = \begin{pmatrix} 0.286 \\ 0.792 \\ -0.177 \\ -0.107 \\ 0.109 \\ -0.542 \\ 0.349 \\ 0.271 \end{pmatrix} \quad \text{monetary} = \begin{pmatrix} 0.413 \\ 0.582 \\ -0.007 \\ 0.247 \\ 0.216 \\ -0.718 \\ 0.147 \\ 0.051 \end{pmatrix}$$

注意: 词向量也叫**(词)嵌入**或**(自然)词表征**，它们是**分布式的表征**。

### Word2Vec概述

Word2Vec是一个学习词向量的框架（Mikolov et al. 2013）。

**思想**:
- 我们有一个很大的文本语料库（"正文"）：一长串单词
- 固定词汇表中的每个单词都由一个vector表示
- 遍历文本中的每个位置$t$，其中包含一个中心词$c$和上下文（"外部"）词$o$
- 使用$c$和$o$的词向量相似性来计算给定$c$时，$o$的概率（反之亦然）
- 不断调整单词向量来最大化此概率

[截图待插入: Skip-gram model架构图，展示w(t)通过projection预测w(t-2), w(t-1), w(t+1), w(t+2)]

参考: Mikolov et al. 2013: "Distributed Representations of Words and Phrases and their Compositionality"

### Word2Vec窗口计算

以一个窗口为例，计算$P(w_{t+j}|w_t)$的过程:

$$P(w_{t-2}|w_t), \quad P(w_{t-1}|w_t), \quad P(w_{t+1}|w_t), \quad P(w_{t+2}|w_t)$$

示例窗口（大小为2）:

| ... | problems | turning | **into** | banking | crises | as | ... |
|-----|----------|---------|----------|---------|--------|-----|-----|
| | outside context words (window size 2) | | center word at position t | | outside context words (window size 2) | | |

滑动窗口:

| ... | problems | turning | into | **banking** | crises | as | ... |
|-----|----------|---------|------|-------------|--------|-----|-----|
| | outside context words (window size 2) | | center word at position t | | outside context words (window size 2) | | |

### Word2Vec目标函数

对每个位置$t = 1, ..., T$，在固定大小为$m$的窗口内预测上下文的单词。给定中心词$w_t$，数据的似然:

$$\text{Likelihood} = L(\theta) = \prod_{t=1}^{T} \prod_{\substack{-m \le j \le m \\ j \ne 0}} P(w_{t+j} | w_t; \theta)$$

其中$\theta$是所有需要优化的变量。

目标函数$J(\theta)$为似然负数的均值:

$$J(\theta) = -\frac{1}{T} \log L(\theta) = -\frac{1}{T} \sum_{t=1}^{T} \sum_{\substack{-m \le j \le m \\ j \ne 0}} \log P(w_{t+j} | w_t; \theta)$$

最小化目标函数 $\Leftrightarrow$ 最大化预测准确率

### Word2Vec概率计算

要最小化目标函数:

$$J(\theta) = -\frac{1}{T} \sum_{t=1}^{T} \sum_{\substack{-m \le j \le m \\ j \ne 0}} \log P(w_{t+j} | w_t; \theta)$$

**问题**: 如何计算$P(w_{t+j}|w_t; \theta)$

**答案**: 对每个单词$w$，使用两个向量:
- $v_w$: 当$w$为中心词时
- $u_w$: 当$w$为上下文词时

这两个向量都是所有参数$\theta$确定的更大向量的子部分。

然后对中心词$c$和上下文单词$o$:

$$P(o|c) = \frac{\exp(u_o^T v_c)}{\sum_{w \in V} \exp(u_w^T v_c)}$$

### Word2Vec预测函数

$$P(o|c) = \frac{\exp(u_o^T v_c)}{\sum_{w \in V} \exp(u_w^T v_c)}$$

各部分含义:
1. **点积操作**比较$o$和$c$的相似度: $u^T v = u \cdot v = \sum_{i=1}^{n} u_i v_i$，更大的点积 = 更大的概率
2. **指数操作**使其为正
3. **在整个词典上进行归一化**，给出概率分布

这是**softmax损失函数**的例子 ($\mathbb{R}^n \to (0,1)^n$):

$$\text{softmax}(x_i) = \frac{\exp(x_i)}{\sum_{j=1}^{n} \exp(x_j)} = p_i$$

- "max" 是因为它放大了最大的$x_i$的概率
- "soft" 是因为它仍然为较小的$x_i$赋予了一些概率
- 在深度学习中频繁用到

### Word2Vec其他细节

**为什么是两个向量？** 更易优化，最后两者均取平均值。但是可以实现每个单词只有一个向量的算法。

**两种变体**:
1. **Skip-grams (SG)**: 预测给定中心词的上下文（"外部"）词（与位置无关）
2. **Continuous Bag of Words (CBOW)**: 从上下文词（袋）预测中心词

**训练损失函数**:
1. 朴素的(naive)softmax（简单，但输出类别特别多时，计算开销大）
2. 使用如分层softmax等优化的变体
3. 负采样

到目前为止，我们解释了朴素softmax。

### CBOW: 目标函数

[截图待插入: CBOW模型结构图，展示INPUT、PROJECTION、OUTPUT三层，上下文词w(t-2), w(t-1), w(t+1), w(t+2)通过SUM投影后预测中心词w(t)]

目标函数:

$$L(\theta) = \prod_{t=1}^{T} P(w_t \mid \{w_{t+j}\}, -m \le j \le m, j \ne 0)$$

上下文词向量平均:

$$\bar{\mathbf{v}}_t = \frac{1}{2m} \sum_{-m \le j \le m, j \ne 0} \mathbf{v}_{t+j}$$

条件概率:

$$P(w_t \mid \{w_{t+j}\}) = \frac{\exp(\mathbf{u}_{w_t} \cdot \bar{\mathbf{v}}_t)}{\sum_{k \in V} \exp(\mathbf{u}_k \cdot \bar{\mathbf{v}}_t)}$$

- **Continuous Bag of Words (CBOW)**: 使用窗口中的**上下文单词**来**预测中心词**

### Seq2Seq模型的通用性

- 这里的一般概念是个**编码器-解码器(encoder-decoder)**模型
  - 一个神经网络接受输入并生成神经表征
  - 另一个神经网络根据该神经表示生成输出
  - 如果输入和输出是序列，我们称其为 seq2seq 模型

- Seq2Seq不止是在MT上好用
- 许多 NLP 任务可以表述为sequence-to-sequence:
  - **文本总结** (长文本→短文本)
  - **对话** (先前的话语→接下来的话语)
  - **解析** (输入文本→以序列的形式输出解析)
  - **代码生成** (自然语言→Python代码)

### Seq2Seq和注意力: 形式化表示

- 编码器隐藏状态: $\mathbf{h}_1, \ldots, \mathbf{h}_N \in \mathbb{R}^h$
- 在时间步$t$，有解码器隐藏状态$\mathbf{s}_t \in \mathbb{R}^h$
- 对这一步计算注意力分数$\mathbf{e}^t$:

$$\mathbf{e}^t = [\mathbf{s}_t^T \mathbf{h}_1, \ldots, \mathbf{s}_t^T \mathbf{h}_N] \in \mathbb{R}^N$$

- 对这一步用softmax来得到注意力分布$\alpha^t$（这是个概率分布且和为1）:

$$\alpha^t = \text{softmax}(\mathbf{e}^t) \in \mathbb{R}^N$$

- 使用$\alpha^t$，对编码器隐藏状态进行加权求和，得到注意力输出$\mathbf{a}_t$:

$$\mathbf{a}_t = \sum_{i=1}^{N} \alpha_i^t \mathbf{h}_i \in \mathbb{R}^h$$

- 最后将注意力输出$\mathbf{a}_t$和解码器隐藏状态$\mathbf{s}_t$拼接，并和非注意力Seq2Seq模型一样继续:

$$[\mathbf{a}_t; \mathbf{s}_t] \in \mathbb{R}^{2h}$$

### Seq2Seq和注意力: 注意力的优势

- 注意力能够显著**提升NMT的性能**
  - 允许解码器专注于源句子的某些部分
- **注意**：可以在翻译时回顾源句子，而无需记住所有内容
- 注意力**解决了瓶颈问题**
  - 注意力允许解码器直接查看源句子，绕开了瓶颈问题
- 注意力**有助于解决梯度消失问题**
  - 提供了捷径到更远的状态
- 注意力提供了**一些可解释性**
  - 可以通过检查注意力分布看到解码器关注的内容
  - 可以直接获得(软)**对齐**
  - 因为从未明确训练过对齐系统，所以这是很有用的
  - 网络只是自己学会了对齐

### 注意力的通用性

**注意力是一种通用的深度学习技术**

- 我们已经看到，注意力是改进机器翻译的序列到序列模型的好方法。
- 但是：注意力在**很多架构**中都能够使用（不仅仅是seq2seq）和**很多任务**（不只是MT）

> **对注意力更一般的定义**:
> - 给定一个向量集合*values*，以及一个向量*query*，**注意力**在query上，计算values加权和技术

- 有时称为**query对values的注意力聚焦**.
- 例如，在 seq2seq+attention 模型中，每个解码器隐藏状态（query）关注所有编码器隐藏状态（values）

### 注意力的通用性 (续)

**注意力是一种通用的深度学习技术**

> **对注意力更一般的定义**:
> - 给定一个向量集合*values*，以及一个向量*query*，**注意力**在query上，计算values加权和技术

**直观上**:
- 加权和运算本质是对values向量信息的**选择性摘要**，由query向量动态决定关注焦点
- 注意力机制实现了**任意规模输入→定长表征**的转换(values向量)，其信息压缩过程始终以query向量为条件

**核心结论**:
- 注意力已成为深度学习模型中强大、灵活、普适的指路牌与记忆操作范式（2010年后源自神经机器翻译的重大创新）
